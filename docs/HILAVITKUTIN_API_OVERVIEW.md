# hilavitkutin-api ‚ÄĒ Design

Consumer-facing types, traits, and platform contracts. Consumers
(polka-dots, saalis, loimu) depend on this crate to define WorkUnits.
The engine crate (hilavitkutin) also depends on it. `#![no_std]`,
no alloc. All index, count, and size types use arvo newtypes.

Depends on `hilavitkutin-ctx` for its `provider_generic!` /
`provider_generic2!` macros and the `Context<P>` provider-tuple
wrapper. WU execute parameter types and the WU-context accessor
traits are expressed via those macros; see "WU Context providers"
below.

## Store types (domain 06)

Three store types, one scheduling mechanism:

```rust
pub struct Resource<T>(PhantomData<T>);  // singleton, one value
pub struct Column<T>(PhantomData<T>);    // collection, morsel-chunked
pub struct Virtual<T>(PhantomData<T>);   // zero-data, DAG edge only
```

The scheduler sees StoreId pairs for edge construction. Does not care
about cardinality. `Write<Resource<X>> ‚Üí Read<Resource<X>>` builds
the same edge as `Write<Column<Y>> ‚Üí Read<Column<Y>>`.

`StoreId` ‚ÄĒ dense index assigned at plan time. Wraps an arvo
fixed-point newtype (not raw u16). `AccessMask` ‚ÄĒ arvo-bitmask
`Mask64` (‚Č§64 stores) or `Mask256` (‚Č§256 stores). All scheduler ops
= single-instruction bitwise.

`AccessSet` ‚ÄĒ trait on tuples. `(Column<PosX>, Column<PosY>,
Resource<Gravity>)` implements AccessSet. Compile-time store
descriptors. Plan phase converts to runtime AccessMask via
monomorphised generic functions (the monomorphised function IS the
type identity ‚ÄĒ no TypeId, no std::any).

ZST erasure: column marker types constrain the DAG, enforce type
safety, and determine morsel sizes at compile time. Erased to plain
indices by execution time.

## ColumnValue trait (domain 05, R4)

```rust
pub trait ColumnValue: Copy + 'static {
    const BIT_WIDTH: usize;  // no default on trait itself
}

// Blanket impl via min_specialization
impl<T: Copy + 'static> ColumnValue for T {
    default const BIT_WIDTH: usize = core::mem::size_of::<Self>() * 8;
}
```

Any `Copy + 'static` type ‚Č§16 bytes is column-storable. Zero friction
for consumers. BIT_WIDTH informs the storage engine for bitpacking.
Sub-byte arvo types specialise (e.g. `UFixed<1,0>` ‚Üí BIT_WIDTH=1).
No pack/unpack ‚ÄĒ type IS the storage (R3, R4).

## WorkUnit trait (domain 09)

```rust
pub trait WorkUnit<Schedule = Always>: Send + Sync + 'static {
    type Read: AccessSet;
    type Write: AccessSet;
    type Hint: SchedulingHint = (Immediate, Atomic, Normal);
    type Ctx: HasColumnReader<Self::Read>
            + HasColumnWriter<Self::Write>
            + HasResourceProvider<Self::Read>
            + HasVirtualFirer<Self::Write>
            + HasEach<Self::Read, Self::Write>
            + HasBatch<Self::Read, Self::Write>
            + HasReduce<Self::Read, Self::Write>;
    const COMMUTATIVE: bool = false;
    fn execute(&self, ctx: &Self::Ctx);
}
```

No NAME const. WU identity is the type itself. Naming is a consumer
concern ‚ÄĒ they can implement their own trait or use wrappers.

Schedule conditions:
- `Always` ‚ÄĒ unconditional (default)
- `On<V>` ‚ÄĒ when virtual V fires

Write-write ambiguity resolved by scheduling hint priority across
three dimensions. Tie-break: most deps first, then deterministic
fallback.

WU authoring guideline: ‚Č§8 unique columns per WU (register pressure
guideline, not engine constraint).

## WU Context providers (domains 09, 19)

The WU execution parameter is a ctx-framework provider tuple ‚ÄĒ
`Self::Ctx` on the `WorkUnit` trait ‚ÄĒ bound by seven accessor
traits declared via `hilavitkutin-ctx`'s `provider_generic!` /
`provider_generic2!` macros. Each accessor carries the relevant
access-set type parameter so compile-time `Contains<S>` checks
can enforce WU-declared access at the call site.

Provider API traits (access-set-aware via ctx parameterised macros):

```rust
trait ColumnReaderApi<R: AccessSet> {
    unsafe fn read<T: ColumnValue>(&self, i: USize) -> T
    where R: Contains<Column<T>>;
}
trait ColumnWriterApi<W: AccessSet> {
    unsafe fn write<T: ColumnValue>(&self, i: USize, v: T)
    where W: Contains<Column<T>>;
}
trait ResourceProviderApi<R: AccessSet> {
    fn resource<T: 'static>(&self) -> &T
    where R: Contains<Resource<T>>;
}
trait VirtualFirerApi<W: AccessSet> {
    fn fire<V: 'static>(&self)
    where W: Contains<Virtual<V>>;
}
trait EachApi<R: AccessSet, W: AccessSet> {
    fn run<F>(&self, f: F);   // per-element, fusible
}
trait BatchApi<R: AccessSet, W: AccessSet> {
    fn run<F>(&self, f: F);   // full morsel slice, non-fusible
}
trait ReduceApi<R: AccessSet, W: AccessSet> {
    fn run<A, F>(&self, init: A, f: F) -> A;
}
```

Accessor traits (generated by `provider_generic!` /
`provider_generic2!`):

```rust
provider_generic!(<R: AccessSet> ColumnReaderApi as HasColumnReader => reader);
provider_generic!(<W: AccessSet> ColumnWriterApi as HasColumnWriter => writer);
provider_generic!(<R: AccessSet> ResourceProviderApi as HasResourceProvider => resources);
provider_generic!(<W: AccessSet> VirtualFirerApi as HasVirtualFirer => virtuals);
provider_generic2!(<R: AccessSet, W: AccessSet> EachApi as HasEach => each);
provider_generic2!(<R: AccessSet, W: AccessSet> BatchApi as HasBatch => batch);
provider_generic2!(<R: AccessSet, W: AccessSet> ReduceApi as HasReduce => reduce);
```

`Contains<S>` ‚ÄĒ sealed compile-time membership trait on tuples.
Accessing a store not declared in Read / Write fails to compile at
the relevant API-trait method's where-clause.

`&self` receiver on every provider method is load-bearing: prevents
LLVM noalias metadata that would reorder writes across fused WUs.
Raw-pointer access `unsafe read / write` pinky-swears ownership
discipline that the engine proves at plan time.

## Scheduling hints (domain 09)

Three axes as `repr(u8)` ZST carriers:

**Urgency:** `Immediate` (255), `Steady` (170), `Relaxed` (85),
`Deferred` (0).

**Divisibility:** `Atomic` (255), `Adaptive` (128),
`Interruptible` (0).

**Significance:** `Critical` (255), `Important` (192), `Normal` (128),
`Opportunistic` (64), `Optional` (0).

`SchedulingHint` trait on tuples: `(U: UrgencyValue, D:
DivisibilityValue, S: SignificanceValue)`. Lower discriminant = higher
priority.

## Platform traits (domain 02)

```rust
pub trait MemoryProvider: Send + Sync {
    unsafe fn allocate(&self, len: usize, align: usize) -> *mut u8;
    unsafe fn deallocate(&self, ptr: *mut u8, len: usize);
    unsafe fn protect(&self, ptr: *mut u8, len: usize,
                      read: bool, write: bool);
}

pub trait ThreadPool: Send + Sync {
    fn spawn(&self, f: impl FnOnce() + Send + 'static);
    fn worker_count(&self) -> usize;
}

pub trait Clock: Send + Sync {
    fn now_ns(&self) -> u64;
}
```

All consumed via monomorphisation. The Scheduler is generic over the
platform types. Storing `dyn MemoryProvider` would violate the dyn
ban. Each tier (no_os, os, std) ships defaults. Consumer overrides
via scheduler builder.

Note: `len`, `align`, `worker_count`, `now_ns` parameters and return
types should use arvo newtypes in the final API (e.g. `ByteCount`,
`Alignment`, `CoreCount`, `Nanoseconds`). Exact arvo type mappings
determined during src CL.

## Resource model (domain 07, R5)

```rust
pub struct Field<T: ColumnValue>(PhantomData<T>);  // scalar ‚Č§16 bytes
pub struct Seq<T, const N: usize>(PhantomData<T>);  // const-sized array
pub struct Map<K, V, const N: usize>(PhantomData<(K, V)>);  // const-sized map
```

No dynamic collections. Allocation size is const generic. Field<T>
is stack-local cached (LLVM promotes to registers). Seq/Map are
arena-allocated. Write resource collections count toward L1 morsel
budget; read-only ride L2 prefetcher.

Resource data is behind pointer indirection to external slab storage.
NOT inline in Context. Writing inline data through `&self` is UB.

## Virtual flag system (domain 10)

Bit-packed fire flags using arvo-bitmask types. One bit per registered
Virtual type. Packed into arvo words, grouped into Mask64/Mask256.
Hierarchical check: word == 0 ‚Üí skip. Affinity-ordered bit assignment
at plan time. Epoch-based reset (increment, not memset).

Per-(virtual, consumer) bit tracking. Clear-on-dispatch. Re-fire
during execution ‚Üí consumer runs again next pass. No lost fires.
