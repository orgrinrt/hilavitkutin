## COMPONENTS

**verkko-crypto** (Level 0, DAG root). No network concepts. **verkko-relay** (Level 3). Encrypted-at-rest storage + opaque transit forwarding. Defines TransitRelay (forwards without decrypting), Membrane (5 ops: PUT/GET/DELETE/TOUCH/SIGNAL, 90-day TTL, idempotent PUT), TerritorialReencryption (consistent hash ring, 32 vnodes/peer, ring coloring constraint checked every heartbeat), ConstantRatePadding (batch_size 10-25, HKDF-derived per macro-epoch).

## MECHANISMS

**Key distribution:** Admin wraps flat ChannelKeys in BroadcastEnvelope (52 bytes/recipient), distributes via KeyBundle + fencing token. Revoke > Grant > Rotate; 10-minute grace post-partition (Revoke-only for stale tokens). LKH tree activates at >= 48 peers.

**Epoch ratchet:** Macro-epoch computed independently (floor(hlc/3600_000)); sync via G-Counter in heartbeat extension. HKDF-Expand forward-only. Micro-epoch advances every 100 frames; resets on macro advance. CPSK re-seed injects fresh DH output into epoch via PCSMechanism, resets micro counter.

**Heartbeat as mesh bus:** Transport header carries metabolic state, gossip mode bit, convergence_active bit, admin bit, RTT, loss, jitter, queue depth, bandwidth, HLC timestamp. Extension region (mesh-owned) carries filter root, epoch, work_capacity, health canary, observation summary. Protocol owns byte layout; mesh owns semantic meaning of bits 2-4.

**Convergence:** Front 1 (control log merge + dominance recompute) gates Front 2 (KeyBundle + polygon CPSK renegotiation, 11 rounds, Lucas 1-factorization). Front 2 gates Front 3 (re-encryption: P1 4h/P2 24h/P3 bandwidth/P4 epoch-forward). Relay-dependent gate predicates cross document boundary via ConvergenceSignalProvider trait (defined in mesh, implemented by relay, injected at runtime; no new DAG edge).

**Content routing:** Keyed Merkle root in every heartbeat for divergence detection; incremental Merkle reconciliation on reconnect (~20 KB/pair after 4-hour partition). Source selection: possession AND access AND reachability AND health AND NOT assembling. Transfer: TransferRequest -> TransferBegin -> bao chunk stream. Integrity failure scars the source peer.

**Relay membrane:** relay_key_id = HMAC-BLAKE3(channel_salt, key_id)[0..8]; 90-day rotation. Re-encryption nonce = BLAKE3("saalis-reenc-v1" || key_new || blob_id || epoch)[0..24]; deterministic, making relay PUTs idempotent. Ring coloring constraint: no same-cluster adjacent vnodes; O(N) greedy reassignment, ~5us check per heartbeat.

**Resource governance (Conductor):** min(CPU, flash, nonce_io_pressure, WiFi_capacity). All resource-budget parameters wrapped in Governed<T> proving governor provenance. MetabolicGovernor is protocol-defined abstract contract; Conductor provides the concrete implementation. NaN in any input: retain current state.

## CONSTRAINTS

UDP payload limit: 1232 bytes (IPv6 MTU 1280 - 40 - 8). Run 11 corrected a 48-byte frame budget error; all prior calculations using 1280 were wrong.
No floating-point in the abstract state machine; fixed-point only (Q16.16 / Q8.24); PhysicalBoundary trait lint-enforced; 6 newtypes (SackRatio, CapacityRatio, TimeDelta, HealthScore, ScarWeight, DominancePenalty).
BLAKE3 domain separation mandatory; DomainSeparator repr(C) enum is the sole API; 9 registered variants; direct blake3::hash calls from protocol crates are forbidden.
Noise protocol names locked: Noise_KK_25519_ChaChaPoly_BLAKE2b / Noise_XX_25519_ChaChaPoly_BLAKE2b (BLAKE2b, not BLAKE2s; 64-bit hardware target).
Nonce reuse under ChaCha20-Poly1305 yields XOR plaintext recovery + universal forgery. Three-layer defense: pre-batch double-write persist (CRC-32C, two sectors), dynamic gap (factor 3), velocity halt at 100% gap ratio. Fail-stop on double corruption.
EpochRatchet strictly forward-only; K_epoch_N cannot be derived from K_epoch_{N+1}; previous keys zeroized after advance; frame keys zeroized immediately after single use.
NaN safety: any NaN in health pipeline, dominance, or governor inputs substitutes previous valid result or type default; no propagation.
Re-encryption nonces are deterministic (PRF over BLAKE3); transport nonces are sequential monotonic counters; the two constructions are disjoint.
Max 12 clusters, ~36 peers, hop_count max 4. No K_mesh; pairwise CPSK. Relay sees connection IDs and frame sizes only; no plaintext.
dyn vtable is forbidden across dlopen boundary; repr(C) descriptors only. Control log is GSet append-only; all entries Ed25519-signed by author DK.

## OPEN ITEMS

**D-003 (Run 12, HIGH):** SansIO polling model: calling convention, buffer ownership, buffer-exhaustion error semantics are unspecified. Two implementations can diverge. Highest-priority Run 12 item.
**D-004 (Run 12, HIGH):** Intra-cluster communication: dest_peer wire field exists; CIK-based Noise KK crypto exists. Session model and dest_peer assignment rules are unspecified. Potential interoperability failure within clusters.
**D-005 (Run 12):** Channel lifecycle: CHANNEL_CREATE entry type (0x0E) is reserved; creation semantics and authorization model are unspecified.
**D-007/D-008 (Run 13):** Relay transit forwarding and failover: hop_count field exists; multi-hop forwarding table, path selection, relay-failure discovery, and backup relay migration are all unspecified. Blocks WAN-only deployments.
**D-009 (Run 12):** Seat failover: host peer dies; seat has no specified recovery path.
**D-011 (Run 13):** DK encryption at rest: how the DK private key is stored encrypted on disk is unresolved.
**D-012 (Run 12):** MetabolicGovernor concrete implementation: abstract contract defined; Conductor parameter mapping is mesh body work.
**D-013 (Run 12):** Reliability mode selection per stream is unspecified.
**OQ-1:** Chunk size default: 1024-byte bao chunks recommended; chunk_size field in TransferBegin enables negotiation; explicit decision needed before merge.
**OQ-2:** Fixed-point for health scores: NaN guard handles the acute case; cross-platform EMA determinism evaluation deferred to Run 12.
**OQ-3:** Offline vs partition detection: PARTITION_ANN entry type exists; the trigger condition is unspecified; requires distributed systems design round.
**OQ-4:** SACK field ordering: hybrid format (W9) is drafted; explicit sign-off from both experts needed before final.
**Convergence body gaps:** Polygon schedule (11 rounds + extension rules) and the 6-step exit sequence are referenced in ops but lack body text in verkko-mesh.
**B10:** verkko-to-saalis application bridge is outside verkko scope; requires saalis design rounds.

## WIRE FORMAT

18 wire format specifications defined (W1-W18, Run 11 Part 5). All multi-byte integers are big-endian.
Key types: VersionProbe (W2, 18-28 bytes), VersionResponse (W3, 20 bytes), Noise Handshake Frame (W4, 17-byte fixed header, magic 0x56524B4F), Data Frame Outer Header (W5, 29 bytes), Encrypted Payload Inner Header (W6, ~10 bytes), Stream Header (W7, 7 bytes), HeartbeatExtension (W8, 18 bytes), SACK Hybrid (W9, 18-byte fixed + 8 bytes/block, max 146 bytes), Matter Chunk (W10, 52-byte fixed header + data + bao_proof), TransferRequest (W11, 44-byte header), TransferBegin (W12, 88-byte fixed), Control Log Entry (W15, 175-byte fixed overhead + payload + 64-byte Ed25519 signature), Invite Token (W16, 188 bytes + 20 bytes/hint + 64-byte admin signature), Relay Operation Header (W17, 13 bytes), SessionResumption Token (W18, 80-byte plaintext / 120-byte encrypted).
Frame budget: 29 outer + ~10 inner + 7 stream + 16 auth tag = 55 bytes typical overhead; 1170 bytes max data payload per frame.
BroadcastEnvelope (verkko-crypto, not in W-series): 849 bytes at 11 recipients; fits one 1232-byte UDP segment.
msg_type registry (16 values): HEARTBEAT(0x01) through CONN_CLOSE(0x10); bit 7 of msg_type signals piggybacked SACK.
