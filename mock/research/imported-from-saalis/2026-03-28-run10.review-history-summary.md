## 2. Consensus — Where All Converged

**Matter -> protocol dependency is undeclared.** All four reviewers confirmed independently.
Fix: add verkko-protocol to matter's Dependencies; specify MatterTransfer's mapping to Stream
(bulk tier) and LEDBAT pacing.

**Wire contracts are required before implementation.** Prose invariants do not constrain byte
layout. Grain boundary concepts (HeartbeatExtension, ConvergenceGate, SourceSelectionPredicate,
DomainSeparatorRegistry entries) need canonical layout + test vectors + canonical generator
before any code is written. Highest-priority output of the run.

**MetabolicState ownership.** Remove "see verkko-mesh: Conductor" from protocol's invariants.
Replace with an abstract governor contract. Conductor in mesh implements it. Resolves the
protocol-mesh soft cycle without merging documents.

**Named convergence fronts.** ControlFront, ConsensusFront, ReencryptionFront replace numbered
fronts. Position-dependent references break on insertion; named references are stable.

**HeartbeatExtension defined concept.** Six consumers share ownership of a fixed 18-byte layout
with no version field or extension mechanism. Needs its own defined concept.

## 3. Unresolved Disagreements

**DomainSeparatorRegistry severity.** SpecAuditor: MEDIUM (risk is zero in v1; catastrophic
only if a future developer violates the rule). DepAnalyst: LOW (rule is specified; enforcement
is implementation quality). Savant: design defect; the fix is a `DomainSeparator` enum as the
sole API surface, making unregistered usage unrepresentable at compile time rather than caught
at runtime. No resolution reached.

**Smooth vs cliff scar penalty.** SpecAuditor proposed `1 + max(0, weighted - 10)^1.5` to
eliminate the cliff at 10.0 weighted units. Savant rejected it: the threshold is an intentional
phase boundary. Smoothing creates an ambiguous transition zone between trusted and demoted.

**Convergence damping coefficients: constants vs calibration items.** SpecAuditor and GapHunter
argued 0.3x/0.1x are calibration items. Savant: the coefficients are tunable, but the existence
of the DAMPED mode is a protocol constant. Setting coefficients to 1.0/1.0 must be forbidden.

## 4. Savant Contributions — Novel Mechanisms and Evolution

**Stage 1** introduced grain boundaries (G1-G4), cleavage planes (C1-C4), and vacancies
(V1-V4). Key finding: all "strong bond" assessments are conditional on wire contracts. Without
them, the lattice is amorphous, not crystalline.

**Stage 2b** introduced four mechanisms:
- Convergence Damping Coefficient: health observations weighted 0.3x/0.1x during convergence.
  Prevents health-convergence feedback loop oscillation.
- Scar Severity Tiering: three tiers (transient 0.5x/90d, concerning 1.0x/180d, severe
  2.0x/365d). Prevents the exponential cliff from punishing temporary household hardware failures.
- Prioritized Wire Contracts: grain boundary concepts first, cross-document bonds second,
  internals third. GapHunter supplied the complementary implementation-priority ordering.
- Seat Isolation Valve: snapshot read path during convergence with write queuing and staleness
  flag. Formalized by SpecAuditor as the CONSISTENT/SNAPSHOT consistency model.

**Stage 3b** named five remaining fracture lines: matter -> protocol still unwritten; 27
BLOCKING wire-format gaps untouched; seat and matter bodies voids; convergence duration
unbounded; health pipeline internal dimensions undefined. Hardened: DAMPED mode is mandatory
(coefficients tunable, existence not). Rejected the smooth scar penalty. Proposed a
`DomainSeparator` enum as type-level constraint: unregistered usage is unrepresentable at
compile time rather than caught at test time.

## 5. Devil's Advocate Challenges — Assumptions Attacked and Resolution

**Defined concepts are not sufficient wire contracts.** EpochRatchet divergence (HKDF-Expand vs
full HKDF, both invariant-compliant) and FoundingHash divergence (LEN encoding ambiguity) are
concrete non-hypothetical failures. Resolution: all reviewers accepted Wire Contracts with no
substantive objection through three rounds.

**Seven documents may not map to seven crates.** 67% of pairwise boundaries have implicit
coupling; likely consolidation to three crates. No reviewer defended the seven-crate mapping.
Unresolved; treated as an implementation-time decision.

**Single health scalar is insufficient for five consumers.** A peer with high uptime and poor
throughput scores well for gateway election but poorly for control log delivery. Partial
resolution: purpose-specific projections are correct; the internal multi-dimensional model must
be specified first; v1 wraps the same scalar until calibration data is available.
