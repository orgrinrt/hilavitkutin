## §1 Plugin Architecture, Store, and the SDK Boundary

### §1.1 Plugin registration

Plugins register via `inventory` + `repr(C)` descriptors. `#[register]` on impl blocks generates descriptors. `HasDescriptor<D>` gates registration at compile time.

| Name | Kind | Purpose |
|------|------|---------|
| `Named` | trait | `const NAME: &'static str` |
| `Registrable` | trait | `Named + Display + Send + Sync + 'static` |
| `HasDescriptor<D>` | trait | Sealed compile-time registration gate |
| `SdkVersion` | struct (data) | ABI version: `major: u16, minor: u16`, `const CURRENT` |
| `Descriptor` | trait | Common `repr(C)` descriptor contract |
| `Registry` | trait | Typed registry: `iter`, `get`, `validate`, `validate_domain` |
| `#[register]` | macro | Universal registration for trait impl blocks |

## §19 Clarifications: Concrete Types → Trait Contracts

**Eliminated pipeline types:** `Discovered`, `EnrichCandidate`, `EnrichSummary`, `SourceCandidate`, `ResolvedSource`, `DownloadOutcome`, `ExtractedFile` — replaced by `ctx.query()` + `ctx.write()`.

**Eliminated containers:** `ExternalIdSet`, `BatchResult`, `BatchError` — query API and inline iteration.

**Eliminated trait hierarchies:** `Diagnosable`, `DiagnosticReport`, `ConnectorWorkUnit`, `DbWorkUnit`, `SystemWorkUnit`, `IntoEvents` — replaced by `WorkUnit` with `type Subsystem` and `type Emits: EventKindSet`.

**New filter ZSTs:** `NeedsEnrichment`, `NoUrl` — registered query filters.

**Valid concrete types:** IR values (`Color`, `Length`, `FontAsset`), credential impls (`ApiKey`, `OAuthToken`), settings derive targets, host-internal types, newtypes, ZSTs.
