## Phase 1: Registration infrastructure

### `saalis-derive/Cargo.toml`
Add dependencies: `syn = { version = "2", features = ["full"] }`,
`quote = "1"`, `proc-macro2 = "1"`, `darling = "0.20"`

### `saalis-derive/src/lib.rs`
Implement real `#[register]` proc macro:
- Parse `ItemImpl` to extract trait name, self type, generics
- Map trait name → descriptor type (ConnectorDescriptor, etc.)
- Generate extern "C" wrapper functions for each trait method
- Generate static descriptor instance with function pointers
- Generate `inventory::submit!` call
- Generate `HasDescriptor<D>` compile-time assert
- Error on non-impl items, non-trait impls, missing Named

### `saalis-sdk/src/lib.rs`
- Add real `impl` blocks for `Registry` default methods
- Add `Display` default impl for types implementing `Named`
- Wire `Descriptor` trait methods to read from struct fields
- Add `SdkVersion::is_compatible_with()` logic

## Phase 2: Schema and persistence

### `saalis-sdk/Cargo.toml`
Add dependency: `serde = { version = "1", default-features = false, features = ["derive", "alloc"] }`

### `saalis-sdk/src/lib.rs`
- Add `Serialize + DeserializeOwned` bounds to `ColumnValue`
- Add serde derives on `Str`, `EntityId`, `ExternalId`, `Port`, `RateLimit`
- Add `ColumnValue` impls with correct affinity for all base types

### `saalis-persistence/Cargo.toml`
Add dependencies: `rusqlite = { version = "0.32", features = ["bundled", "backup"] }`,
`sieve-cache = "0.2"`, `rusqlite_migration = "1"`,
`serde = "1"`, `serde_json = "1"`

### `saalis-persistence/src/lib.rs`
Implement:
- `ColdStore` struct with `rusqlite::Connection` writer + reader
- `ColdStore::open(path)` — create DB, set WAL pragmas, bootstrap schema table
- `ColdStore::generate_ddl(descriptors)` — CREATE TABLE from column descriptors
- `ColdStore::load_table::<M>()` — SELECT all, deserialize via serde
- `ColdStore::flush_dirty(dirty_set)` — INSERT OR REPLACE, serialize via serde
- `DirtySet` — `HashSet<(EntityId, &'static str)>` tracking dirty pairs
- `_saalis_schema` table: create if absent, version tracking
- Migration runner: execute OpDescriptors as SQL

## Phase 3: Context and scheduler

### `saalis-sdk/src/lib.rs`
- `ConnectorContext<C>` — add real method stubs that the host fills via function pointers
- `QueryBuilder<M>` — add `of()`, `filter()`, `execute()` method signatures
- `WriteBuilder<M>` — add `set()` method signature
- These are trait-object-free: the host constructs the context with concrete function pointers

### `saalis-scheduler/Cargo.toml`
Add dependencies: `parking_lot = "0.12"`, `flume = "0.11"`

### `saalis-scheduler/src/lib.rs`
Implement:
- `Dag` struct: adjacency list from WorkUnit triggers/emits
- `Dag::build(descriptors)` — construct from registry, topological sort
- `Scheduler` struct: owns DAG, cadence timers, event queue
- `Scheduler::tick()` — check timers, collect events, dispatch work units
- `Scheduler::dispatch(work_unit, context)` — call execute(), collect results
- Epoch flush trigger after batch

`tracing = "0.1"`, `tracing-subscriber = { version = "0.3", features = ["env-filter"] }`,

### `saalis-core/src/lib.rs`
Split into modules:
- `mod plugin` — inventory collection, descriptor validation
- `mod settings` — config loading, 7-layer resolution
- `mod notify` — notification dispatch (log-based v1)
- `mod storage` — file management (basic locations)

### `saalis-core/src/main.rs` (NEW)
Boot sequence:
1. Parse CLI args
2. Load config.toml
3. Open ColdStore
4. Collect and validate descriptors
5. Run migrations
6. Build scheduler DAG
7. Load initial data
8. Start axum server

## Execution order

1. Phase 1 (derive + SDK registration) — everything depends on this
2. Phase 2 (persistence) — data must persist before anything useful
3. Phase 3 (context + scheduler) — connectors need context to work
4. Phase 4 (core) — wires everything, provides main.rs
5. Phase 5 (web) — UI to see results
6. Phase 6 (extensions) — the actual connectors

`cargo check` after each phase. `cargo mock` after phase 6.
At the end: `cargo run` should boot, catalogue Steam games, and
display them in a browser gallery.
