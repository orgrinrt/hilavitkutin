# saalis-primitives — Design Round

**Date:** 2026-03-17
**Scope:** saalis-primitives only

## Current state

Nuked. Empty `src/lib.rs` stub. Empty `DESIGN.md.tmpl`.

The crate is `#![no_std]`, zero dependencies. It is the leaf of the
dependency graph — every other crate depends on it transitively.

## What belongs here

Raw `repr(C)` atoms that cross the ABI boundary. Nothing else.
No alloc, no traits, no behavior, no methods. Just data layouts
and constants.

### repr(C) enums

All enums have explicit discriminants. Lower value = higher
priority where an ordering relationship exists.

| Enum | Variants | Purpose |
|------|----------|---------|
| `Priority` | `Immediate=0`, `High=1`, `Normal=2`, `Low=3`, `Background=4` | Work unit scheduling priority |
| `ErrorClass` | `Transient=0`, `Fixable=1`, `Permanent=2` | Error taxonomy discriminant |
| `HealthStatus` | `Healthy=0`, `Degraded=1`, `Unhealthy=2` | Connector health |
| `Urgency` | `Critical=0`, `High=1`, `Normal=2`, `Low=3` | Notification urgency |
| `FieldKind` | `Text=0`, `Password=1`, `Number=2`, `Toggle=3`, `Select=4`, `Range=5`, `LocalePicker=6` | Settings UI field type |
| `Op` | `Eq=0`, `Ne=1`, `Lt=2`, `Le=3`, `Gt=4`, `Ge=5` | Query filter comparison operators |
| `SettingsScope` | `Global=0`, `System=1`, `Profile=2` | Settings resolution depth |

### ABI version constant

`SAALIS_SDK_ABI_VERSION: u32` — packed as `major << 16 | minor`.
Current value: `0x0001_0000` (version 1.0).

Used by the host to validate descriptor compatibility at startup.
The descriptor's `sdk_version` field is a const pointer to this
value (see saalis-sdk design).

### What does NOT belong here

- Traits (those go in saalis-sdk)
- `Str` or any alloc types (those go in saalis-sdk)
- Methods on enums (behavior goes in saalis-sdk)
- Newtypes like `EntityId` (those go in saalis-sdk)
- Anything that needs `alloc` or `std`

## Questions

None — this crate is straightforward. The enums and constant are
the same as what we had before the nuke. No design changes needed.
