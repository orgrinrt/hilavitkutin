# Crate Split: saalis-contracts extracted from saalis-sdk

**Date:** 2026-03-17
**Context:** Circular dependency between sdk and derive. SDK
re-exports #[register] from derive, derive needs to know about
SDK's trait contracts.

## Decision

Split the SDK into two crates:

- **saalis-contracts** — all trait contracts, ZSTs, newtypes,
  descriptor structs. `no_std + alloc`. The actual content.
- **saalis-sdk** — thin facade: `pub use saalis_contracts::*` +
  `pub use saalis_derive::register`. What extensions depend on.

## Updated dependency graph

```
saalis-primitives           (no_std, zero deps)
       ↑
saalis-contracts            (no_std + alloc — traits, ZSTs, newtypes)
       ↑       ↑
saalis-derive  |            (proc-macro — depends on contracts)
       ↑       |
saalis-sdk                  (facade: re-exports contracts + derive)
     ↑  ↑  ↑
     |  |  saalis-scheduler
     |  |       ↑
     |  saalis-persistence
     |       ↑
  saalis-core
       ↑
  saalis-web

Extensions → saalis-sdk only
```

## What goes where

| Crate | Contains |
|-------|----------|
| `saalis-primitives` | repr(C) enums, ABI const |
| `saalis-contracts` | ALL trait definitions, ZSTs, newtypes, descriptor structs, ColumnValue, Str |
| `saalis-derive` | `#[register]` proc macro, `#[derive(Settings)]` |
| `saalis-sdk` | `pub use saalis_contracts::*; pub use saalis_derive::register;` |
