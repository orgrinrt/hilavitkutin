## Context

saalis is pivoting from a client-server model to a peer mesh. Every
peer runs the same binary with the same runtime, configured for
different roles. This topic covers the architectural changes needed
across the crate taxonomy, verkko protocol, and contracts.

## Scope

### Peer model
- No server/client distinction — every peer is capable
- Role-configured via toml (connector, downloader, player, relay, ui)
- Same binary, same hilavitkutin runtime everywhere
- "Dumb client" model dissolved

### Trust and topology
- Cluster = household (1-6 devices, LAN)
- Inter-cluster = friend mesh (WAN, ~12 clusters, ~36 peers)
- Trust tiers: household (full), friend (partial+verify), relay (transport)
- Gateway election: QoS scoring + 10% threshold + peer ID tiebreak
- Transit relay: peers route for each other from connectivity graph

### Verkko mesh extensions
- verkko-gossip: PlumTree broadcast, sans-IO (from iroh-gossip)
- verkko-blobs: BLAKE3 verified streaming (from iroh-blobs bao-tree)
- verkko-relay: relay protocol, transit forwarding, NAT traversal
- verkko-client: dissolves or becomes verkko-peer
- NAT traversal: OBSERVED_ADDR in INIT_ACK, hole punch, relay-first
- DHT discovery via Mainline DHT (BEP 44)
- Fallback chain: mDNS → invite hint → cached peers → DHT → relay

### Encryption
- Two tiers: plain+MAC (heartbeats) and ChaCha20-Poly1305 AEAD (everything else)
- Benchmarked: AEAD is fastest option on modern hardware
- Path validation: E2EE PING/PONG for connection migration
- Key hierarchy: CIK, CSS, CPSK, RSK, DK
- Invite: ticket + 6-emoji SAS verification
- Relay storage: salted hash keys, 90-day rotation

### Contracts reorganisation
- Dissolve host/client/shared module split
- Every peer needs connector traits, instruction handling, storage
- Unified peer contracts — no module distinction by role
- hilavitkutin-ctx provider pattern unchanged

### Crate taxonomy
- saalis-server → saalis (one binary)
- saalis-web → optional browser UI peer
- verkko-client → dissolves or becomes verkko-peer
- New: verkko-gossip, verkko-blobs, verkko-relay
- New: saalis-mesh (peer roles, content maps, sync)
- Workspace Cargo.toml changes
- Dependency graph update
