## Pass 2.5 (reverted): Cap-typed engine const generics

Hypothesis: flip every engine `<const MAX_*: usize>` to `<const MAX_*: Cap>` with `[(); cap_size(MAX_X)]:` where-clauses and `[T; cap_size(MAX_X)]` arrays. Apply across all 27 engine + api source files. The change brings the engine→arvo Pass-3 boundary into Cap→Cap shape so `BandwidthReducer::reduce_bandwidth` / `BlockPartitioner::partition_blocks` / `SpectralBipartitioner::bipartition` can be called over the engine's `MAX_UNITS` directly, with no `cap_of(MAX_UNITS)` bridge (which triggers rustc ICE `arvo_sparse::rcm::rcm_reorder::{constant#1}` panic when propagated through associated types).

Empirical result during apply: with the cap-typing applied across all const-generic positions in the engine plus where-clause expansion across every impl/struct/fn header, `cargo check -p hilavitkutin` rustc-time exceeded 22 minutes without completing. Pattern signature: rustc consumed ~98% CPU for the entire window, indicating the trait solver was iterating over the cartesian product of `where [(); cap_size(MAX_X)]:` constraints across the engine's deep generic stacks. This is the failure mode `.claude/rules/arvo-compile-time-last.md` names: compile-time blow-up is the wrong escape valve when the runtime cost it buys is small relative to the dev-iteration cost it inflicts. Reverted.

Path forward: deferred. Three viable lines:
1. Arvo offers `usize` bridge adapters (`fn reduce_bandwidth_usize<const N: usize>(...) where [(); N]:`). Keeps engine const-generics as `usize`. The bridge is then inside arvo, not propagated through every engine call site.
2. Engine Cap refactor lands behind a feature gate (`cap-typed-engine`) so the slow compile only affects builds that explicitly opt in. Defaults stay on usize+stubs.
3. The Cap refactor lands as a tractable per-crate slice (api first, then plan, then dispatch) with intermediate cargo-check confirmations to keep regression bounded.

None of those is in scope for this megaround. Pass 3 proceeds with `rcm_reorder` / `block_diagonalise` / `spectral_partition` as passthrough stubs; `PlanError::PhaseAlignmentMismatch` is reachable via direct graph-shape construction in the test, not via the arvo path. Task #121 (`cap_size lint:allow revisit`) remains open and absorbs the deferral.

This pass-as-attempted was reverted. The src CL section documents the attempt + the compile-time data so future sub-rounds know which approaches were tried and why they were backed off.
