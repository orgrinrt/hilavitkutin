## `mock/crates/hilavitkutin-str`

### `Cargo.toml`

Add `arvo-bits.workspace = true` under `[dependencies]`.

### `src/handle.rs`

Rewrite the `Str` definition and `impl`:

```rust
use arvo_bits::Bits;

#[repr(transparent)]
#[derive(Copy, Clone, Eq, PartialEq, Hash, Debug, Default)]
pub struct Str(Bits<32>);

impl Str {
    pub const CONST_ORIGIN_FLAG: Bits<32> = Bits::new(0);
    pub const RUNTIME_MASK: Bits<32> = Bits::new(1u64 << 31);
    pub const ID_MASK: Bits<32> = Bits::new(0x0FFF_FFFF);

    #[doc(hidden)]
    pub const fn __make(id: u32) -> Self {
        // lint:allow(no-bare-numeric) lint:allow(arvo-types-only)
        // reason: macro-expansion entry — str_const!() produces a
        // masked u32 from the FNV-1a hash and passes it here verbatim;
        // tracked: #72
        Self(Bits::new((id as u64) & Self::ID_MASK.bits()))
    }

    #[doc(hidden)]
    pub const fn __runtime(id: u32) -> Self {
        // lint:allow(no-bare-numeric) lint:allow(arvo-types-only)
        // reason: same macro-expansion entry path; tracked: #72
        Self(Bits::new(((id as u64) & Self::ID_MASK.bits()) | Self::RUNTIME_MASK.bits()))
    }

    pub const fn is_const(self) -> bool {
        (self.0.bits() & Self::RUNTIME_MASK.bits()) == 0
    }

    pub const fn is_runtime(self) -> bool {
        !self.is_const()
    }

    pub const fn id(self) -> u32 {
        // lint:allow(no-bare-numeric) lint:allow(arvo-types-only)
        // reason: returns the 28-bit ID projected from the Bits<32>
        // container; callers hand-off to u32-typed consumer APIs
        // (hash maps keyed on u32); tracked: #72
        (self.0.bits() & Self::ID_MASK.bits()) as u32
    }

    #[doc(hidden)]
    pub const fn raw_bits(self) -> u64 {
        self.0.bits()
    }
}
```

### `tests/handle.rs`

Rewrite all `.0` accesses to use semantic API:

- `Str::default()` — compare with `is_const() == true`, `id() == 0`, `is_runtime() == false`.
- `Str::__make(0xFFFF_FFFF)` — check `id() == 0x0FFF_FFFF` (masked) and `is_const()`.
- `Str::__make(0x0012_3456)` — check `id() == 0x0012_3456` and `is_const()`.
- `Str::__runtime(0x1234)` — check `is_runtime()` and `id() == 0x1234`.
- `Str::__runtime(0xFFFF_FFFF)` — check `is_runtime()` and `id() == 0x0FFF_FFFF` (masked).
- Round-trip test (`__make(0x42)` vs `__runtime(0x42)` differ) — check `a != b` and both `id() == 0x42`.
- Identity test (`__make(7)` == `__make(7)`) — `PartialEq` derived.

The `raw_bits()` accessor is available if a future test needs the raw bit pattern; none of the current tests do.

### `tests/str_const.rs`

Current assertion `assert_eq!(h.0 & !Str::ID_MASK, 0)` becomes:

```rust
assert_eq!(h.raw_bits() & !Str::ID_MASK.bits(), 0);
```

`Str::ID_MASK.bits()` returns the u64 projection from `Bits<32>`; the test uses bitwise ops on u64 which is internal-structure logic, not public API.

### `src/macros.rs`

No change to the macro body: `Str::__make(...)` still accepts u32.

### `src/interner.rs`

No change: interner calls `Str::__runtime(id)` with a u32 ID.
