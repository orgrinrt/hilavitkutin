**Scope:** `ExtensionDescriptor` function-pointer slots — `Option<fn>` to `MaybeNull<fn>`

# Current state

`hilavitkutin-extensions/src/descriptor.rs` declares two nullable function-pointer fields on the `#[repr(C)] ExtensionDescriptor`:

```rust
pub init_fn: Option<unsafe extern "C" fn(host_ctx: *mut c_void) -> ExtensionAbiStatus>,
pub shutdown_fn: Option<unsafe extern "C" fn(host_ctx: *mut c_void) -> ExtensionAbiStatus>,
```

Both carry `lint:allow(no-bare-option)` annotations tracked under #206. The justification text records that `Option<fn>` was the only Rust-level type with a documented null-pointer-niche ABI guarantee at the time; `notko::Maybe` carried no such guarantee, so substituting it would have changed the over-the-wire layout.

# Changes proposed

Swap both fields to `MaybeNull<unsafe extern "C" fn(...)>` and drop the two `lint:allow(no-bare-option)` annotations.

`notko::MaybeNull<T: NicheFilled>` (shipped in notko `dev`) is a `#[repr(transparent)]` newtype over `Maybe<T>` with a sealed `NicheFilled` marker trait. The sealed trait admits only payload types whose invalid bit pattern is all zeros: `&T`, `&mut T`, `NonNull<T>`, `NonZero*`, and `extern` / `unsafe extern` / plain / `unsafe` `fn` pointers of arities 0..=8. Per-instantiation `const _LAYOUT_ASSERT` evaluates `size_of::<MaybeNull<T>>() == size_of::<T>()` at compile time, and the module pins each supported shape via `const _: () = MaybeNull::<T>::_LAYOUT_ASSERT;`. Runtime tests verify that `MaybeNull::isnt()` transmutes to all zeros for every shape.

Vocabulary alignment: the stack already says "one word for presence, uniform across the stack" (see `lib.rs` module docs). `Option<fn>` is the last remaining public API escape hatch inside the hilavitkutin crates. Replacing it closes that gap without layout cost.

Host-side call sites currently pattern match via `if let Some(f) = descriptor.init_fn`; they migrate to `if let Maybe::Is(f) = descriptor.init_fn.into_maybe()`. The proc-macro emission in `hilavitkutin-extensions-macros` emits `Some(fn_ident)` / `None`; it migrates to `::hilavitkutin_extensions::MaybeNull::new(fn_ident)` / `::hilavitkutin_extensions::MaybeNull::isnt()`. The macro emission routes through a `doc(hidden)` re-export of `notko::MaybeNull` from `hilavitkutin-extensions` so consumer crates do not need a direct `notko` dep solely for the emission path.

Lifecycle prose in `DESIGN.md.tmpl` currently reads "If `None`, init is a no-op." Updated to "If absent (`MaybeNull::isnt()`), init is a no-op." — same semantics, reflects the new type.

# Non-goals

Notko's `MaybeNull` API naming (`isnt()` / `is_some()` / `is_none()`) is being revised in notko PR #2 to read naturally as pointer vocabulary (`null()` / `is_non_null()` / `is_null()`). The rename lands downstream of this round; a follow-up sync round here will track it.
