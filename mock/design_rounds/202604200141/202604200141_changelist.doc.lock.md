## Purpose

Land the type + trait skeleton for hilavitkutin-persistence. The
five DESIGN-listed types (`ColdStore`, `SieveCache`, `StringTable`,
`Manifest`, `PersistenceContext`) ship as concrete types with
stub bodies. Full rkyv-serialisation and mmap-backed file I/O
defer to a follow-up round once a concrete consumer (saalis or
engine dogfooder) drives the wire-format decisions.

Reasoning for skeleton-first:
- Unblocks the hilavitkutin engine (Task #35) from depending on
  persistence types without waiting for the full impl.
- Removes the five nuked-state `design-doc-source-mismatch`
  push-gate violations.
- Encapsulates the rkyv integration in a separate round where the
  no_std + no_alloc discipline can be proven against rkyv's
  concrete constraints without derailing other work.
- Matches the validated "small rounds" pattern: one cohesive
  concern per round.

## Deliverables this round

- `ColdStore` struct — carries the data directory path + a
  handle-typed interner + MemoryProvider handle. Public methods:
  `open`, `flush`, `load`, `snapshot` — all stubbed (TODO-bodied
  via `#[doc(hidden)]`-wrapped `todo!()` or explicit
  `unimplemented!`). Signatures match DESIGN.
- `SieveCache<K, V>` struct — wraps a `ColumnStorage`-style host
  type. WeightedSieveCache algorithm + `insert` / `evict` / `get`
  methods stubbed.
- `StringTable` struct — `(content_hash, bytes)` pair archive
  type. Concrete layout defined; archive format stubbed behind an
  internal `Archive` impl.
- `Manifest` struct + `TableMeta` inner struct — both with the
  fields DESIGN specifies.
- `PersistenceContext` struct — bundles the interner + memory
  provider references as DESIGN describes.
- A `PersistenceError` enum with variants for the common failure
  modes: IO, Archive, Invariant, Missing.
- `Archive` impl for `Str` — DESIGN's identity+content-hash
  translation. This is the simplest real archive impl and can
  ship now since it's pure logic on `Str`'s bit layout.
- Unit tests: 
  - `Manifest` construction + serde-style round trip
    (archive-stub level).
  - `PersistenceContext` accessor methods.
  - `Str` archive: runtime handle → content hash → runtime
    handle (via test-local interner).
  - `SieveCache` eviction order (in-memory only, no rkyv).

## Deferred to follow-up round

- Full rkyv `Archive` implementations for `Manifest`, `TableMeta`,
  `StringTable`. Requires rkyv 0.8's no_std+no_alloc modes vetted
  against MemoryProvider-backed buffers.
- Actual file I/O (mmap via MemoryProvider). Depends on engine
  round shipping a concrete MemoryProvider that supports
  `mmap` / `munmap` / `protect`.
- Weighted eviction policy tuning (current stub returns
  FIFO as a safe default).
- `snapshot` backup mechanism.

All deferrals captured in `BACKLOG.md.tmpl`.
