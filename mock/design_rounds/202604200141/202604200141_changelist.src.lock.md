### 5. `src/archive_str.rs`

Concrete `Archive` impl for `Str` per DESIGN's identity +
content-hash translation. This is the ONLY real archive impl
this round — pure bit-layout logic, no rkyv dependency yet.

```rust
use hilavitkutin_str::{Str, ArenaInterner, StringInterner, const_fnv1a};

/// Evict a Str handle to its on-disk u32 representation.
///
/// Const handles (bit 31 = 0) carry content hashes directly (28-bit
/// FNV truncation); pass through.
/// Runtime handles (bit 31 = 1) need to be resolved to bytes, then
/// content-hashed from those bytes.
pub fn evict_str<A: ArenaInterner>(
    handle: Str,
    interner: &StringInterner<A>,
) -> u32 {
    if handle.is_const() {
        handle.id()
    } else {
        let s = interner.resolve(handle);
        (const_fnv1a(s) & 0x0FFF_FFFF) as u32
    }
}

/// Inject a content-hashed u32 back as a runtime handle.
```

## Impl-time decisions (pre-committed)

1. **No rkyv dep this round.** Skeleton compiles without it; the
   real archive impls follow in a dedicated round where rkyv's
   no_std + no_alloc story gets vetted against MemoryProvider.
2. **Const-sized arrays everywhere.** MAX_TABLES = 256,
   MAX_COLUMNS_PER_TABLE = 64 as starting constants. Consumer
   demand shapes future widening.
3. **SieveCache uses FIFO ordering** as the safe stub. Real SIEVE
   scan lands with weighted-eviction tuning.
4. **`ColdStore` methods return stub values.** `open` succeeds
   with a default Manifest; `load` returns Missing; `flush` and
   `snapshot` are no-op Ok. Tests exercise the signatures, not
   behaviour.
5. **`evict_str` and `inject_str` are real.** They're pure
   bit-layout logic on `Str` and can ship now.
6. **No `Archive` trait defined here.** The DESIGN-level `Archive
   trait` description is aspirational; rkyv provides the real
   trait when it enters. This round's `evict_str` / `inject_str`
   are free functions, not trait impls.
