# Source changelist: arvo Rounds 1-8 adoption + typed-const completion

Mechanical adoption round. Per `cl-claim-sketch-discipline.md`, every
substantive shape change carries a `## CHANGE:` block with verification.

The work has three groups:

1. Pin advance (Cargo.lock).
2. Mask64 surface deletion (api-level alias was dead surface; engine
   already has its own const-generic AccessMask<MAX_STORES>).
3. Typed-const adoption in Depth (USize::ZERO / R::D + USize::ONE)
   plus the `const_trait_impl` feature gates that let those calls
   compile in const context.

## CHANGE: arvo + notko git-dep pins advance from `52261d8` / `30bfd17` to current dev HEADs

File: `mock/Cargo.lock`
Verification: `grep 'source = "git+ssh://git@github.com/orgrinrt/arvo' mock/Cargo.lock | sort -u` returns one line ending in `91c314e1` or later. Same shape for notko returns one line ending in `adc0e306` or later.

Rationale: covers arvo Rounds 1-8 (substrate-fundamentals
const-trait lift, Mask chassis collapse, Bounded blanket, etc.).
Driver command: `cargo update -p arvo -p arvo-bits -p arvo-bitmask
-p arvo-graph -p arvo-sparse -p arvo-spectral -p arvo-comb -p
arvo-hash -p arvo-refit -p notko`.

## CHANGE: api-level `AccessMask` alias deleted

File: `mock/crates/hilavitkutin-api/src/id.rs`
Verification: `! grep "Mask64\|AccessMask" mock/crates/hilavitkutin-api/src/id.rs` returns success (no matches).

Rationale: the alias `pub type AccessMask = Mask64;` is unused
substrate. Engine ships its own const-generic
`AccessMask<const MAX_STORES: usize>` at
`mock/crates/hilavitkutin/src/plan/access.rs:18`. Workspace-wide
grep confirms zero downstream consumers. Per
`no-legacy-shims-pre-1.0.md` the alias is deleted, not migrated to
`Mask<Bits<64, Hot, Unsigned>>`. The `use arvo_bitmask::Mask64;`
import goes too.

Module doc comment line 4 (`scheduler operates on AccessMask
bitwords`) no longer applies and is removed.

## CHANGE: hilavitkutin-api enables `const_trait_impl` feature

File: `mock/crates/hilavitkutin-api/src/lib.rs`
Verification: `grep "feature(const_trait_impl)" mock/crates/hilavitkutin-api/src/lib.rs` returns one match.

Rationale: arvo's USize const-Add and Identity surface are
conditionally-const. Calling `R::D + USize::ONE` and referencing
`USize::ZERO` from the `Depth` impls requires the consumer crate to
opt into the feature gate.

## CHANGE: Depth impl `const D: USize = USize(0)` migrates to typed `USize::ZERO`

File: `mock/crates/hilavitkutin-api/src/builder.rs:165`
Verification: `grep "const D: USize = USize::ZERO" mock/crates/hilavitkutin-api/src/builder.rs` returns one match. `! grep "const D: USize = USize(0)" mock/crates/hilavitkutin-api/src/builder.rs` returns success.

Rationale: arvo Round 7 (#50) shipped
`impl const Identity for USize` via the Bounded blanket forward.
PR #46's reviewer flagged this site as nit 1; the workaround
`USize(0)` was forced because arvo had not yet shipped the typed
constant.

## CHANGE: Depth recursive impl migrates from `USize(R::D.0 + 1)` to typed `R::D + USize::ONE`

File: `mock/crates/hilavitkutin-api/src/builder.rs:168`
Verification: `grep "const D: USize = R::D + USize::ONE" mock/crates/hilavitkutin-api/src/builder.rs` returns one match. `! grep "USize(R::D.0 + 1)" mock/crates/hilavitkutin-api/src/builder.rs` returns success.

Rationale: arvo Round 1 (#42) shipped `impl const Add for USize`
plus the Identity blanket. The construction
`USize(R::D.0 + 1)` (raw-field addition then re-wrap) is
unnecessary once typed `+` works in const context.

## CHANGE: Depth doc comment matches new shape

File: `mock/crates/hilavitkutin-api/src/builder.rs:102`
Verification: `grep "<()>::D == USize::ZERO" mock/crates/hilavitkutin-api/src/builder.rs` returns one match, and `grep "R::D + USize::ONE" mock/crates/hilavitkutin-api/src/builder.rs` returns at least one match.

Rationale: keeps the rustdoc surface aligned with the typed-const
shape the Depth impls now use.

## CHANGE: hilavitkutin-str enables `const_trait_impl` feature

File: `mock/crates/hilavitkutin-str/src/lib.rs`
Verification: `grep "feature(const_trait_impl)" mock/crates/hilavitkutin-str/src/lib.rs` returns one match.

Rationale: the `arvo_bits::bitfield!` macro expansion for
`StrLayout` now invokes conditionally-const associated functions
(`<u32 as BitPrim>::mask_low`, `<Bits<32> as ConstPartialEq>::const_eq`)
inside the const fn bodies it generates. Without the feature gate
the consumer crate can't compile the macro expansion against the
post-Round-1 substrate.

## Validation pass

After all the above changes land, the validation gate is
`cargo check --workspace --tests` from `mock/`. The previously-
observed errors all resolve:

- `error[E0432]: unresolved import arvo_bitmask::Mask64` is gone
  because the import and the consuming alias were removed.
- `error[E0658]: const trait impls are experimental` and the
  conditionally-const-call companions are gone because the consumer
  feature gate is now declared.

The library check green is the lock criterion. Test-side rehab
(`#291`) is a separate concern; this round does not chase it.
