# Src CL: arvo-refit adoption + #290 lint:allow sweep

**Date:** 2026-04-30
**Round:** 202604301630
**Phase:** SRC-PLAN

Mechanical adoption of the new arvo refit API. Three classes of
edit:

1. Workspace `Cargo.toml` and per-crate `Cargo.toml`s flip
   `arvo-narrow` to `arvo-refit`.
2. Two source files flip `use arvo_narrow::Narrow;` to
   `use arvo_refit::Narrow;` (and pull `Widen` where the sweep
   adds widen calls).
3. Four `lint:allow tracked: #290` annotations removed; the
   underlying call sites rewritten to use `Widen<T>::widen_to`,
   `BitsRefitCtor::from_widened`, or `BitsRefitCtor::from_narrowed`
   from the post-arvo-PR-#40 surface.

### `mock/crates/hilavitkutin-persistence/src/archive_str.rs`

Three call-site edits.

Replace `use arvo_narrow::Narrow;` with `use arvo_refit::{Narrow, Widen};`.

Site 1 (line 29):

```rust
ContentHash::from_raw(handle.id().to_raw() as u64) // lint:allow(...)
```

becomes:

```rust
ContentHash::from_raw(handle.id().widen_to())
```

Site 2 (line 34):

```rust
ContentHash::from_raw(const_fnv1a(s) & (Str::ID_MASK.to_raw() as u64)) // lint:allow(...)
```

becomes:

```rust
ContentHash::from_raw(const_fnv1a(s) & Str::ID_MASK.widen_to())
```

Site 3 (line 65):

```rust
let lookup_hash = ContentHash::from_raw(masked_bits.to_raw() as u64); // lint:allow(...)
```

becomes:

```rust
let lookup_hash = ContentHash::from_raw(masked_bits.widen_to());
```

The `widen_to()` calls compile via the cross-Bits Widen impl
(when the source is a `Bits<M, S, Sign>`) or the cross-domain
primitive-to-Bits impl (when the source is a u32 widening to u64).
The destination type is inferred from the `from_raw(u64)` argument
position.

### `mock/crates/hilavitkutin-str/src/macros.rs`

Two call-site edits inside `str_const!`.

Lines 22-24 (the `__ENTRY` static):

```rust
hash: $crate::Str::__make(::arvo_bits::Bits::<28>::from_raw(
    ($crate::const_fnv1a($s) & 0x0FFF_FFFF) as u32, // lint:allow(...)
)),
```

becomes:

```rust
hash: $crate::Str::__make(::arvo_bits::Bits::<28>::from_narrowed(
    $crate::const_fnv1a($s)
)),
```

Lines 27-29 (the returned handle):

```rust
$crate::Str::__make(::arvo_bits::Bits::<28>::from_raw(
    ($crate::const_fnv1a($s) & 0x0FFF_FFFF) as u32, // lint:allow(...)
))
```

becomes:

```rust
$crate::Str::__make(::arvo_bits::Bits::<28>::from_narrowed(
    $crate::const_fnv1a($s)
))
```

`Bits::<28>::from_narrowed::<u64>` (with `u64` inferred from the
arg type) calls `narrow_to::<28>` on u64, which masks to 28 bits
and casts to the dispatched 28-bit container (u32 under `Hot`).
The explicit `& 0x0FFF_FFFF` mask and the `as u32` cast both fold
into the trait machinery.
