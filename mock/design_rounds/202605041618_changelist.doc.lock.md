Error shapes are fixed enum variants with no heap payloads. OS error codes are carried as `arvo::USize` values (non-negative on both unix errno and Windows GetLastError ranges). On Windows, `LinkError::LoadFailed { platform_code }` always reflects `GetLastError()` at the failure site. On Unix, the value is the thread-local `errno` read via the per-libc helper (`__errno_location` on Linux/Android, `__error` on Darwin and BSD); since POSIX does not require `dlopen` to set `errno`, the value is best-effort and reflects the most recent libc syscall failure during the load. **After:**

```markdown
Error shapes are fixed enum variants with no heap payloads. OS error codes are carried as `arvo::USize` values (non-negative on both unix errno and Windows GetLastError ranges). On Windows, `LinkError::LoadFailed { platform_code }` always reflects `GetLastError()` at the failure site. On Unix, the value is the thread-local `errno` read via the per-libc helper (`__errno_location` on Linux/Android, `__error` on Darwin and BSD); since POSIX does not require `dlopen` to set `errno`, the value is best-effort and reflects the most recent libc syscall failure during the load. On unsupported unixes (Solaris, Haiku, etc.) the value is `0`. `LinkError::PathEncodingUnsupported` distinguishes a path the loader cannot transcode from a path that the OS loader rejects after handoff. v1 fires it on Windows for non-ASCII paths and paths exceeding the classic `MAX_PATH` (260 wchars); Unix paths are opaque bytes and never produce it. Full UTF-8 to UTF-16 transcoding plus `\\?\` LongPath support is a separate scope.
```
