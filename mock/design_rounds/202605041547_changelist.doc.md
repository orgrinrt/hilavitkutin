# Doc changelist: surface real errno on Unix dlopen failure

This is a small round. The shipping API surface does not change. The only doc edit is a clarification of the `LinkError::LoadFailed { platform_code }` semantics on Unix to capture that the value is best-effort errno, not POSIX-mandated.

Error shapes are fixed enum variants with no heap payloads. OS error codes are carried as `arvo::USize` values (non-negative on both unix errno and Windows GetLastError ranges).

**After:**

```markdown
Error shapes are fixed enum variants with no heap payloads. OS error codes are carried as `arvo::USize` values (non-negative on both unix errno and Windows GetLastError ranges). On Windows, `LinkError::LoadFailed { platform_code }` always reflects `GetLastError()` at the failure site. On Unix, the value is the thread-local `errno` read via the per-libc helper (`__errno_location` on Linux/Android, `__error` on Darwin and BSD); since POSIX does not require `dlopen` to set `errno`, the value is best-effort and reflects the most recent libc syscall failure during the load. On unsupported unixes (Solaris, Haiku, etc.) the value is `0`.
```
