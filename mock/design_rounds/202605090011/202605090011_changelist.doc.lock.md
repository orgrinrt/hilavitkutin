# Doc CL — descriptor v1.1 abi hardening

Doc changes for round 202605090011. The src CL follows lock.

The audit's F1+F2+F3 findings on `ExtensionDescriptor` are closed by:

- `tag: u32` (literal ASCII "HILE" encoded little-endian, value
  `0x454C4948`) as the descriptor's first field.
- `descriptor_size: u32` as the second field, recording the
  `core::mem::size_of` of the descriptor at extension build time.
  The host validates `descriptor_size >= host's size_of` (greater-or-equal,
  not equality, to permit forward-evolution into a v2 layout that
  appends fields without invalidating v1.1 readers).
- `abi_version: AbiVersion` (was bare `u32`). The newtype is
  `#[repr(transparent)]` over `u32`, wire-identical. `HOST_ABI_VERSION`
  bumps from `AbiVersion(1)` to `AbiVersion(2)`.
- `name_len`, `capabilities_len`, `required_host_caps_len` all flip
  from `USize` to bare `u32`, fixing the pointer-width-dependent wire
  layout (F1).
- New `pub const MAX_DESCRIPTOR_LIST_LEN: u32 = 1 << 20;` ceiling for
  every length field. Validated at descriptor-read time (F3).
- New `ExtensionError` variants:
  `DescriptorTagMismatch { found: u32 }`,
  `DescriptorSizeTooSmall { advertised: u32, minimum: u32 }`,
  `DescriptorBoundsExceeded { field: &'static str, len: u32 }`.

Per the no-legacy-shims-pre-1.0 rule, v1 disappears outright; no
deprecation alias for `AbiVersion(1)`, no v0 read path. Extensions
recompile.
