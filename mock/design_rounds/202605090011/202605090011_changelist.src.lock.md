# Src CL — descriptor v1.1 abi hardening

Source changes that materialise the v1.1 ExtensionDescriptor shape
locked in the doc CL. Per the cl-claim-sketch-discipline workspace
rule, every type-shape change carries a structured `## CHANGE:` block
with a verification command.

## Layout

The new `ExtensionDescriptor` field order is fixed and load-bearing.
The host's read order depends on the layout: tag first (so a wrong
layout cannot pass any subsequent check by coincidence),
descriptor_size second (so the host knows what range of bytes it can
trust), abi_version third (so the v1.1 prefix is established before
field-layout-dependent reads happen), the rest after.

```rust
#[repr(C)]
pub struct ExtensionDescriptor {
    pub tag: u32,
    pub descriptor_size: u32,
    pub abi_version: AbiVersion,
    pub name_ptr: *const u8,
    pub name_len: u32,
    pub version: ExtensionVersion,
    pub capabilities_ptr: *const CapabilityEntry,
    pub capabilities_len: u32,
    pub required_host_caps_ptr: *const CapabilityId,
    pub required_host_caps_len: u32,
    pub init_fn: Option<unsafe extern "C" fn(host_ctx: *mut c_void) -> ExtensionAbiStatus>,
    pub shutdown_fn: Option<unsafe extern "C" fn(host_ctx: *mut c_void) -> ExtensionAbiStatus>,
}
```

## Tests added

### `mock/crates/hilavitkutin-extensions/tests/descriptor_validation.rs` (new file)

The test fixture builds three malformed descriptors at static memory and feeds them through the host load path's validation helper directly (the helper is private; test access is via a `#[cfg(test)] pub` re-export in the lib root, scoped to test config so the public surface stays unchanged).

- `tag_mismatch_surfaces_DescriptorTagMismatch`: descriptor with `tag = 0xDEADBEEF`. Expects `ExtensionError::DescriptorTagMismatch { found: 0xDEADBEEF }`.
- `size_too_small_surfaces_DescriptorSizeTooSmall`: descriptor with valid tag, `descriptor_size = 16` (smaller than `core::mem::size_of::<ExtensionDescriptor>()`). Expects `ExtensionError::DescriptorSizeTooSmall { advertised: 16, minimum: <host's size_of> }`.
- `bounds_exceeded_name_surfaces_DescriptorBoundsExceeded`: descriptor with valid tag, valid size, valid version, `name_len = MAX_DESCRIPTOR_LIST_LEN + 1`. Expects `ExtensionError::DescriptorBoundsExceeded { field: "name", len: MAX_DESCRIPTOR_LIST_LEN + 1 }`.
- `bounds_exceeded_capabilities_surfaces_DescriptorBoundsExceeded`: same shape but on `capabilities_len`. Expects `field: "capabilities"`.
- `bounds_exceeded_required_host_caps_surfaces_DescriptorBoundsExceeded`: same shape but on `required_host_caps_len`. Expects `field: "required_host_caps"`.
- `valid_descriptor_passes`: a fully-valid descriptor with all bounds satisfied. Expects `Outcome::Ok(())`.
