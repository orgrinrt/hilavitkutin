# Doc Changelist: hilavitkutin-api sink / collector / codec contracts

**Date:** 2026-04-20
**Phase:** DOC
**Scope:** `hilavitkutin-api` doc templates + root mockspace templates.
**Topic:** `202604201048_topic.sink-collector-codec-contracts.md`
**Type:** additive; no breaking change to existing API surface.

## Motivation

The `hilavitkutin-api` crate currently covers pipeline-execution
contracts (access/context/hint/platform/store/work_unit). Public APIs
outside pipeline execution — clause's parser, validator, codegen —
have no trait contract for "receive items of type T" or "emit bytes".
The 2026-04-20 fidelity audit surfaced `Vec<SyntaxError>` and
`Vec<u8>` returns in shipped code as a direct consequence. This round
adds the missing contract layer so the sweeps (#71/#72/#73) can
reshape those returns to trait-bound sinks.

All decisions come from the committed topic file. Scope is maximal:
capability atoms, three named composites, sink combinators, stateful
codec pair with oneshot extension, tuple blanket impls (arity 2+3).

## Changes proposed

### 1. `mock/crates/hilavitkutin-api/DESIGN.md.tmpl`

**Insert a new major section** before `## What this crate does NOT contain`
and after `## Virtual flag system (domain 10)`. Title:

```
## Consumer contract surface (capability / sink / codec)
```

The section documents:

- The trait-first framing (short paragraph: "every position that
  accepts items is a trait bound, not a concrete container").
- Capability atoms module (`capability`): `Push<T>`, `BulkPush<T>`,
  `Len`, `Capacity`, `BoundedPush<T>`, `Full`.
- Named composites module (`sink`): `Collector<T>`, `DiagnosticSink<E>`,
  `ByteEmitter` (with blanket impls over the matching capability
  bounds), and the three combinators `NullSink`, `CountingSink<T>`,
  `TeeSink<'a, A, B>`.
- Codec module (`codec`): `Encoder<T>`, `Decoder<T>`, `EncoderExt<T>`,
  `DecoderExt<T>`, `DecodeError` (`#[non_exhaustive]` enum:
  `Truncated`, `Invalid`, `OverLength`).
- Short worked example: `fn parse<E: DiagnosticSink<SyntaxError>>(...)`
  where previously `parse() -> Result<_, Vec<SyntaxError>>`.
- Named as the "contract surface" to reinforce the saalis-contracts
  positioning.

Approx length: 90–110 lines. Uses code blocks that mirror the topic
file sketch.

### 2. `mock/crates/hilavitkutin-api/DESIGN.md.tmpl` — resource-model note

Append one paragraph to `## Resource model (domain 07, R5)` clarifying
that `Seq<T, N>` / `Map<K, V, N>` are **type-level descriptors** (not
containers), and consumers receive items into them via `Push<T>` /
`BoundedPush<T>` implementations driven by the scheduler's arena.
This closes an inferrable-but-unwritten gap that caused the drift.

### 3. `mock/crates/hilavitkutin-api/DESIGN.md.tmpl` — platform-trait cleanup

types should use arvo newtypes in the final API". This round is
**not** the one that lands those renames — that's the hilavitkutin
sweep (#72). Expand the note to reference #72 explicitly so readers
