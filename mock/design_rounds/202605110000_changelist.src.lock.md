## CHANGE: migrate test fixtures to USize / Bool / Nanos / Slot wrappers

Files: `crates/hilavitkutin/tests/{core_assignment,plan_inputs,adapt_config,dispatch_types,resource_accumulator,resource_snapshot,strategy,thread_pool,platform_os}.rs`
Verification: `cargo test -p hilavitkutin --tests` runs every test file to completion with zero failures. `cargo test --test platform_os --features platform-os -p hilavitkutin` covers the os-tier file independently.

Migration patterns applied uniformly:

- Bare `0` against an arvo `USize` field replaced with `USize::ZERO`
  (importing `arvo::Identity` for the `ZERO` const).
- Bare integer literal `N` against an arvo `USize` field or argument
  replaced with `USize(N)`, suffixed with
  `// lint:allow(no-bare-numeric) reason: ...; tracked: #399`.
- Bare integer literal against `FiberId(pub u16)` / `PhaseId(pub u8)`
  field stays bare (the source already wraps the bit-width). Suffixed
  with `// lint:allow(no-bare-numeric) reason: ...; tracked: #399`.
- `assert!(x.is_empty())` where `is_empty()` returns `Bool` replaced
  with `assert_eq!(x.is_empty(), Bool::TRUE)` (and `Bool::FALSE`
  symmetrically).
- `assert!(!*b)` over `b: &Bool` replaced with
  `assert_eq!(*b, Bool::FALSE)` (Bool does not impl `Not`).
- `Slot(N)` replaced with `Slot(UFixed::from_raw(N))` since
  `Slot(pub UFixed<64,0,Hot>)`. Read-side comparisons go through
  `.0.to_raw()` because `UFixed` lacks `Debug` for `assert_eq!`.
- `assert_eq!(metrics.phase_completion_time_ns, 0)` replaced with
  `assert_eq!(metrics.phase_completion_time_ns.to_raw(), 0)` for
  the same `Debug` reason.
- `b >= a` over two `Nanos` values replaced with comparison of
  `b.to_raw() >= a.to_raw()` since `UFixed` lacks `PartialOrd` + the
  `Display` for the panic message.
- `f.body.is_none()` (where `body: Maybe<...>`) replaced with
  `f.body.isnt()`. Notko `b8509c9` exposes `is()` / `isnt()`, not
  `is_some()` / `is_none()`.
- `AdaptConfig::default()` replaced with explicit
  `AdaptConfig::new(PhaseId(0), USize::ZERO, USize::ZERO, USize::ZERO)`
  since the source dropped the `Default` impl.
