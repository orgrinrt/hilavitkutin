## Impl-time decisions

1. **u128 backing for AccessMask + DirtyMask** — skeleton. arvo-
   bitmask swap is a BACKLOG entry; keeping the surface API
   (empty/set/contains/overlaps/union_with) stable across the
   swap lets downstream impl rounds land first.
2. **Bool-matrix DependencyGraph** — skeleton. arvo-graph CSR swap
   is a BACKLOG entry; same surface-API stability argument.
3. **Twelve step functions as stubs** — each lands in a separate
   downstream round so PRs stay small and reviewable.
4. **`build_plan` body is `todo!()`** — the orchestration of the
   12 steps is itself a BACKLOG entry (depends on all step bodies
   existing first).
5. **No test coverage for `todo!()` surfaces** — noisy; would need
   `#[should_panic]`. Skip entirely.
6. **Manual Debug impl on `DependencyGraph`** — `[[bool; N]; N]`
   is not auto-Debug for large N. Impl prints "DependencyGraph
   { size: N }" only in skeleton.
