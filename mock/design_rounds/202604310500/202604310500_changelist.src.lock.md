# Source Changelist: Str forwards mask constants to StrLayout

**Date:** 2026-04-21
**Phase:** SRC
**Topic:** `202604310500_topic.str-layout-mask-forward.md`
**Doc lock:** `202604310500_changelist.doc.lock.md`

## `mock/crates/hilavitkutin-str/src/handle.rs`

- Drop `Str::CONST_ORIGIN_FLAG` (no consumer uses it).
- Change `Str::RUNTIME_MASK: Bits<32, Hot> = Bits::<32>::new(1u64 << 31)`
  to `Str::RUNTIME_MASK: Bits<32, Hot> = StrLayout::origin_MASK`.
- Change `Str::ID_MASK: Bits<32, Hot> = Bits::<32>::new(0x0FFF_FFFF)`
  to `Str::ID_MASK: Bits<32, Hot> = StrLayout::id_MASK`.

Values are identical; definitions move to StrLayout. Consumer names
preserved.

## `mock/crates/hilavitkutin-persistence/src/archive_str.rs`

Switch `Bits::<28>::new(value)` call sites to `value.into()` where
the context is non-const. Three sites in `inject_str`:

- `let masked_bits = Bits::<28>::new(content_hash.bits() & Str::ID_MASK.bits());`
  → `let masked_bits: Bits<28, Hot> = (content_hash.bits() & Str::ID_MASK.bits()).into();`
- `let back = Bits::<28>::new(const_fnv1a(resolved) & Str::ID_MASK.bits());`
  → `let back: Bits<28, Hot> = (const_fnv1a(resolved) & Str::ID_MASK.bits()).into();`

## `mock/crates/hilavitkutin-str/src/interner.rs`

- `Str::__runtime(Bits::<28>::new(id as u64))` → `Str::__runtime((id as u64).into())`
  (non-const call path inside `intern()` / `intern_static()`).
- `Str::__make(Bits::<28>::new(const_fnv1a(s) & 0x0FFF_FFFF))` in
  `lookup_const_by_value` — same treatment via `.into()`.

## `mock/crates/hilavitkutin-str/tests/handle.rs`

Tests that currently write `Bits::<28>::new(0xFFFF_FFFF)` for clarity
stay as-is — explicit typing reads better in tests than type-elided
`.into()`.

## Verification

- `cargo test -p hilavitkutin-str` — all handle / interner / ergonomics
  / str_const tests pass.
- `cargo test -p hilavitkutin-persistence` — archive_str tests pass.
- `cargo mock --lint-only` — no new drift; dropped mask constants do
  not regress any lint position.
