# Current state

`mock/crates/hilavitkutin/src/strategy/mod.rs` has one trait
`StrategySelector` with one method `select`, plus `DefaultSelector`
implementing it:

```rust
pub trait StrategySelector {
    fn select(&self, record_count: u64, depth: u32, fibers: u32, roots: u32) -> Strategy;
}
```

Four bare numeric params: `record_count: u64`, `depth: u32`,
`fibers: u32`, `roots: u32`. The lint reports four violations (two per
line × two overlapping lints).

# Changes proposed

Replace all four bare primitives with `arvo::USize`. Each parameter is
semantically a count; the variable name carries the domain meaning so
a single `USize` substitution reads clearly at the call site.

The const `10_000` literal inside the selector body stays a bare
`usize` integer literal — `arvo-types-only` and `no-bare-numeric`
focus on type positions, not integer literals. The comparison
`record_count < 10_000` dereferences via `USize: Deref<Target=usize>`
(already provided by arvo). Arithmetic like `depth / fibers / 2`
continues to work through deref.

`Strategy` and `PhaseStrategy` enum discriminants are unchanged.

# Non-goals

- No new semantic alias — `record_count: USize` already self-documents
  via the parameter name; an opaque `RecordCount` newtype would add
  nothing here.
- No changes to the threshold `10_000` or comparison logic.
- No doc template changes (source-only round).
