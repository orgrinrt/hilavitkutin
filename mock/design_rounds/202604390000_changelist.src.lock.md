## `morsel.rs`

Import `arvo::USize`.

- `pub start: u64` → `pub start: USize`
- `pub len: u32` → `pub len: USize`
- `new(start: u64, len: u32)` → `(start: USize, len: USize)`
- `end() -> u64` → `end() -> USize`. Body: `self.start + self.len as
  u64` → `USize(*self.start + *self.len)` (deref both, wrap result).
  Can no longer be const — or arvo::USize supports arithmetic. Check
  post-edit and add lint:allow if needed.
- `is_empty() -> bool` — body `self.len == 0` becomes `*self.len == 0`.

Drop `Hash` derive if arvo USize doesn't provide it. Default also.

## `progress.rs`

Imports: `use arvo::USize;`.

- `new(start: usize) -> Self` → `new(start: USize) -> Self`. Body
  `AtomicUsize::new(start)` → `AtomicUsize::new(*start)`.
- `store(&self, value: usize)` → `(&self, value: USize)`. Body:
  `self.0.store(value, Ordering::Release)` → `self.0.store(*value,
  Ordering::Release)`.
- `load(&self) -> usize` → `load(&self) -> USize`. Body:
  `self.0.load(Ordering::Acquire)` → `USize(self.0.load(Ordering::
  Acquire))`.

`AtomicUsize` itself stays — it's an atomic-memory ABI concrete. The
type in the `use` statement triggers no lint (imports are scanned
differently).
