## 1. `Str` bit-layout — `pub u32` → private `Bits<32>`

### Current (hilavitkutin-str/src/handle.rs)

```rust
#[repr(transparent)]
#[derive(Copy, Clone, Eq, PartialEq, Hash, Debug, Default)]
pub struct Str(pub u32);

impl Str {
    pub const CONST_ORIGIN_FLAG: u32 = 0;
    pub const RUNTIME_MASK: u32 = 1 << 31;
    pub const ID_MASK: u32 = 0x0FFF_FFFF;
    pub const fn __make(id: u32) -> Self { Self(id & Self::ID_MASK) }
    pub const fn __runtime(id: u32) -> Self { Self((id & Self::ID_MASK) | Self::RUNTIME_MASK) }
    pub const fn is_const(self) -> bool { (self.0 & Self::RUNTIME_MASK) == 0 }
    pub const fn is_runtime(self) -> bool { !self.is_const() }
    pub const fn id(self) -> u32 { self.0 & Self::ID_MASK }
}
```

Tripped lints: `no-public-raw-field` (pub u32), `arvo-types-only` (every u32), `no-bare-numeric`.

### Proposed

```rust
use arvo_bits::Bits;

#[repr(transparent)]
#[derive(Copy, Clone, Eq, PartialEq, Hash, Debug, Default)]
pub struct Str(Bits<32>);  // private field

impl Str {
    /// Origin-flag bit pattern for const-interned handles (all zeros).
    pub const CONST_ORIGIN_FLAG: Bits<32> = Bits::new(0);
    /// Mask with bit 31 set — runtime-origin handles.
    pub const RUNTIME_MASK: Bits<32> = Bits::new(1u64 << 31);
    /// Mask for the 28-bit ID (bits 27-0).
    pub const ID_MASK: Bits<32> = Bits::new(0x0FFF_FFFF);

    #[doc(hidden)]
    pub const fn __make(id: u32) -> Self { ... }
    #[doc(hidden)]
    pub const fn __runtime(id: u32) -> Self { ... }
    pub const fn is_const(self) -> bool { ... }
    pub const fn is_runtime(self) -> bool { ... }
    pub const fn id(self) -> u32 { ... }
    /// Raw bit-pattern (low 32 bits of a `u64`). For tests and debug only.
    #[doc(hidden)]
    pub const fn raw_bits(self) -> u64 { self.0.bits() }
}
```

The `__make` / `__runtime` builders continue to take `u32` for ergonomics — the macro sites (`str_const!()`) already emit a `u32` from the FNV-1a hash. Internally they construct a `Bits<32>` via `Bits::new(value as u64)`.

The masks keep u32-ranged semantic values but carry the `Bits<32>` type. `raw_bits()` is a new escape hatch for the existing structural tests, but the assertions inside them collapse to `h.id() == expected_id` and `s.is_runtime()` — far cleaner than masking against `.0`.

Adds a dep: `arvo-bits.workspace = true` in `hilavitkutin-str/Cargo.toml`. Per `.claude/rules/cargo.md`, extension crates may depend on arvo substrate; this is the first such dep for hilavitkutin-str.

### Decision D1

`Bits<32>` — not `UFixed<32, 0, Hot>`. Str is not an arithmetic value; it is a packed identity with a structural bit layout (flag + reserved + ID). `Bits<N>` is the arvo substrate for "opaque N-bit pattern"; Str fits that shape exactly. A future dedicated `Bitfield` primitive that carries named sub-ranges would be an even better fit — tracked as task #127, not in scope here.

## 2. Scheduling-hint axis discriminants — exact-width `UFixed`

### Current (hilavitkutin-api/src/hint.rs)

```rust
pub trait UrgencyValue: ... {
    const VALUE: u8;
}
impl UrgencyValue for Immediate { const VALUE: u8 = 255; }
impl UrgencyValue for Steady    { const VALUE: u8 = 170; }
impl UrgencyValue for Relaxed   { const VALUE: u8 = 85; }
impl UrgencyValue for Deferred  { const VALUE: u8 = 0; }

pub trait DivisibilityValue: ... { const VALUE: u8; }
impl DivisibilityValue for Atomic        { const VALUE: u8 = 255; }
impl DivisibilityValue for Adaptive      { const VALUE: u8 = 128; }
impl DivisibilityValue for Interruptible { const VALUE: u8 = 0; }

pub trait SignificanceValue: ... { const VALUE: u8; }
impl SignificanceValue for Critical      { const VALUE: u8 = 255; }
impl SignificanceValue for Important     { const VALUE: u8 = 192; }
impl SignificanceValue for Normal        { const VALUE: u8 = 128; }
impl SignificanceValue for Opportunistic { const VALUE: u8 = 64; }
impl SignificanceValue for Optional      { const VALUE: u8 = 0; }
```

Drift count: 12 `u8` positions (3 trait items + 9 impl items).

Each axis has a known, fixed cardinality:

| Axis | Markers | Count | Min bits |
|------|---------|------:|---------:|
| Urgency | Immediate, Steady, Relaxed, Deferred | 4 | 2 |
| Divisibility | Atomic, Adaptive, Interruptible | 3 | 2 |
| Significance | Critical, Important, Normal, Opportunistic, Optional | 5 | 3 |

u8 wastes 6 bits per axis for urgency/divisibility and 5 bits for significance. Over a pipeline with many WUs, packing three axes into 7 bits (plus 1 reserved) instead of 24 bits materially reduces the scheduling-hint footprint.

### Proposed

Three domain aliases, each with the narrowest `UFixed<N, 0, Hot>` that covers the axis:

```rust
/// How soon the WU must run. 4 levels fit in 2 bits.
pub type Urgency = UFixed<2, 0, Hot>;

/// Whether the WU can be split or paused. 3 levels fit in 2 bits.
pub type Divisibility = UFixed<2, 0, Hot>;

/// Relative importance of the WU's output. 5 levels fit in 3 bits.
pub type Significance = UFixed<3, 0, Hot>;
```

Each trait's `VALUE` const becomes the matching alias:

```rust
pub trait UrgencyValue: ... {
    const VALUE: Urgency;
}
impl UrgencyValue for Immediate { const VALUE: Urgency = Urgency::from_raw(3); }
impl UrgencyValue for Steady    { const VALUE: Urgency = Urgency::from_raw(2); }
impl UrgencyValue for Relaxed   { const VALUE: Urgency = Urgency::from_raw(1); }
impl UrgencyValue for Deferred  { const VALUE: Urgency = Urgency::from_raw(0); }
```

Discriminant values collapse from u8-space (0, 85, 170, 255) to their natural 2-bit enumeration (0, 1, 2, 3) — the old spacing existed only to fit the u8 container; the exact-width type encodes the natural ordering directly. `PartialOrd` on `UFixed<N, 0, Hot>` lifts from the inner storage so the ordering property (`Immediate > Steady > Relaxed > Deferred`) holds.

Analogous for Divisibility (3 levels → 0, 1, 2) and Significance (5 levels → 0, 1, 2, 3, 4).

### Decision D2

Use domain aliases, not bare arvo primitives, at the trait-item positions. This is the semantic-alias rule from `feedback_semantic_aliases_over_naked_newtypes.md`: a trait field typed `UFixed<2, 0, Hot>` has no name; typed `Urgency` it has one.

### Decision D3

Discrete values (0..n-1), not spaced values. The old 255/170/85/0 spacing was u8-container-thinking; the new alias carries exactly as many bits as the domain needs, and the natural ordinal is the discriminant. Tests switch from `VALUE == 255` to `VALUE == Urgency::from_raw(3)`.
