# Current state

Each hilavitkutin crate has a `BACKLOG.md.tmpl` documenting deferred
design items. Entries often mention types (Str, USize, hint axes),
but there's no explicit reminder keeping new entries conformant with
the type-surface rules. Future contributors can quietly drift into
`u32` / `bool` / `Option` / bare `&str` shorthand in promissory notes,
which leaks into implementations when the promise is picked up.

# Changes proposed

Prepend a single compact reminder block to every BACKLOG.md.tmpl:

```markdown
> **Type-surface reminder**: new entries use `arvo::USize` /
> `Bool` / `UFixed<N, 0, S>` instead of `usize` / `bool` / `u*`;
> `notko::Maybe` / `Outcome` instead of `Option` / `Result`;
> `hilavitkutin_str::Str` instead of `&str` / `String`. See
> `rules/type-surface.md`.
```

Six hilavitkutin BACKLOG files touched. The Str reminder is
hilavitkutin-specific; the arvo-half of the sweep (#92) uses the
same block without the Str line.

# Non-goals

- No retroactive rewriting of existing entries.
- No lint change.
