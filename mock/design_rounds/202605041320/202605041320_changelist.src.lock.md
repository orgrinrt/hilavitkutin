# Source changelist: PR #48 review fixes

Three findings, four `## CHANGE:` blocks. Per
`cl-claim-sketch-discipline.md`.

## CHANGE: arena_intern byte-buffer write uses pointer-arithmetic-only access

File: `mock/crates/hilavitkutin-providers/src/interner.rs`
Verification: `grep -A1 "Append bytes to the buffer" mock/crates/hilavitkutin-providers/src/interner.rs | grep "self.bytes.get() as \*mut u8"` returns one match. `! grep "(\*self.bytes.get()).as_mut_ptr()" mock/crates/hilavitkutin-providers/src/interner.rs` returns success (the old `&mut` reborrow shape is gone).

The byte-buffer write changes from
`(*self.bytes.get()).as_mut_ptr().add(cursor)` to
`(self.bytes.get() as *mut u8).add(cursor)`. Symmetric with
`arena_resolve`'s read which already used raw-pointer-only
access. Eliminates the `&mut [u8; BYTES]` reborrow that would
conflict with a live `&str` borrow returned from a previous
`arena_resolve` call under Stacked Borrows / Tree Borrows.

The accompanying SAFETY comment is updated to call out the
Stacked Borrows reasoning explicitly: append-only allocator
invariant means the bytes at `[cursor..cursor+len]` cannot be
read by any outstanding `&str`, since outstanding handles point
into prior ranges only.

## CHANGE: arena_intern entry-table write uses pointer-arithmetic-only access

File: `mock/crates/hilavitkutin-providers/src/interner.rs`
Verification: `grep "entries_ptr.add(count).write(Entry" mock/crates/hilavitkutin-providers/src/interner.rs` returns one match. `! grep "let entries = &mut \*self.entries.get()" mock/crates/hilavitkutin-providers/src/interner.rs` returns success.

The entry-table write changes from
`let entries = &mut *self.entries.get(); entries[count] = ...`
to
`let entries_ptr = self.entries.get() as *mut Entry; entries_ptr.add(count).write(...)`.
Same Stacked Borrows reason as the byte-buffer write.

`arena_resolve`'s read does
`(self.entries.get() as *const Entry).add(idx).read()` returning
an `Entry` by value, so no shared borrow into the entry table
remains live across an intern. The symmetric write here keeps
the unsafe surface honest.

## CHANGE: lib.rs module doc comment replaces em-dash with colon

## CHANGE: smoke test exercises resolve-borrow soundness

File: `mock/crates/hilavitkutin-providers/tests/smoke.rs`
Verification: `grep "fn resolve_borrow_survives_subsequent_intern" mock/crates/hilavitkutin-providers/tests/smoke.rs` returns one match.

The new test holds a `&str` returned by `arena_resolve` live
across a subsequent `arena_intern` call and asserts the prior
borrow remains valid. This test would have failed under
Miri (Stacked Borrows / Tree Borrows) on the pre-fix
implementation; it documents the soundness invariant the round's
unsafe-surface fixes preserve.
