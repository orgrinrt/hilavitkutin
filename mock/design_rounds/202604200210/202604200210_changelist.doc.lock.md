## Purpose

First engine sub-round. Lands the platform-tier implementations of
hilavitkutin-api's MemoryProviderApi / ThreadPoolApi / ClockApi.

Three tiers per DESIGN domain 02:

- **os tier** (default) — raw syscalls: `mmap` / `munmap` for
  memory, `pthread_create` + `pthread_join` for threads,
  `clock_gettime(CLOCK_MONOTONIC)` for Clock. Linux + macOS
  only.
- **std tier** (fallback) — `std::alloc` + `std::thread` +
  `std::time::Instant`. Windows + any platform without the
  syscalls.
- **no_os tier** — trait contracts only; the consumer ships its
  own impls. Just documentation + feature flag.

Tier selection via cfg feature flags on the hilavitkutin crate.

## Deliverables

- `platform::os` module — Linux + macOS raw-syscall impls.
- `platform::std_tier` module — std-based impls (`std_tier` since
  `std` is a reserved name).
- Cfg feature flags: `platform-os` (default), `platform-std`,
  `platform-no-os`. Mutually exclusive via `compile_error!`.
- `platform::OsMemoryProvider` / `OsThreadPool` / `OsClock` types.
- `platform::StdMemoryProvider` / `StdThreadPool` / `StdClock` types.
- Tests covering:
  - OsMemoryProvider allocate/deallocate round-trip.
  - OsClock monotonic progression.
  - StdMemoryProvider likewise.
  - StdClock likewise.
  - (ThreadPool tests are hard under no-op workunits; defer
    to the dispatch round 5a3 which needs the pool for real.)
