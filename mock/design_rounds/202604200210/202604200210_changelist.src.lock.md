### 2. `src/platform/mod.rs` — tier dispatch

```rust
#[cfg(all(feature = "platform-os", not(any(feature = "platform-std", feature = "platform-no-os"))))]
mod os;
#[cfg(all(feature = "platform-std", not(any(feature = "platform-os", feature = "platform-no-os"))))]
mod std_tier;

#[cfg(feature = "platform-os")]
pub use os::{OsMemoryProvider, OsThreadPool, OsClock};
#[cfg(feature = "platform-std")]
pub use std_tier::{StdMemoryProvider, StdThreadPool, StdClock};
// no_os tier: no re-exports; consumer provides its own impls.
```

### 3. `src/platform/os.rs` — raw syscalls

`OsMemoryProvider` impls `hilavitkutin_api::MemoryProviderApi` via:
- `allocate`: `libc::mmap(null, len, PROT_READ|PROT_WRITE, MAP_PRIVATE|MAP_ANON, -1, 0)`.
- `deallocate`: `libc::munmap(ptr, len)`.
- `protect`: `libc::mprotect(ptr, len, prot_flags)` — stub to Ok
  this round (BACKLOG full integration).

`OsThreadPool` impls `ThreadPoolApi` via:
- `spawn(f)`: `libc::pthread_create(...)`. Generic over the
  closure via a trampoline; stored in a Box<dyn FnOnce>? No —
  hilavitkutin bans dyn. Instead: monomorphised spawn per
  consumer; the skeleton here uses a thread-local slot pattern.
  Full thread-pool with work-queue integration lands in 5a4.
- `worker_count()`: returns a fixed value (1) for skeleton;
  follow-up round connects to `sysconf(_SC_NPROCESSORS_ONLN)`.

`OsClock` impls `ClockApi` via:
- `now_ns()`: `libc::clock_gettime(CLOCK_MONOTONIC)`.

Tests cover allocate / deallocate / clock monotonicity. ThreadPool
tests defer to 5a4.

### 4. `src/platform/std_tier.rs` — std fallback

`StdMemoryProvider` via `std::alloc::{alloc, dealloc, Layout}`.
`StdThreadPool` via `std::thread::spawn` (via fn pointers + an
internal launcher to avoid dyn).
`StdClock` via `std::time::Instant::now()`.

Same test surface.

### 5. Tests

- `tests/platform_os.rs` (cfg gated) — allocate/dealloc/clock.
- `tests/platform_std.rs` (cfg gated) — same, with std-tier.

## Impl-time decisions (pre-committed)

1. **Only one platform tier active at a time** — exclusivity
   enforced at crate root via `compile_error!`.
2. **`OsThreadPool::spawn` skeleton** — uses a minimal pthread
   trampoline over a `fn()` thin-pointer; real generic-closure
   support lands with 5a4. For now `spawn` accepts only a
   parameterless `fn()`.
3. **`OsMemoryProvider::protect` stubs to Ok** — real mprotect
   wiring lands with the persistence mmap-file round.
4. **`worker_count`** returns 1 in this skeleton. 5a4 connects
   to sysconf.
5. **no_os tier has no src** — it's pure trait contracts from
   hilavitkutin-api. The feature flag exists so consumers can
   build without any platform tier when they ship their own.
