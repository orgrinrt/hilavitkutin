## Impl-time decisions

1. **Skeleton types only** — ResourceSnapshot holds u64 placeholder
   slots; real type erasure + column-value-width-specific storage
   lands when 5a2/5a3 expose the WU-side data layout needs.
2. **fn-pointer combiner** (no closure) — generic-closure support
   lands with 5a4.
3. **NonNull<T> newtypes** (ResourcePtr / ColumnPtr) — LLVM noalias
   support via `#[repr(transparent)]` wrappers.
