### 4. AccessSet sealed trait (access.rs)

```rust
pub trait AccessSet: sealed::Sealed + 'static {
    const LEN: usize;
}

mod sealed { pub trait Sealed {} }
impl sealed::Sealed for () {}
impl<A: 'static> sealed::Sealed for (A,) {}
// ... up to 12-tuple
```

Macro-expand tuple impls for arities 0..=12.

### 5. Contains<S> sealed trait (access.rs)

```rust
pub trait Contains<S>: AccessSet {}
```

Macro-expand position impls: for each (arity, position), generate
the `impl<...> Contains<T_position> for (T_0, ..., T_arity)`.
Arities 1..=12.

### 6. ColumnValue trait (column_value.rs)

```rust
#![feature(min_specialization)]

pub trait ColumnValue: Copy + 'static {
    const BIT_WIDTH: usize;
}

impl<T: Copy + 'static> ColumnValue for T {
    default const BIT_WIDTH: usize = core::mem::size_of::<Self>() * 8;
}
```

Specialisations for arvo sub-byte widths: `UFixed<{IBits(1)},
{FBits::ZERO}, Hot>` → 1, similar for 2, 4 bits.

### 7. Provider API traits (context.rs)

Seven traits matching DESIGN exactly. Example:

```rust
pub trait ColumnReaderApi<R: AccessSet> {
    unsafe fn read<T: ColumnValue>(&self, i: arvo::newtype::USize) -> T
    where R: Contains<Column<T>>;
}
```

Seven total: ColumnReaderApi<R>, ColumnWriterApi<W>,
ResourceProviderApi<R>, VirtualFirerApi<W>, EachApi<R, W>,
BatchApi<R, W>, ReduceApi<R, W>.

### 8. Accessor traits (context.rs) via ctx macros

provider_generic2!(<R: AccessSet, W: AccessSet> ReduceApi as HasReduce => reduce);

### 9. WorkUnit trait + schedule markers (work_unit.rs)

```rust
#![feature(associated_type_defaults)]

pub struct Always;
pub struct On<V>(core::marker::PhantomData<V>);

pub trait WorkUnit<Schedule = Always>: Send + Sync + 'static {
    type Read: AccessSet;
    type Write: AccessSet;
    type Hint: SchedulingHint;
    type Ctx: HasColumnReader<Self::Read>
            + HasColumnWriter<Self::Write>
            + HasResourceProvider<Self::Read>
            + HasVirtualFirer<Self::Write>
            + HasEach<Self::Read, Self::Write>
            + HasBatch<Self::Read, Self::Write>
            + HasReduce<Self::Read, Self::Write>;
    const COMMUTATIVE: bool = false;
    fn execute(&self, ctx: &Self::Ctx);
}
```

No default on `type Hint` because the `SchedulingHint`
implementing tuple requires specific markers — consumer provides.

### 10. Scheduling hints (hint.rs)

Sealed trait families with `repr(u8)` ZST carriers:

```rust
pub trait UrgencyValue: sealed::Sealed { const VALUE: u8; }
pub trait DivisibilityValue: sealed::Sealed { const VALUE: u8; }
pub trait SignificanceValue: sealed::Sealed { const VALUE: u8; }

#[repr(u8)] pub struct Immediate;   impl UrgencyValue for Immediate { const VALUE: u8 = 255; }
#[repr(u8)] pub struct Steady;      impl UrgencyValue for Steady    { const VALUE: u8 = 170; }
// ... per DESIGN discriminants ...

pub trait SchedulingHint: sealed::Sealed { /* marker */ }
impl<U: UrgencyValue, D: DivisibilityValue, S: SignificanceValue>
    SchedulingHint for (U, D, S) {}
```

## Impl-time decisions (pre-committed)

1. **PhantomData for all store + resource markers.** Zero size,
   `#[repr(transparent)]`.
2. **Sealed traits via private module.** One shared `sealed::Sealed`
   module for AccessSet, Contains, and the three SchedulingHint
   axis traits. Prevents external impls.
3. **Macro-expanded tuple impls.** AccessSet + Contains via
   declarative macro, arities 0..=12. Arity cap 12 matches DESIGN
   guideline (WU ≤8 columns + headroom).
4. **ColumnValue blanket via `default const BIT_WIDTH`** —
   min_specialization. Consumer specialises for known sub-byte
   types.
5. **7 accessor traits via ctx macros.** No hand-written HasX traits.
6. **Platform traits use ctx's `provider!` (non-parameterised).**
   Runtime behaviours consumed by engine via monomorphisation, no
   dyn.
7. **No panic-stub bodies in API traits.** All API trait methods
   are `fn` signatures — consumer's concrete impls provide bodies.
   api crate provides no runtime implementation.
8. **`type Hint: SchedulingHint` without default** — consumer
   provides; default lives on the engine-side Ctx construction.
9. **fn `spawn` signature deferred.** `impl FnOnce() + Send + 'static`
   is the standard shape but may need a trait object or generic —
   subagent picks the no-dyn-compatible form.
