## Purpose

First hilavitkutin-api round, post-reconciliation with hilavitkutin-ctx.
Lays the consumer-facing type surface. The rescoping from the
abandoned earlier #34 attempt (archived at
`design_rounds/202604200054-abandoned/`): the WorkUnit execution
parameter is now a ctx-framework provider tuple (leveraging the
`provider_generic!` / `provider_generic2!` macros shipped in round
202604200055), not a bespoke `Context<R, W>` wrapper.

Contract surface shipped this round:

1. **Store types** — `Resource<T>`, `Column<T>`, `Virtual<T>` as
   PhantomData-backed ZSTs.
2. **`StoreId`** — `#[repr(transparent)]` newtype wrapping
   `arvo::newtype::USize`. Plan-time dense index.
3. **`AccessMask`** — alias to `arvo_bitmask::Mask64`.
4. **`AccessSet`** — sealed trait on tuples up to 12 elements.
5. **`Contains<S>`** — sealed compile-time membership trait.
6. **`ColumnValue`** — `Copy + 'static + BIT_WIDTH: usize`
   (min_specialization blanket default to
   `size_of::<Self>() * 8`).
7. **Provider API traits** for WU-context access:
   - `ColumnReaderApi<R: AccessSet>` with generic
     `read<T: ColumnValue>(&self, i: USize) -> T where
     R: Contains<Column<T>>`.
   - `ColumnWriterApi<W: AccessSet>` with generic
     `write<T: ColumnValue>(&self, i: USize, v: T) where
     W: Contains<Column<T>>`.
   - `ResourceProviderApi<R: AccessSet>` with generic
     `resource<T: 'static>(&self) -> &T where
     R: Contains<Resource<T>>`.
   - `VirtualFirerApi<W: AccessSet>` with generic
     `fire<V: 'static>(&self) where W: Contains<Virtual<V>>`.
   - `EachApi<R: AccessSet, W: AccessSet>` with
     `run<F>(&self, f: F)`.
   - `BatchApi<R: AccessSet, W: AccessSet>` with
     `run<F>(&self, f: F)`.
   - `ReduceApi<R: AccessSet, W: AccessSet>` with
     `run<A, F>(&self, init: A, f: F) -> A`.
8. **Accessor traits** (via ctx's `provider_generic!` /
   `provider_generic2!` macros):
   - `HasColumnReader<R>`, `HasColumnWriter<W>`,
     `HasResourceProvider<R>`, `HasVirtualFirer<W>`,
     `HasEach<R, W>`, `HasBatch<R, W>`, `HasReduce<R, W>`.
9. **`WorkUnit<Schedule = Always>`** trait —
   `Read: AccessSet`, `Write: AccessSet`,
   `Hint: SchedulingHint = (Immediate, Atomic, Normal)`,
   `Ctx: HasColumnReader<Self::Read> + HasColumnWriter<Self::Write>
    + HasResourceProvider<Self::Read>
    + HasVirtualFirer<Self::Write>
    + HasEach<Self::Read, Self::Write>
    + HasBatch<Self::Read, Self::Write>
    + HasReduce<Self::Read, Self::Write>`,
   `const COMMUTATIVE: bool = false`,
   `fn execute(&self, ctx: &Self::Ctx)`.
10. **Schedule markers:** `Always`, `On<V>`.
11. **Scheduling hints** — 3-axis sealed traits with `repr(u8)`
    ZST carriers (Urgency / Divisibility / Significance) per
    DESIGN discriminants. `SchedulingHint` trait on 3-tuples.
12. **Platform traits** — `MemoryProvider`, `ThreadPool`, `Clock`
    as simple (non-parameterised) ctx providers via `provider!`.
    Parameter types: `arvo::newtype::USize` for lengths/counts,
    `arvo::newtype::Bool` for flags, raw `u64` for `now_ns`.
13. **Resource model** — `Field<T>`, `Seq<T, const N: Cap>`,
    `Map<K, V, const N: Cap>` as ZST markers.

## Per-crate doc changes

### hilavitkutin-api — `mock/crates/hilavitkutin-api/DESIGN.md.tmpl`

Substantial rewrite of the Context / WU execution section:
- Remove the `Context<'a, R: AccessSet, W: AccessSet>` section
  entirely. WU execute parameter is now `&Self::Ctx` where `Ctx`
  is a provider tuple bound by the 7 `HasX` accessor traits.
- Add a "WU Context providers" section listing the 7 accessor
  traits and their API traits, with the compile-time
  access-set enforcement (`Contains`) spelled out.
- Update the "ops registration" prose (`each` / `batch` /
  `reduce`): now lives on separate `HasEach<R, W>` /
  `HasBatch<R, W>` / `HasReduce<R, W>` providers rather than
  methods on Context.
- Add a top-of-file note that this crate depends on
  `hilavitkutin-ctx` for its `provider_generic!` /
  `provider_generic2!` macros and `Context<P>` wrapper.

### hilavitkutin-api — `mock/crates/hilavitkutin-api/BACKLOG.md.tmpl`

Existing BACKLOG (from round 202604200054) is preserved from the
abandoned feature branch. Additions this round:
- `Mask256` variant of `AccessMask` (already tracked).
- Inherent `Context<P>` delegation for hilavitkutin-api's
  accessor traits — deferred because Rust's orphan rule forbids
  it from downstream crates. Revisit if ctx moves to a proc-macro
  or if hilavitkutin-api adopts a newtype wrapper pattern.
- Arena runtime layout for Seq / Map (engine concern).
- Virtual flag bit-assignment algorithm (engine concern).
- Contains<S> proof-obligation macros (ergonomic helper).

### hilavitkutin-api — `Cargo.toml`

Add `hilavitkutin-ctx = { path = "../hilavitkutin-ctx" }` as a
dependency. Everything else unchanged.

## Comparison to deprecated changelist

No deprecated predecessor. The original #34 attempt's doc CL
(archived at
`design_rounds/202604200054-abandoned/202604200054_changelist.doc.deprecated.md`)
was abandoned mid-phase after the Context reconciliation decision
landed. The earlier content is a strict subset of this round's
scope with ONE difference: Context is no longer a bespoke
`Context<'a, R, W>` struct in hilavitkutin-api. Every type,
trait, marker, and semantic from the abandoned CL appears in this
round's scope except that one wrapper, which is replaced by the
ctx-framework provider-tuple pattern.
