# Current state

`resource/` has 25 bare-primitive warnings across three files:

- `accumulator.rs`: `ConvergenceBuffer<T, const N: usize>` plus
  `i: usize` accessors.
- `cache.rs`: `ResourceCache<R, const N: usize>` plus `i: usize`
  accessors.
- `snapshot.rs`: `Slot(pub u64)` placeholder payload,
  `ResourceSnapshot<const N: usize>`, and `i: usize` accessors.

Three categorical patterns:

1. **Const-generic array size** (`const N: usize`) — structurally
   required by Rust grammar; same class as #121. Five sites.
2. **Positional index** (`i: usize`) — clean `USize` substitution via
   `Deref<Target=usize>` for slot indexing. Six sites.
3. **Placeholder payload** (`Slot(pub u64)`) — one site. Carries both
   `no-public-raw-field` and `arvo-types-only`. Instead of dual
   lint:allow, rewrap as `UFixed<64, 0, Hot>` right now. The upstream
   comment notes "real storage lives in per-width variants landing
   with 5a2 / 5a3"; this keeps the placeholder discipline-compliant
   until the real types ship.

# Changes proposed

## Axis 1 — const-generic N (lint:allow, grammar-bound, tracked #121)

Five sites get trailing `// lint:allow(no-bare-numeric) reason:
const-generic array size; rust grammar requires usize; tracked: #121`:

| File | Site |
|------|------|
| `accumulator.rs` | struct `ConvergenceBuffer<T, const N: usize>` |
| `accumulator.rs` | impl block `const N: usize` |
| `cache.rs` | struct `ResourceCache<R, const N: usize>` |
| `cache.rs` | impl block `const N: usize` |
| `snapshot.rs` | struct `ResourceSnapshot<const N: usize>` |
| `snapshot.rs` | impl Default + impl inherent `const N: usize` (each line) |

## Axis 2 — positional index → `USize`

Six accessor method signatures swap `i: usize` → `i: USize`. Body
derefs via `*i` where the raw `usize` is needed for indexing.

## Axis 3 — `Slot` payload becomes `UFixed<64, 0, Hot>`

`snapshot.rs`: redefine
```rust
pub struct Slot(pub UFixed<{IBits(64)}, {FBits::ZERO}, Hot>);
```
Update all initialisers: `Slot(0)` → `Slot(UFixed::from_raw(0))`.
