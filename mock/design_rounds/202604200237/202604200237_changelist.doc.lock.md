## Purpose

Second-to-last engine sub-round. Ships the public type surface +
function scaffolding for the plan stage — the analysis phase that
turns WU declarations into an `ExecutionPlan`. Skeleton only: every
algorithm step stubs to `todo!()`; downstream rounds fill in the
bodies one at a time.

The plan stage is the biggest sub-round of the engine crate and
depends on arvo-graph, arvo-sparse, arvo-bitmask, arvo-spectral,
arvo-comb primitives. Landing the skeleton now unblocks dispatch
(5a3) and thread (5a4) rounds that need to reference the plan
types in their own signatures.

## Deliverables

### Plan types (domains 11, 14, 15, 16)

All under new `src/plan/` submodule tree. Scheduler already has a
separate simpler `scheduler/plan.rs` (ExecutionPlan / LaneAssignment)
which stays untouched.

- `NodeId(u32)`, `FiberId(u16)`, `PhaseId(u8)` — newtypes,
  `#[repr(transparent)]`, `Copy`, `Eq`, `Hash`.
- `AccessMask<const MAX_STORES: usize>` — which stores a WU touches.
  Backed by `u128` for MAX_STORES ≤ 128; skeleton uses `u128` only
  (document larger backing in BACKLOG).
- `PlanInputs<const MAX_UNITS: usize, const MAX_STORES: usize>` —
  descriptor bundle consumed by `build_plan`. Const arrays of
  `AccessMask` per unit + record count estimate + commutativity
  flags.
- `DependencyGraph<const MAX_UNITS: usize>` — adjacency matrix
  `[[bool; MAX_UNITS]; MAX_UNITS]`. Swap for arvo-graph CSR when
  arvo-graph gains const-generic support (BACKLOG).
- `FiberGrouping<const MAX_UNITS: usize, const MAX_FIBERS: usize>` —
  per-unit fiber assignment.
- `PhaseBoundaries<const MAX_PHASES: usize>` — waist-derived phase
  splits.
- `ColumnClassification` — per-fiber enum
  `{ Internal, Input, Output }`.
- `DirtyMask<const MAX_STORES: usize>` — incremental-skip
  propagation mask; `u128`-backed like AccessMask.

### Twelve algorithm-step stubs

Each as a separate function with the right signature; body is
`todo!()`. Gives downstream rounds a crisp seam to slot impl into.

1. `build_dag` — DAG from AccessMask overlap.
2. `topo_sort` — topological sort + node renumbering.
3. `upward_rank` — critical-path annotation.
4. `detect_waists` — waist → phase boundaries.
5. `rcm_reorder` — RCM reordering → fiber grouping order.
6. `block_diagonal` — D-M → phase validation.
7. `spectral_partition` — spectral partition for >5 fibers.
8. `group_fibers` — greedy + matrix-chain DP.
9. `size_morsels` — per-fiber morsel sizing.
10. `adaptive_config` — per-phase adaptive configs.
11. `classify_columns` — per-fiber column classification.
12. `propagate_dirty` — dirty propagation masks.

### Deferrals (all → BACKLOG)

- Actual algorithm bodies for all 12 steps (separate rounds).
- AccessMask / DirtyMask on arvo-bitmask backing (skeleton uses
  u128).
- DependencyGraph on arvo-graph CSR (skeleton uses bool matrix).
