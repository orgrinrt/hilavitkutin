## Per-file edits

Two mechanical patterns repeat across the affected files. Pattern A is the import-path migration. Pattern B is the const-generic literal-form migration.

### Pattern A: `arvo::newtype::{...}` → `arvo::{...}`

The import names stay identical. The path moves from the removed `newtype` submodule to the crate root re-export.

Apply at:

### Pattern B: `IBits(N)` / `FBits(N)` literal → `ibits(N)` / `fbits(N)` helper

Apply at:

`crates/hilavitkutin-api/src/hint.rs`. Three sites. All of them appear in const-generic positions inside `pub type` aliases over `UFixed<I, F, S>` and inside `from_raw` calls used at construction.

Specifically:

- `pub type Urgency = UFixed<{ IBits(2) }, { FBits::ZERO }, Hot>` becomes `pub type Urgency = UFixed<{ ibits(2) }, { FBits::ZERO }, Hot>`.
- The same shape repeats on the other type aliases that use `IBits(3)` (or any other narrow integer) at the I-position.
- `Urgency::from_raw(N)` literal usages in inherent const items stay untouched. `from_raw` accepts the raw container type, not `IBits`. Pattern B applies only to const-generic positions, not value positions.

Note: `FBits::ZERO` continues to work. Round D added `ZERO` and `ONE` constants to the meta-bits wrapper. The macro emits `pub const ZERO: Self = Self(MetaCarrier::from_raw(0))`, so `FBits::ZERO` resolves to a value of the new shape transparently.

Note: importing the helper. Where `hint.rs` previously needed only `IBits` and `FBits`, it now needs `IBits`, `FBits`, and `ibits`. The new import becomes `use arvo::{IBits, FBits, ibits};`. The crate root re-exports both the wrapper types and the const-fn helpers (`ibits`, `fbits`, `width`) per arvo's `pub use arvo_storage::{...}` line in `arvo/src/lib.rs`.
