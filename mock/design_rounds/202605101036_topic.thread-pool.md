## Decisions locked

### Axis A. Executor trait shape

Open `trait Executor: Sealed`, monomorphised. Declared in `hilavitkutin-api`. Engine ships exactly one default impl `HybridExecutor` in `hilavitkutin::thread`. `Scheduler<Cfg, E: Executor = HybridExecutor>` propagates the type. The Sealed bound prevents non-substrate impls from violating the M-protocol (Topic 3 S7 cross-thread atomic ordering); advanced consumers ship their own sealed-bridge impl crate if they need a different shape.

The trait's method set is constrained by Topic 5 axis G (lock-free dispatch) and Topic 3 S7 (atomic-ordering protocol). Concrete shape to be drafted from the constraints during src CL; expected core methods: per-worker mainloop entry, park/unpark primitive, phase-barrier-arrive hook. `try_steal_from(other_core: USize)` deliberately NOT included in v1 (work-stealing is BACKLOG).

### Axis B. Three-tier parking middle (Topic 5 axis G `spin → ? → park`)

Per-OS atomic-wait primitives. Linux: `futex(2)` `FUTEX_WAIT_PRIVATE` / `FUTEX_WAKE_PRIVATE`. macOS: `__ulock_wait2` / `__ulock_wake`. Windows: `WaitOnAddress` / `WakeByAddressSingle`. Direct kernel-call wrappers live in `OsThreadPool`; std-tier alternative uses `std::thread::park_timeout` with 50µs grain.

Tier selection driven by `predicted_wait_ns` per phase (see axis G):
- `< 2µs`: spin-only (skip middle + park tiers)
- `2µs to 50µs`: spin budget exhausted → futex/ulock/WaitOnAddress
- `> 50µs`: skip spin, park immediately via futex (long-tail wait)

Threshold constants in `WakeStrategy`. Documented as platform-tuned defaults.

### Axis C. Worker data plumbing

`Pin<&'frame PoolFrame>` propagated via generics. Lifetime threads through `Scheduler<'frame, Cfg, E>`, `ExecutionPlan<'frame, MAX_*...>`, and the per-worker entry function. PoolFrame is pinned to its scheduler-owned arena; lifetime invariance prevents lift-out.

**Fallback:** if the sketch shows the lifetime propagation tangles into `AccessSet`/`WorkUnit::Ctx` bounds unacceptably (specifically: requires every consumer's `impl WorkUnit` declaration to thread `<'frame>`), fall back to `*const PoolFrame` raw pointer with documented safety contract. ### Axis E. Heterogeneous-core P/E classification

Ship per-OS classification in v1.

- **x86_64:** CPUID leaf 0x1A. Returns CoreClass per logical core.
- **Linux:** read `/sys/devices/system/cpu/cpuN/topology/cluster_id` + cpufreq `scaling_max_freq` as fallback. cluster_id post-6.0 separates P/E.
- **macOS:** `sysctlbyname("hw.perflevel0.physicalcpu")` returns P-core count; remaining are E-cores. IOKit fallback if sysctl absent.
- **Windows:** `GetSystemCpuSetInformation` returns per-CPU efficiency class.

CoreAssignment uses CoreClass: critical-path trunks pin to P-cores, branches and convergence to E-cores. Stub-fallback path (all-P) is the safety net when detection fails.

### Axis G. Pool shutdown / teardown protocol

`PoolFrame` carries `shutdown: AtomicBool`. Workers check after each morsel; on shutdown=true, exit mainloop cleanly (no further work pulls). `Scheduler::Drop`:
1. Stores `true` into `shutdown`.
2. Calls futex/ulock/WakeByAddressAll on every parked worker's wait slot.
3. Joins each ThreadHandle in `pool.thread_handles`.

One Relaxed atomic load per morsel boundary on the worker side (cheap). No thread cancellation primitives used.

### Axis I. Phase-barrier algorithm

Centralised atomic counter + futex-wait. `PoolFrame.phase_arrived: AtomicU32`. Each worker on phase exit:
1. Increments `phase_arrived` with Release ordering.
2. If value < N: futex-wait on the phase_arrived address.
3. If value == N (last arriver): reset counter to 0, futex-wake-all.

Documented v1 simple shape. Tree-barrier follow-up only if benches at 32+ cores show measurable cacheline ping-pong on the centralised counter. Bench: `mock/benches/barrier_scaling_axis_i` (overnight workflow).

### Axis J. Predictive-parking plumbing

Per-phase atomic slot. `PoolFrame.predicted_wait_ns: [AtomicU32; MAX_PHASES]`. AdaptWu writes per phase at ScheduleEnd. On phase entry, each worker reads its phase slot with Relaxed, picks tier:
- `< 2µs` → spin-only
- `2µs to 50µs` → spin then futex
- `> 50µs` → skip spin, park immediately

Threshold constants live in `WakeStrategy` (publicly tunable). One Relaxed load per phase; branch-predictable thresholds.

### Axis K. WakeStrategy granularity

Per-CoreClass split. `WakeStrategy` field set extends:

```rust
pub struct WakeStrategy {
    pub p_spin_iters: USize,
    pub e_spin_iters: USize,
    pub futex_threshold_ns: USize,  // 2µs default
    pub park_threshold_ns: USize,   // 50µs default
}
```

`ThreadPool::build()` reads CoreClass per worker, picks `p_spin_iters` or `e_spin_iters`. P-cores spin longer (they cost more to leave parked); E-cores spin shorter to yield quickly. Defaults: P=128, E=32 (sketch validates these or escalates to bench).

Removes `PureSpin` / `PurePark` enum variants — instead, expressible via the same struct: `p_spin = e_spin = usize::MAX` is pure-spin; `p_spin = e_spin = 0` is pure-park.
