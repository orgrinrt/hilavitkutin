## Deliverables

### S1 — create `mock/crates/hilavitkutin-api/src/capability.rs`

Six pub items (five traits + one marker struct):

- `pub trait Push<T>`
  - `fn push(&mut self, item: T);` — required; infallible.
- `pub trait BulkPush<T>: Push<T>`
  - `fn push_bulk(&mut self, items: &[T]) where T: Copy` — default
    implementation pushes per-item in order.
- `pub trait Len`
  - `fn len(&self) -> USize;` — required.
  - `fn is_empty(&self) -> Bool` — default `(*self.len() == 0).into()`.
- `pub trait Capacity`
  - `fn capacity(&self) -> USize;` — required.
  - `fn remaining(&self) -> USize;` — required.
- `pub trait BoundedPush<T>: Push<T> + Capacity`
  - `fn try_push(&mut self, item: T) -> Outcome<(), Full>;` — required.
  - Doc note: implementors MUST NOT silently drop; overflow is
    always a refusal via `Outcome::Err(Full)`.
- `pub struct Full;` — `#[derive(Debug, Clone, Copy, PartialEq, Eq)]`.

Imports: `arvo::{Bool, USize}`, `notko::Outcome`.

### S2 — create `mock/crates/hilavitkutin-api/src/sink.rs`

Three named traits (blanket-impl shortcuts), three combinators,
tuple blanket impls for arity 2 and 3.

- `pub trait Collector<T>: Push<T> {}` — blanket `impl<T, S> Collector<T> for S where S: Push<T> + ?Sized`.
- `pub trait DiagnosticSink<E>: Push<E> + Len {}` — same shape.
- `pub trait ByteEmitter: Push<u8> + BulkPush<u8> {}` — same shape.

Combinators:

- `pub struct NullSink;` — `#[derive(Debug, Clone, Copy, Default)]`.
  - `impl<T> Push<T> for NullSink { #[inline(always)] fn push(&mut self, _: T) {} }`
  - `impl<T> BulkPush<T> for NullSink { #[inline(always)] fn push_bulk(&mut self, _: &[T]) where T: Copy {} }`

- `pub struct CountingSink<T> { count: USize, _m: PhantomData<fn(T)> }` —
  `#[derive(Debug)]`; `Default` manual (USize has no `Default`
  const-of-zero guarantee; use `USize(0)`). `#[derive(Clone, Copy)]`
  plus a `new() -> Self` constructor.
  - `impl<T> Push<T> for CountingSink<T> { fn push(&mut self, _: T) { self.count = USize(*self.count + 1); } }`
  - `impl<T> Len for CountingSink<T> { fn len(&self) -> USize { self.count } }`

- `pub struct TeeSink<'a, A: ?Sized, B: ?Sized> { pub a: &'a mut A, pub b: &'a mut B }` —
  no derives (contains mutable refs).
  - `impl<'a, T, A, B> Push<T> for TeeSink<'a, A, B> where T: Clone, A: Push<T> + ?Sized, B: Push<T> + ?Sized { fn push(&mut self, item: T) { self.a.push(item.clone()); self.b.push(item); } }`

Tuple blanket impls — arity 2 for one-type disjunction (pass two
sinks of the same T) AND for mixed-type disjunction (one sink each
for T and U):

- `impl<T, A, B> Push<T> for (A, B) where A: Push<T>` — no-op on B
  wouldn't work; instead tuple blanket only handles *mixed* scenarios
  via DIFFERENT axis — actually the intent of tuple impls is:
  consumer calls `parse(&mut (diagnostic_sink, byte_sink), ...)` and
  the parser code internally does `ctx.0.push(diag)` and
  `ctx.1.push_bulk(&bytes)` via structural access. So tuples don't
  need trait-level `Push<T>` impls; they're used via direct field
  access. **Revise:** drop "tuple blanket impls" from this round's
  source. Call sites can take two separate sink parameters instead.
  Document this revision at the top of `sink.rs` and update the
  DESIGN sketch follow-ups in a short addendum CL if needed.

(Alternative path if tuple impls are still desired: emit
`impl<T, A: Push<T>, B: Push<T>> Push<T> for (A, B)` that forwards to
BOTH (effectively a statically-fanout tee). This duplicates `TeeSink`
functionality for tuple syntax. Skip in this round; revisit if
consumer demand surfaces.)

Imports: `core::marker::PhantomData`, `arvo::{Bool, USize}`,
`crate::capability::{BulkPush, Len, Push}`.

### S3 — create `mock/crates/hilavitkutin-api/src/codec.rs`

- `pub trait Encoder<T>`:
  - `fn feed<B: ByteEmitter>(&mut self, v: &T, out: &mut B);`
  - `fn finish<B: ByteEmitter>(self, out: &mut B);`
- `pub trait Decoder<T>`:
  - `fn feed<'a, S: Push<T>>(&mut self, chunk: &'a [u8], out: &mut S) -> Outcome<&'a [u8], DecodeError>;`
  - `fn finish(self) -> Outcome<(), DecodeError>;`
- `pub trait EncoderExt<T>: Encoder<T> + Sized`:
  - `fn encode_one<B: ByteEmitter>(self, v: &T, out: &mut B) { self.feed(v, out); self.finish(out); }` —
    note: `self.feed` requires `&mut self`; takes ownership of
    encoder in `encode_one`. Sketch uses `let mut s = self; s.feed(v, out); s.finish(out);`.
  - `impl<T, E: Encoder<T>> EncoderExt<T> for E {}` — blanket.
- `pub trait DecoderExt<T>: Decoder<T> + Sized`:
  - `fn decode_all<S: Push<T>>(self, bytes: &[u8], out: &mut S) -> Outcome<(), DecodeError>` —
    internal body: `let mut d = self; let rest = d.feed(bytes, out)?; if !rest.is_empty() { Outcome::Err(DecodeError::OverLength) } else { d.finish() }`.
  - `impl<T, D: Decoder<T>> DecoderExt<T> for D {}` — blanket.
- `pub enum DecodeError`: `#[non_exhaustive]`, `#[derive(Debug, Clone, Copy, PartialEq, Eq)]`.
  Variants: `Truncated`, `Invalid`, `OverLength`.

Imports: `notko::Outcome`, `crate::capability::Push`,
`crate::sink::ByteEmitter`.

### S5 — create `mock/crates/hilavitkutin-api/tests/sink_smoke.rs`

Minimal smoke test covering the three combinators + blanket shortcuts.

- Define an in-test `struct VecSink<T> { items: [MaybeUninit<T>; 16], count: usize }` or similar bounded sink (no alloc). Provide `impl<T: Copy> Push<T>` that appends into `items[count]` + increments count. Implement `Len`.
- Assert `NullSink` accepts `Push<u32>` pushes without panicking and `NullSink.push_bulk(&[1u32, 2, 3])` returns.
- Assert `CountingSink::<u32>::default().len() == USize(0)` pre-push; after 5 pushes, `len() == USize(5)`.
- Assert `TeeSink { a: &mut sink_a, b: &mut sink_b }` where `sink_a: VecSink<u32>` and `sink_b: CountingSink<u32>` after 3 pushes has `sink_a.count == 3` and `sink_b.len() == USize(3)`.
- Assert a function `fn drain<S: Collector<u32>>(s: &mut S, items: &[u32])` monomorphises over `VecSink<u32>`, `NullSink`, and `CountingSink<u32>` — proves the blanket impl.

### S6 — create `mock/crates/hilavitkutin-api/tests/codec_smoke.rs`

Minimal Encoder + Decoder round-trip for `u32` values.

- Define an in-test `ByteBuf` that implements `Push<u8> + BulkPush<u8>` via a fixed `[u8; 256]` and cursor (no alloc).
- Define an in-test `U32Le` implementing `Encoder<u32>` via little-endian write and `Decoder<u32>` via little-endian read; both as unit structs.
- Round-trip: encode `&[5u32, 42, 7]` via `U32Le.encode_one` calls (or repeated feed + finish), then `U32Le.decode_all(&buf.bytes[..buf.cursor], &mut output_sink)`, assert output_sink received `[5, 42, 7]`.
- Negative case: call `decode_all` on a truncated chunk; assert `Outcome::Err(DecodeError::Truncated)`.
- Negative case: call `decode_all` on a chunk with trailing garbage; assert `Outcome::Err(DecodeError::OverLength)`.

## Known-safe cleanups (apply during implementation if surfaced)

- If tuple blanket impls block on coherence (`impl<T, A: Push<T>, B: Push<T>> Push<T> for (A, B)`
  conflicts with anything), drop them per the S2 note.
- If `USize` arithmetic (`USize(*self.count + 1)`) feels noisy, wrap
  in a `private fn inc(&mut USize) { ... }` helper inside `sink.rs`

  `self.len() == 0` with a newtype, suppress locally with `#[allow(clippy::comparison_chain)]`
  or materialise via `Bool::from(*self.len() == 0)`.
