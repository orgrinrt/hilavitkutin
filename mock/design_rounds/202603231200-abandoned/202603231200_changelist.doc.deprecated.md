- DSE + stores-at-end (rust-pipe pattern) — unchanged (domain 17)
- Head+tail convergence — unchanged (domain 20)
- Pipeline parallelism between phases — unchanged (domain 11)
- Pre-allocated thread pool — unchanged (domain 20)

## Crate structure

```
arvo                   numeric primitives (standalone)
arvo-bitmask           fixed-width bitmask types (standalone)
arvo-sparse            sparse matrix (deps: arvo-bitmask, arvo)
arvo-graph             DAG algorithms (deps: arvo-bitmask, arvo)
arvo-spectral          spectral methods (deps: arvo-sparse)
arvo-comb              combinatorial optimisation (deps: arvo-bitmask, arvo)
hilavitkutin-api       consumer-facing types, traits, platform contracts
hilavitkutin           engine (scheduling, dispatch, codegen, platform impls)
hilavitkutin-build     build-time optimisations (LLVM, MIR, codegen, cfg)
```

Runtime dependency graph:
```
consumer
  └─ hilavitkutin
       ├─ hilavitkutin-api
       ├─ arvo, arvo-bitmask, arvo-graph, arvo-sparse
       ├─ arvo-spectral (deps: arvo-sparse)
       └─ arvo-comb (deps: arvo-bitmask, arvo)
```

Build-time dependency:
```
hilavitkutin-api/build.rs  → hilavitkutin-build
hilavitkutin/build.rs      → hilavitkutin-build
consumer/build.rs          → hilavitkutin-build
```
