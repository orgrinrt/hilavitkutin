# Arbitrary-time linking invariant (carried forward)

All three new or renamed crates preserve the arbitrary-time linking invariant locked in the prior round and reinforced in this round's topic 2:

- `hilavitkutin-linking`: every Library handle is independent; the lifetime-to-handle tie enforces that dropping a library invalidates outstanding symbols at compile time. No global registry state.
- `hilavitkutin-extensions`: every Extension handle is independent; `ExtensionHost::load` returns one handle per call; no `load_all_extensions` entry point exists; drop runs one extension's shutdown and closes one library.
- `hilavitkutin-extensions-macros`: emitted output is per-extension-static; the macro produces one descriptor constant and one exported-symbol fn per extension; no cross-extension registration, no linker-magic, no assumption of load ordering.

This rule is documented in D2 (linking DESIGN), D5 (extensions-host DESIGN), and D8 (extensions-macros DESIGN), and reinforced in D11 (MAIN.md.tmpl) and D12 (cargo.md.tmpl).
