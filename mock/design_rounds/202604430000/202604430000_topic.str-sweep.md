## Character of each site

- **hash.rs:** FNV-1a is a documented 64-bit algorithm. `u64` width
  is load-bearing (changing it changes the hash). `&str` input is
  the byte source. Not fungible.
- **interner.rs:** Two categorical ways `&str` shows up. First, the
  `trait ArenaInterner` contract — the consumer implements
  `arena_intern(&self, s: &str) -> u32` to take the user's string
  and return a 28-bit id. That `&str` is the boundary between the
  caller's string memory and the interner; it cannot be `Str`
  (`Str` is the output of interning, not the input). Second, the
  `&str` return on `arena_resolve` is the resolved string content,
  as the consumer sees it. Both are boundary `&str`, and the whole
  reason `hilavitkutin-str` exists is to wrap these. The `u32`
  return is the 28-bit id (reserved for future 32-bit variants).
- **handle.rs:** `is_const()` / `is_runtime()` are predicates — the
  canonical `bool → Bool` flip. Only one internal consumer
  (interner.rs:65).
- **section.rs:** `as usize` cast on `offset_from()` result. Slice
  length from raw pointer math requires host-width usize; no way
  around it under `#![no_std]`.

# Changes proposed

- Add `arvo.workspace = true` to hilavitkutin-str/Cargo.toml
  (currently depends on `arvo-bits` only). Needed so handle.rs
  can import `arvo::Bool`.
- **handle.rs**: flip `pub const fn is_const(self) -> bool` and
  `pub const fn is_runtime(self) -> bool` to return `Bool`.
  `is_runtime` body: `Bool(!self.is_const().0)`.
- **interner.rs line 65**: update the one consumer of `is_const()`
  from `if s.is_const() { ... }` to `if s.is_const().0 { ... }`.
- **All other sites** (hash.rs consts/inputs/returns, interner.rs
  trait contract + internals, section.rs pointer-math casts):
  trailing inline `lint:allow(no-bare-numeric)
  lint:allow(arvo-types-only) lint:allow(no-bare-string)` with
  category-specific reasons:
  - FNV-1a sites: "FNV-1a 64-bit algorithm constant; hash width
    fixed by algorithm; tracked: #72".
  - ArenaInterner trait sites: "interner boundary — &str is the
    input string the arena wraps into `Str`; u32 is the 28-bit id
    width; tracked: #72".
  - Section pointer-math: "slice length from raw pointer
    `offset_from`; host-width usize by rustc; tracked: #72".
  - `interner.rs` internal helpers (`lookup_const_by_value`,
    `str_eq`): "interner-internal &str math; mirrors boundary
    width; tracked: #72".

# Non-goals

- No Bool flip in hash.rs or interner.rs — those are algorithm
  input/output, not predicates.
