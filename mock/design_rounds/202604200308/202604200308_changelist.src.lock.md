### `src/thread/wake.rs` (~30 lines)

```rust
/// Wake strategy for a pool thread waiting on work.
#[derive(Copy, Clone, Debug, Eq, PartialEq, Hash)]
pub enum WakeStrategy {
    /// Spin `spin_iters` rounds then park. Default per DESIGN (128).
    HybridSpinPark { spin_iters: u32 },
    /// Busy-wait forever, never park.
    PureSpin,
    /// Park immediately, no spinning.
    PurePark,
}

impl WakeStrategy {
    /// Default hybrid strategy: 128 spin iters per DESIGN.
    pub const fn default_hybrid() -> Self {
        Self::HybridSpinPark { spin_iters: 128 }
    }
}

impl Default for WakeStrategy {
    fn default() -> Self { Self::default_hybrid() }
}
```

use super::WakeStrategy;

/// Pre-allocated pool. Stub this round — real OS thread spawning
/// is feature-gated on a future `threading-std` feature (see
/// BACKLOG).
#[derive(Copy, Clone, Debug)]
pub struct ThreadPool {
    pub thread_count: u16,
    pub spin_budget: u32,
    pub wake_strategy: WakeStrategy,
}

impl ThreadPool {
    /// Construct a stub pool record. No threads are spawned this
    /// round; follow-up round wires real spawning under
    /// `threading-std`.
    pub const fn new(core_count: u16, wake: WakeStrategy) -> Self {
        let spin_budget = match wake {
            WakeStrategy::HybridSpinPark { spin_iters } => spin_iters,
            WakeStrategy::PureSpin => u32::MAX,
            WakeStrategy::PurePark => 0,
        };
        Self {
            thread_count: core_count,
            spin_budget,
            wake_strategy: wake,
        }
    }
}

impl Default for ThreadPool {
    fn default() -> Self {
        Self::new(1, WakeStrategy::default_hybrid())
    }
}

/// Per-core role record. Parallel-array layout mirrors the pattern

## Impl-time decisions

1. **`ThreadPool` is `Copy` + const-new.** Skeleton record — no
   heap allocation, no real threads. Matches the `AdaptConfig`
   style and lets callers use it from const contexts.
2. **`WakeStrategy::default_hybrid` pegged at 128.** Per DESIGN
   "spin 128 → park". Tunable via the enum discriminant if a
   caller needs a different budget.
3. **`CoreAssignment` uses `u16::MAX` sentinel for unassigned
   `trunk_index`.** No `Option<u16>` needed — matches the
   existing plan-module pattern of sentinel-in-u16.
4. **`CoreAssignment::morsel_size_multiplier` default = 100 (1.0x).**
   Integer basis-points-style multiplier avoids float in a
   no-std + no-alloc context. Follow-up rounds can scale by
   dividing by 100.
5. **`Convergence` is not `Copy` (owns `ProgressCounter`).** The
   counter is an `AtomicUsize` newtype — cloning would mean two
   consumers updating independent atomics, which defeats the
   convergence contract. Construct-in-place via `const new`.
6. **`AdaptMode = PhaseStrategy` alias, not re-export of the
   enum.** The alias is in `adapt/mod.rs`; callers can write
   `adapt::AdaptMode::MaxFuse` without importing from strategy.
   No duplication.
7. **`classify_cores` stays `[CoreClass; 256]` (not generic).**
   Signature churn is cheap; a follow-up round that actually
   implements detection can generalise if it needs to.
8. **`update_adapt` takes `&mut [AdaptConfig; MAX_PHASES]`** —
   in-place update; avoids returning a big array by value.
