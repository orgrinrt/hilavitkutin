# Src CL: round 202605042200, Kit trait reshape + WorkUnitBundle/StoreBundle/Replaceable substrate

## Substantive shape (recap)

Kit trait reshapes from `pub trait Kit<B> { type Output; fn install(self, builder: B) -> Self::Output; }` (a builder-transformer) to a declarative bundle:

```rust
pub trait Kit {
    type Units: WorkUnitBundle;
    type Owned: StoreBundle;
}
```

Substrate primitives that the new shape rests on (all in `hilavitkutin-api`):

```rust
pub trait WorkUnitBundle {
    type AccumRead: AccessSet;
    type AccumWrite: AccessSet;
}

pub trait StoreBundle {}

pub trait Replaceable {}

#[marker]
pub trait ContainsAll<L>: AccessSet {}
```

`SchedulerBuilder` becomes typestate-only (no `MAX_UNITS` / `MAX_STORES` / `MAX_LANES` const generics). `.add_kit::<K: Kit>()` is type-only (no value-level `install` call); appends `K::Units` and `K::Owned` via type-level `Concat`. `.build()` proves `Stores: ContainsAll<Wus::AccumRead> + ContainsAll<Wus::AccumWrite>`. `Scheduler::replace_resource::<T: Replaceable>(value)` lands on the engine.

Deletions: `Buildable<Stores>`, `WuSatisfied<A>`, `BuilderExtending<B>`, `BuilderResource<T>`. No deprecation aliases (per `no-legacy-shims-pre-1.0.md`).

`read!` / `write!` macros change cell shape from `(H, T)` flat-tuple to `Cons<H, T>`.

#### CHANGE: add trait `WorkUnitBundle` with recursive impls on Empty + Cons<W, T>

File: `mock/crates/hilavitkutin-api/src/work_unit.rs` (or a new sibling module)

```rust
/// Cons-list bundle of `WorkUnit` types with accumulated `Read` /
/// `Write` access sets.
///
/// `AccumRead` is the projected union of every WU's `Read` over the
/// bundle. `AccumWrite` is the symmetric projection over `Write`.
/// Recursive impls on `Empty` (zero accumulators) and `Cons<W, T>`
/// (concat W::Read into T::AccumRead, W::Write into T::AccumWrite).
pub trait WorkUnitBundle {
    type AccumRead: AccessSet;
    type AccumWrite: AccessSet;
}

impl WorkUnitBundle for Empty {
    type AccumRead = Empty;
    type AccumWrite = Empty;
}

impl<W, T> WorkUnitBundle for Cons<W, T>
where
    W: WorkUnit,
    T: WorkUnitBundle,
    W::Read: Concat<T::AccumRead>,
    W::Write: Concat<T::AccumWrite>,
    <W::Read as Concat<T::AccumRead>>::Out: AccessSet,
    <W::Write as Concat<T::AccumWrite>>::Out: AccessSet,
{
    type AccumRead = <W::Read as Concat<T::AccumRead>>::Out;
    type AccumWrite = <W::Write as Concat<T::AccumWrite>>::Out;
}
```

### Step 5: `hilavitkutin-kit/src/lib.rs`: rewrite `Kit<B>` → declarative `Kit { Units, Owned }`

#### CHANGE: rewrite Kit trait declaration

File: `mock/crates/hilavitkutin-kit/src/lib.rs:11-31`

Replaces the entire `pub trait Kit<B>` (with `Output` and `install`) with:

```rust
//! Kit preset trait for the hilavitkutin scheduler builder.
//!
//! `no_std`. Depends on `hilavitkutin-api` for the substrate
//! primitives (`WorkUnitBundle`, `StoreBundle`) that bound the
//! associated types. A Kit declaration is purely a type-level
//! statement; the engine reads `K::Units` and `K::Owned` at compile
//! time from `.add_kit::<K>()`.

#![no_std]
#![recursion_limit = "512"]

use hilavitkutin_api::{StoreBundle, WorkUnitBundle};

/// A declarative bundle of WorkUnits and Owned stores.
///
/// `Units` names the WorkUnits the kit contributes (Cons-list of
/// `WorkUnit` types satisfying `WorkUnitBundle`). `Owned` names the
/// Resources / Columns / Virtuals the kit owns (Cons-list of
/// store-marker types satisfying `StoreBundle`).
///
/// Kit authors implement `Kit` by naming `Units` and `Owned` as
/// associated types. The engine's `.add_kit::<K>()` reads these at
/// compile time and accumulates them into the SchedulerBuilder
/// typestate.
///
/// No `install` method, no `B` parameter, no `Output`: the kit is
/// type-level only. App-side wiring (default values for Resources,
/// Replaceable opt-in for replaceable types) lives at the call site.
pub trait Kit {
    /// Cons-list of WorkUnit types this kit contributes. Use
    /// `read!` / `write!` macros from `hilavitkutin-api` to emit the
    /// shape, or `hilavitkutin_api::Empty` for none.
    type Units: WorkUnitBundle;

    /// Cons-list of Owned store-marker types this kit owns
    /// (`Resource<T>` / `Column<T>` / `Virtual<T>`). Use `read!` /
    /// `write!` macros to emit the shape, or
    /// `hilavitkutin_api::Empty` for none.
    type Owned: StoreBundle;
}
```
