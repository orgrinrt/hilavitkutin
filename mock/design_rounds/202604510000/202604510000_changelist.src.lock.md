## Src-side changes

### `mock/crates/hilavitkutin-extensions/src/descriptor.rs`

- Add `use arvo::USize;` at the top of the file.
- `ExtensionDescriptor.name_len: usize` -> `ExtensionDescriptor.name_len: USize`.
- `ExtensionDescriptor.capabilities_len: usize` -> `ExtensionDescriptor.capabilities_len: USize`.
- `ExtensionDescriptor.required_host_caps_len: usize` -> `ExtensionDescriptor.required_host_caps_len: USize`.
- `ExtensionDescriptor.abi_version: u32` stays. Append `// lint:allow(arvo-types-only, no-bare-numeric, no-public-raw-field) tracked: #206 — FFI wire u32, no wire-stable arvo newtype for 32 bits yet.`
- `ExtensionVersion` struct gets a leading block-comment explaining the bare-`u16` policy decision, and each of the four `u16` fields gets a trailing `// lint:allow(arvo-types-only, no-bare-numeric, no-public-raw-field) tracked: #206` annotation.
- `AbiVersion(pub u32)` already from round 202604500000. Append `// lint:allow(no-public-raw-field) tracked: #206` to the `pub u32` field — the raw field is the deliberate transparent-newtype FFI constructor pattern.
- `CapabilityId(pub u64)` — append `// lint:allow(no-public-raw-field) tracked: #206` to the `pub u64` field for the same reason.

### `mock/crates/hilavitkutin-extensions/src/host.rs`

- Wherever the code reads `descriptor.name_len` / `descriptor.capabilities_len` / `descriptor.required_host_caps_len` and compares to a `usize` or indexes arrays with it, add `.0` to unwrap the `USize`. Prefer `USize::deref` when the expression would benefit from autoderef; otherwise `name_len.0` at the use site.

### `mock/crates/hilavitkutin-extensions-macros/src/emit.rs` (and adjacent)

- The macro's emission of `ExtensionDescriptor` struct literal: emit `USize(<len>)` instead of bare `<len>: usize` for the three `_len` fields. All other fields unchanged.
