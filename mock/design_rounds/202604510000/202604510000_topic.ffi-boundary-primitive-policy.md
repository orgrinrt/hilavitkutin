# Topic: FFI-boundary primitive policy + hilavitkutin-extensions cleanup

**Date:** 2026-04-24
**Phase:** TOPIC
**Scope:** `hilavitkutin-extensions`. Defines the policy (resolves #206) and applies it across the crate's `#[repr(C)]` descriptor surface and the extension error shape.

## Current state

Primitive-vocabulary lints (`arvo-types-only`, `no-bare-numeric`, `no-public-raw-field`) fire on `#[repr(C)]` descriptor types in `hilavitkutin-extensions`. Round 202604500000 (#212, conservative pass) wrapped three non-FFI-crossing surfaces (`AbiVersion` newtype, `DESCRIPTOR_SYMBOL: &CStr`, `AbiVersionMismatch` payload). The `#[repr(C)]` descriptor fields themselves were deferred pending a policy decision (#206, #207 parked).

The undecided fields:

- `ExtensionDescriptor.abi_version: u32`
- `ExtensionDescriptor.name_ptr: *const u8`, `name_len: usize`
- `ExtensionDescriptor.version: ExtensionVersion` where `ExtensionVersion` has four `u16` fields
- `ExtensionDescriptor.capabilities_ptr: *const CapabilityEntry`, `capabilities_len: usize`
- `ExtensionDescriptor.required_host_caps_ptr: *const CapabilityId`, `required_host_caps_len: usize`
- `CapabilityId(pub u64)` with `#[repr(transparent)]` — already a newtype but carries a `pub u64` field that flips `no-public-raw-field`.

## Policy proposal

Two tiers based on wire-stability of the chosen Rust type:

1. **arvo `#[repr(transparent)]` newtypes whose backing is wire-identical to the C type.** Use at every `#[repr(C)]` FFI descriptor field that matches. Concretely:
   - `USize(pub usize)` wraps `usize`. Wire layout identical to `usize`. Use for all `len` fields paired with pointers in the descriptor.
   - `Bool(pub bool)` wraps `bool`. Wire layout identical. Use when a descriptor field is a bool flag.
   - `Cap(pub USize)` for const-generic capacity slots (none in the current descriptor; listed for completeness).
2. **Bare primitives with explicit `lint:allow`** for FFI-wire fields whose Rust analog is not layout-stable. This covers:
   - `abi_version: u32`, `CapabilityId(pub u64)`, `ExtensionVersion { major: u16, minor: u16, patch: u16, _reserved: u16 }`. `UFixed<N, 0, S>` is `#[repr(transparent)]` but the backing container width depends on `S: Strategy` (Warm may pick a wider container than requested for speed). An FFI descriptor cannot depend on that.
   - Raw pointer fields (`*const u8`, `*const CapabilityEntry`, `*const CapabilityId`). Rust has no "safe" newtype for a raw foreign pointer.

Each bare-primitive site carries a trailing `// lint:allow(<lint-name>) tracked: #206` comment with a short reason. The comment is the contract: "this field is a declared FFI-wire position; bare primitive is the deliberate choice."

## Why not UFixed at FFI sites

`UFixed<I, F, S>` is `#[repr(transparent)]` over `<S as UContainerFor<{I+F}>>::T`. The container picks a primitive per the Strategy. `Hot` minimises width to the smallest host type that holds `I+F` bits; `Warm` defaults pragmatically; `Cold` may go wider. None of the strategies guarantee "exactly a `u32`" for a 32-bit logical width across all platforms, because the container type trait is implementation-defined per Strategy. FFI requires a fixed ABI footprint; bare `u32` is the honest declaration.

A future arvo primitive `UWire<N>` (fixed backing per N bits, no Strategy axis) would resolve this. Out of scope for this round; noted in hilavitkutin-extensions BACKLOG.

## Changes proposed

### `ExtensionDescriptor` (src + DESIGN)

- `name_len: usize` -> `name_len: USize`
- `capabilities_len: usize` -> `capabilities_len: USize`
- `required_host_caps_len: usize` -> `required_host_caps_len: USize`
- `abi_version: u32` stays, with `// lint:allow(arvo-types-only,no-bare-numeric,no-public-raw-field) tracked: #206` — FFI-wire u32, no layout-stable arvo newtype yet.
- Raw pointer fields stay, with matching `lint:allow` where any lint fires.

### `ExtensionVersion` (src + DESIGN)

- `major / minor / patch / _reserved: u16` stay, with one `lint:allow` block-comment covering the struct's rationale.

### `CapabilityId` (src)

- `pub u64` field annotated with `lint:allow(no-public-raw-field) tracked: #206` — the public raw field is the deliberate FFI-wire constructor pattern.

### BACKLOG: `UWire<N>` arvo primitive

Add a BACKLOG item referring to arvo's BACKLOG: introduce `UWire<N>` for wire-stable N-bit unsigned at `#[repr(C)]` boundaries. When shipped, flip the `lint:allow` sites to `UWire`.

## Out of scope

- `ExtensionMeta::NAME: &'static [u8]` — this is a consumer trait the `#[export_extension]` macro reads. Changing to `&'static str` or `&'static CStr` ripples into the macro and downstream contract crates. Separate round.
- The `no-public-raw-field` lint's behaviour for `#[repr(transparent)]` over a primitive, as a category. If the lint should learn that pattern as legitimate, that's a lint-pack change tracked in `mockspace-hilavitkutin-stack-lints`, not this crate.
- `hilavitkutin-linking::Library::resolve` signature change to `&CStr`. Another round.
