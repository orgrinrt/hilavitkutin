# Premise

A prior session downgraded several lints as a cop-out to unblock a proc-macro crate's alloc usage (eventually fixed at the lint-pack level; see `feedback_no_alloc_no_std_framing.md`). Other lints sit at `off` at commit and `warn` at build/push as a "graceful ramp" pending the sweep tracked as #72 — which is now `completed`, so the ramp can end.

The framing (see `.claude/rules/no-alloc-no-std-framing.md`): lints are the stack's guard rails. Severity-tuning to make commits pass is the cop-out path. If a lint fires, the code is wrong; redesign. The only legitimate exceptions are platform-imposed trait-method signatures, algorithm-fixed widths, and substrate crates that define the primitives other crates consume. These are documented, rare, and accompanied by a linked task.

# Decision

Restore the ten "pending sweep" lints to `error` at commit, build, and push. Fix every resulting violation honestly:

1. **Trait bound first**: if the signature takes or returns a concrete heap container, switch to a trait bound (`impl IntoIterator<Item = T>`, `&[T]`, `&mut impl Collector<T>`, etc.). Collections do not cross ecosystem API boundaries.
2. **Designed primitive second**: if the type should be a substrate primitive (`USize`, `Bool`, `Str`), use it. No bare `usize` / `bool` / `&str`.
3. **Trait-method impl allowance**: `core::Iterator::next() -> Option<T>`, `core::TryFrom::try_from() -> Result<T, E>`, and similar platform-imposed trait-method signatures are the sole legitimate `lint:allow` sites. Each carries a task ID.

No severity tuning. No cop-out allows. If a pattern legitimately needs a persistent exception, it is a rule refinement at the lint-pack level, not a severity tune in this project.

# Risk and fallback

This round's fix count could be substantial. If the exposed violations exceed what a single round can honestly fix without compromising quality:

- Split by crate into per-crate rounds.
- Keep the severity flip in this round, fix one crate per round, ratchet severity at the per-crate lint-scope level in `forbidden-imports` rules so the default-level stays at the target while specific crates ramp individually.

The fallback must never be "downgrade the lint back". The floor this round sets is the new baseline.
