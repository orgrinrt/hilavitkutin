# Doc CL: round 202605042200 — Kit trait split + Replaceable + BuilderResource removal

## What this round changes (substantive)

The substrate adopts a new shape for `Kit`. The v0.1 Kit was a builder-transformer: `pub trait Kit<B> { type Output; fn install(self, builder: B) -> Self::Output; }`. Round-4 replaces this with a declarative Kit that names what it owns:

```rust
pub trait Kit {
    type Units: WorkUnitBundle;
    type Owned: StoreBundle;
}
```

Engine surface change: `SchedulerBuilder<Wus, Stores>` becomes a typestate that accumulates `Wus` and `Stores` through `.add_kit::<K>()` calls (no `K::install`). `.build()` proves `Stores: ContainsAll<Wus::AccumRead> + ContainsAll<Wus::AccumWrite>` and returns a `Scheduler<Wus, Stores>`.

Replaceability lands as an opt-in marker:

```rust
pub trait Replaceable {}
```

Apps can call `Scheduler::replace_resource::<T>(value)` only when `T: Replaceable`. The bound is the substrate's only enforced annotation on Owned types; visibility is a per-kit-crate envelope, not a per-Owned-type knob (per audit C2/M4 + S5 finding).

`BuilderResource<T>` and the sealed-trait bridge layer (`Buildable<Stores>`, `WuSatisfied<A>`, `BuilderExtending<B>`) are deleted entirely per `no-legacy-shims-pre-1.0.md`. Their roles are subsumed by the new Kit shape plus `ContainsAll`.

`feature(marker_trait_attr)` is the substrate keystone for the recursive HList AccessSet pattern; the doc CL captures this risk plainly. The bitset fallback shape lives in `mock/research/marker_trait_attr_bitset_fallback.md` for citation.

**Edit section: SchedulerBuilder definition (locate via grep `SchedulerBuilder` headings).**

The shape change:

- v0.1: `SchedulerBuilder<MAX_UNITS, MAX_STORES, MAX_LANES, Wus, Stores>` carried const generics for size caps.
- Round 4: drop the cap const generics. The new shape is `SchedulerBuilder<Wus, Stores>`. Wus and Stores are HList typestates.

- The `.add_kit::<K: Kit>()` method: rewrites entirely. New signature:
  ```rust
  pub fn add_kit<K: Kit>(self) -> SchedulerBuilder<
      <K::Units as Concat<Wus>>::Out,
      <K::Owned as Concat<Stores>>::Out,
  >
  where
      K::Units: Concat<Wus>,
      K::Owned: Concat<Stores>,
  ```
  No `install` call; the type-level Concat does all the work.
- The `.build()` method: rewrite where-clause. New shape:
  ```rust
  impl<Wus: WorkUnitBundle, Stores: AccessSet + StoreBundle> SchedulerBuilder<Wus, Stores>
  where
      Stores: ContainsAll<Wus::AccumRead> + ContainsAll<Wus::AccumWrite>,
  ```
  The proof shape replaces the existing `Buildable<Stores>` reference.

**Add new method documentation: `Scheduler::replace_resource::<T>(value)`** (locate Scheduler section via grep `Scheduler` heading; add this method after the existing `.run()` and similar sections):

- Signature: `pub fn replace_resource<T: Replaceable>(&mut self, _new: T)`.
- Semantics: replaces the existing Resource<T> instance in the scheduler's data plane with the supplied value. Static bound on T: Replaceable rejects non-opt-in types at compile time.
- Cite S5's negative test (E0277 with a high-quality diagnostic listing the three Replaceable types).
- Note: the method is app-only; called between `Scheduler::builder().build()` and `scheduler.run()`. Not a hot-path operation.

**Edit cross-references throughout.** Wherever the existing prose references `Buildable<Stores>`, `WuSatisfied<A>`, `BuilderExtending<B>`, or `BuilderResource<T>`, rewrite to reference the new substrate (`ContainsAll<L>`, `WorkUnitBundle::AccumRead/Write`, `StoreBundle`).
