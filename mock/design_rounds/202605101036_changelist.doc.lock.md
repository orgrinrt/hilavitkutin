### Tuning surface (where each knob lives)

The plan-affecting cap, micro-morsel cadence, drift cap, and spin budgets are associated `const` items on the `RunCfg` trait. Defaults are written into the trait declaration as `default` impls; consumer impls override by writing the const explicitly. WakeStrategy ratios are fields on the `WakeStrategy` struct, which consumers construct (or accept the substrate default `WakeStrategy::default_hybrid()`) and pass to `ThreadPool::with_wake_strategy()`. Example apps are substrate-shipped demos, not a config surface.

```rust
pub trait RunCfg: Sealed {
    type Out;
    type Err;
    // ... existing assoc types from Topic 1

    /// Cap on resources marked `PlanAffecting`. Drives the dirty-bitmask
    /// width. Default 256 (32-byte bitmask, fits one cache line).
    const MAX_PLAN_AFFECTING_RESOURCES: USize = USize(256);

    /// Records between micro-morsel inner-loop sync points. Pow2 cap.
    /// Default 64 (one cache line of f32-shaped data).
    const MICRO_MORSEL_INTERVAL: USize = USize(64);

    /// Max inter-fiber misalignment in records before forced realign.
    /// Pow2 cap. Default 32 (half a micro-morsel).
    const MAX_DRIFT_RECORDS: USize = USize(32);
}
```

### Defaults table

| Topic | Surface | Default | Tradeoff |
|-------|---------|---------|----------|
| 6 axis K | `WakeStrategy { p_spin_iters, e_spin_iters }` fields on the builder struct | `p_spin_iters = 128`, `e_spin_iters = 32` (4:1) | Matches Apple Silicon P/E latency asymmetry. Workloads with cheap-to-restart E-core work can raise `e_spin_iters` to keep them warmer; workloads with rare phase transitions can lower both. |
| 7 axis C `MICRO_MORSEL_INTERVAL` | `RunCfg::MICRO_MORSEL_INTERVAL` assoc const | `64` | Modulo bench shows pow2 const 2.2x faster than npow2. Compute-heavy workloads with low per-record work raise to 128 to halve sync overhead; I/O-shaped workloads lower to 32 for responsiveness. |
| 7 axis C `MAX_DRIFT_RECORDS` | `RunCfg::MAX_DRIFT_RECORDS` assoc const | `32` | Same pow2 reasoning. Anomaly-rich workloads lower to 16; uniform-cost workloads raise to 64. |
| 8 axis B | `RunCfg::MAX_PLAN_AFFECTING_RESOURCES` assoc const | `256` | 32-byte bitmask fits one cache line; dirty-check scan stays free. Apps composing many sized providers can raise to 512 (still single-line); apps with strict envelopes can lower to 16 or 32 for tighter compile-time bitmask. |
| 9 axis B | CSR `DependencyGraph` backing (no threshold needed) | CSR canonical at every N | Bench `dep_graph_csr_9` shows CSR wins across the full sweep. No consumer knob: dense matrix is structurally inferior at every size; we ship the better one. |
| 11 axis B | `examples/` (substrate-shipped demos, not config) | `word_count` / `particle_collision` / `audio_mix` | User-swapped `particle_step` for `particle_collision` to stress anomaly Virtuals + WakeStrategy under irregular load. Demos pick three distinct value-prop shapes (string interner + map-reduce / branchy SIMD with phase coupling / streaming multi-phase). |

The substrate documents tradeoffs at each knob (in rustdoc on the assoc const / field). It does not enforce a "good" range; consumers pick.

### Consumer-override examples

```rust
// Tight-envelope app (1-4 plan-affecting resources only):
impl RunCfg for MyTinyApp {
    const MAX_PLAN_AFFECTING_RESOURCES: USize = USize(8);
    // other defaults inherited
}

// Compute-heavy app (low sync overhead):
impl RunCfg for MyComputeApp {
    const MICRO_MORSEL_INTERVAL: USize = USize(128);
    const MAX_DRIFT_RECORDS: USize = USize(64);
}

// Cool-running app (warm E-cores):
let pool = ThreadPool::builder()
    .with_wake_strategy(WakeStrategy {
        p_spin_iters: USize(128),
        e_spin_iters: USize(64),
        ..WakeStrategy::default_hybrid()
    })
    .build()?;
```
