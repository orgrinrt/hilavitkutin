### `CoreProgram`

Per-core dispatch program produced by Topic 4 codegen. Holds:
- `dispatch_approach: DispatchApproach` (Topic 4 axis B: A/D/E variants ship)
- `fiber_shapes: [FiberShape; MAX_FIBERS_PER_CORE]` (Topic 4 axis D)
- `morsel_size: USize` (Topic 3 step 9; consumed at runtime by morsel loop)
- `micro_morsel_interval: USize` (Topic 7 axis C)
- `progress_slot_idx: USize` (index into `PoolFrame.progress_slots[]`, Topic 6 axis G)
- `phase_arrived_offset: USize` (which bit of `PoolFrame.phase_arrived` this core owns; Topic 6 axis I)

`CoreProgram` is `#[repr(C)]` and lives in the plan-stage scratch arena (Topic 4 axis E + Topic 5 axis D hot/cold split). Pin<&'frame CoreProgram> is what the worker entry receives (Topic 6 axis C, validated by the GAT sketch).

### Worker entry

```rust
fn worker_entry<'frame, Cfg, E>(
    pool: Pin<&'frame PoolFrame<MAX_CORES, MAX_PHASES>>,
    program: Pin<&'frame CoreProgram<...>>,
    core_id: USize,
) where E: Executor { ... }
```

Per Topic 6 axis A, `Executor::run` is the trait method dispatched here. The default `HybridExecutor::run` body:

1. Loop until `pool.shutdown.load(Relaxed)`:
2.   For each phase in program:
3.     Read `pool.predicted_wait_ns[phase]` to pick tier (Topic 6 axis J).
4.     Run the phase's morsel loop (Topic 7 axes B-E).
5.     Increment `pool.phase_arrived`; futex-wait if < N (Topic 6 axis I).
6.     On last arriver: wake all.
7. Exit cleanly on shutdown.
