# Doc changelist: hilavitkutin-providers v0

The round ships a new ecosystem crate `hilavitkutin-providers`
plus the `InternerKit` shape consumers wire onto the
SchedulerBuilder via `add_kit`. This CL covers the documentation
edits the new crate and supporting templates need.

## Doc template edits

### `mock/crates/hilavitkutin-providers/README.md.tmpl` (new)

Five-line crate summary per the per-crate `README.md.tmpl` rule.
Names the crate's purpose (default Resource-backed providers for
the scheduler) and its dep direction (consumes hilavitkutin-api +
hilavitkutin-str + hilavitkutin-kit, no engine).

### `mock/crates/hilavitkutin-providers/DESIGN.md.tmpl` (new)

The crate's shipping contract. Sections:

- **What it ships in v0.** The interner module and four exported
  types (`InternerApi`, `HasInterner`, `MemoryArena`,
  `InternerKit`).
- **`InternerApi` surface.** Methods, the
  `Send + Sync + 'static` bound rationale.
- **`HasInterner` accessor.** Mirrors the api crate's
  `HasMemoryProvider` / `HasClock` pattern.
- **`MemoryArena<CAP, ENTRIES>` shape.** Inline storage, `Cell`
  and `UnsafeCell` interior mutability, id-encoding scheme,
  capacity overflow returns sentinel id.
- **`InternerKit<CAP, ENTRIES>` shape.** One-line `install` body,
  registers `Resource<StringInterner<MemoryArena<CAP, ENTRIES>>>`.
- **What v0 deliberately does not ship.** The `M: HasMemoryProvider`
  parameter from the original task description (deferred to v1
  when a real consumer needs runtime slab allocation), other
  providers (ColumnStorage, ClockProvider default, MemoryProvider
  default) deferred per task scope.

### `mock/crates/hilavitkutin-providers/BACKLOG.md.tmpl` (new)

Two entries:

- **M-parameterised arena variant.** `MemoryArena<M, CAP>` over a
  consumer-supplied `MemoryProviderApi`. Gated on a real consumer
  needing runtime slab allocation (viola's first integration may
  surface this).
- **Sync arena variant.** Today the `Cell` + `UnsafeCell` pattern
  makes `MemoryArena` `!Sync`. A `SyncArena` variant using
  spinlocks or atomics lands when multi-thread scheduling
  exercises shared interner access.

### `mock/DESIGN.md.tmpl`

Add a new row to the crate table for `hilavitkutin-providers`:

```markdown
| `hilavitkutin-providers` | Default Resource-backed providers for the scheduler. v0 ships InternerKit (interner via inline-storage MemoryArena + StringInterner). Future modules add ColumnStorage / Clock / Memory defaults per consumer demand. Standalone (no engine dep). |
```

The grouping prose at the bottom (paragraph naming
hilavitkutin-ctx / -persistence / -str / -kit as standalone
ecosystem crates) gets one sentence appended naming providers as
the same shape: standalone, consumed by the engine via Kit
installation, no reverse dep.

### `mock/PRINCIPLES.md.tmpl`

No edit needed at this level. The crate's identity is captured by
the DESIGN table row and its own DESIGN.md.tmpl. Principles cover
the substrate-wide invariants the new crate inherits.

### `mock/mockspace.toml`

Three additions, all of which are NEW config (not gate
modifications):

1. `crate_colors` table: add
   `hilavitkutin-providers = "#E6F2E6 | #2D6A2D"` (the standard
   ecosystem-crate green).
2. `layers` list: add `"Ext:providers"` after `"Ext:kit"`.
3. `[lints.forbidden-imports]` rules list: add a new entry for the
   `hilavitkutin-providers` scope. Forbidden list is `hilavitkutin::*,
   hilavitkutin_ctx::*, hilavitkutin_persistence::*, hilavitkutin_build::*,
   hilavitkutin_linking::*, hilavitkutin_extensions::*`. Allowed
   imports include `hilavitkutin_api::*`, `hilavitkutin_str::*`,
   `hilavitkutin_kit::*`, and the substrate.

The pre-tool guard rule on mockspace.toml allows additions of new
config and new lint rules at their designed severity. None of the
above lower a gate.
