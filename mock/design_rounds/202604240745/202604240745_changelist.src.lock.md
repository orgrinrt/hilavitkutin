## S6 — `mock/crates/hilavitkutin-extensions/src/symbol.rs`: new file

```rust
//! Typed symbol handles and the sealed ExtensionSymbol trait.

use core::marker::PhantomData;

pub struct Symbol<'ext, T: ExtensionSymbol> {
    // raw pointer resolved by the loader, typed via T
    ptr: *const core::ffi::c_void,
    _marker: PhantomData<(&'ext (), T)>,
}

pub trait ExtensionSymbol: sealed::Sealed {}

pub struct StaticDescriptor<'ext, T: 'static> {
    _marker: PhantomData<&'ext T>,
}

// Sealed marker impls for FFI-safe types:
// - extern "C" fn(...) -> R  (common arities)
// - StaticDescriptor<'_, T: 'static>
// - *const c_void (escape hatch for advanced consumers)

impl<'ext, T: ExtensionSymbol> Symbol<'ext, T> {
    pub fn get(&self) -> &T {
        // SAFETY: the pointer is valid for the lifetime of the Extension,
        // which this Symbol borrows via the 'ext lifetime.
        unsafe { &*(self.ptr as *const T) }
    }

    pub(crate) fn from_raw(ptr: *const core::ffi::c_void) -> Self {
        Self { ptr, _marker: PhantomData }
    }
}

mod sealed {
    pub trait Sealed {}
}
```

Impls for `ExtensionSymbol` cover `extern "C" fn()` through `extern "C" fn(A, B, C, D)` (zero to four argument arities for v1; extend as needed in follow-up rounds), `StaticDescriptor<'_, T>` for arbitrary `T: 'static`, and `*const c_void` as a pointer-escape.

## S10 — `mock/crates/hilavitkutin-extensions/tests/basic.rs`: integration tests

Tests build a cdylib fixture at test time, load it via `Extension::load`, resolve known symbols, and verify behaviour. Fixtures live at `tests/fixtures/simple_plugin/` with its own tiny Cargo.toml declaring `crate-type = ["cdylib"]`.

Test matrix:
- Load a known-good fixture path (success).
- Load a non-existent path (returns `ExtensionError::PathNotFound` or `LoadFailed` — platform-dependent which).
- Resolve a known symbol from the loaded fixture (returns `Symbol`).
- Resolve a missing symbol (returns `ExtensionError::SymbolMissing`).
- Call the resolved function pointer through `Symbol::get()`, verify the correct value returned.
- Drop the `Extension` and verify no use-after-free (compile-time check via lifetime; runtime sanity via drop order tests).

`compatibility_check` is exercised separately against a known version-bytestring input.
