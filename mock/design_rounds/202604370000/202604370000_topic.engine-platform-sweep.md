# Current state

`platform/` has 32 warnings across `os.rs` and `std_tier.rs`. The
module is a thin shim translating between `hilavitkutin_api::platform`
traits (`MemoryProviderApi`, `ThreadPoolApi`, `ClockApi`) and platform
primitives (libc mmap / pthread, std::alloc / std::thread,
CLOCK_MONOTONIC / Instant).

Three categories of site:

1. **Trait impl signatures** mirroring `*mut u8` from
   `MemoryProviderApi` (9 sites across both files). Already carry
   structural lint:allow in the api crate (round 202604330000); the
   engine-side impls need matching annotation.
2. **Platform-ABI body casts** — `as usize`, `as u64`,
   `transmute::<usize, fn()>`, `align_of::<usize>()`. Cannot be
   arvo-typed: they address C ABI pointer sizes (`as usize` in
   trampoline, `align_of::<usize>` for `Layout` ABI). 6 sites.
3. **`StdClock::now_ns` — out of sync with api.** Returns `u64`
   instead of the new `Nanos`. Update the signature and wrap via
   `Nanos::from_raw`. Caught as a lint warning but also a latent
   compile error under `--features platform-std` (not the default).
   1 site (paired with one body cast line).

# Changes proposed

## Axis 1 — api-trait impl raw-pointer sigs (6 sites)

Trailing inline `lint:allow(no-bare-numeric) lint:allow(arvo-types-only)
reason: allocator ABI raw pointer; tracked: #72` on each
`allocate`/`deallocate`/`protect` sig in `os.rs` and `std_tier.rs`.

## Axis 2 — platform-ABI body sites (6 sites)

Trailing inline `lint:allow(no-bare-numeric) lint:allow(arvo-types-only)
reason: platform ABI; <specific>; tracked: #72` on:

- `os.rs` L113 `let raw = f as usize;` (pthread_create arg smuggling)
- `os.rs` L129 `let raw = arg as usize;` (trampoline entry)
- `os.rs` L133 `transmute::<usize, fn()>` (recover fn pointer)
- `os.rs` L196 `let raw = ... as u64` (CLOCK_MONOTONIC nanos literal)
- `std_tier.rs` L37 `align_of::<usize>()` (Layout ABI default)
- `std_tier.rs` L44 `align_of::<usize>()` (Layout ABI fallback)

## Axis 3 — `StdClock::now_ns` alignment with `Nanos`

```rust
// before
fn now_ns(&self) -> u64 {
    let epoch = EPOCH.get_or_init(Instant::now);
    Instant::now().duration_since(*epoch).as_nanos() as u64
}

// after
fn now_ns(&self) -> Nanos {
    let epoch = EPOCH.get_or_init(Instant::now);
    let raw = Instant::now().duration_since(*epoch).as_nanos() as u64; // lint:allow(...) reason: Duration::as_nanos returns u128; truncate to u64 for Nanos; tracked: #72
    Nanos::from_raw(raw)
}
```

Import `hilavitkutin_api::Nanos`.

# Non-goals

- No changes to `mod.rs` — only platform trait re-exports, already clean.
- Not refactoring platform ABI to accept arvo newtypes — `usize` bit
  patterns *are* the ABI; wrapping would defeat the trampoline purpose.
