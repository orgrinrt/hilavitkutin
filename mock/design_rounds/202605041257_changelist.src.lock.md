## CHANGE: new module `mock/crates/hilavitkutin-providers/src/interner.rs`

File: `mock/crates/hilavitkutin-providers/src/interner.rs`
Verification:
- `grep "pub trait InternerApi" mock/crates/hilavitkutin-providers/src/interner.rs` returns one match.
- `grep "pub trait HasInterner" mock/crates/hilavitkutin-providers/src/interner.rs` returns one match.
- `grep "pub struct MemoryArena" mock/crates/hilavitkutin-providers/src/interner.rs` returns one match.
- `grep "pub const fn default_interner" mock/crates/hilavitkutin-providers/src/interner.rs` returns one match.

The module ships:

1. `pub trait InternerApi: 'static` with `intern(&self, s: &str) -> Str` and `resolve(&self, s: Str) -> Maybe<&str>` methods.
2. `pub trait HasInterner` with associated `Provider: InternerApi` and `interner(&self) -> &Self::Provider`.
3. `pub struct MemoryArena<const BYTES: usize, const ENTRIES: usize>` with byte-buffer + entry-table interior mutability via `Cell` + `UnsafeCell`. Implements `arvo_str::ArenaInterner`. Sentinel-id overflow on capacity exceeded.
4. Blanket impl `impl<A: ArenaInterner + 'static> InternerApi for StringInterner<A>` so the consumer's wired `Resource<StringInterner<MemoryArena<...>>>` automatically satisfies the `HasInterner::Provider: InternerApi` bound at the Context tuple.
5. `pub const fn default_interner<const BYTES: usize, const ENTRIES: usize>() -> StringInterner<MemoryArena<BYTES, ENTRIES>>` builds a ready-to-register interner. Consumers wire it onto the scheduler with `builder.resource(default_interner::<BYTES, ENTRIES>())`.
