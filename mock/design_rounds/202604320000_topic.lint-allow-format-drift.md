# Topic: lint:allow comment format drift across hilavitkutin source

**Date:** 2026-04-21
**Status:** TOPIC phase
**Task:** #72 slice (residual; collateral finding from arvo #129 audit)

## Context

Task #80 landed the `reason: <why>; tracked: #<task-id>` format for
`lint:allow(...)` comments. Nine sites in hilavitkutin source still
carry the pre-#80 form `-- <reason>` or no structured reason at all.
The `lint-allow-requires-task-id` lint emits warnings for each.

## Sites

- `hilavitkutin-api/src/hint.rs:18` — `pub trait Sealed {} // lint:allow(undocumented_type)`
- `hilavitkutin-api/src/lib.rs:21` — `pub(crate) trait Sealed {} // lint:allow(undocumented_type)`
- `hilavitkutin-persistence/src/manifest.rs:17,23` — old form `-- tracked: #72; array-size const must be usize`
- `hilavitkutin-persistence/src/lib.rs:34` — old form `-- StringTable ... not std String`
- `hilavitkutin-str/src/interner.rs:21,25` — old form `-- interner wrapper name`
- `hilavitkutin-str/src/lib.rs:25` — old form `-- StringInterner is the no-alloc wrapper`
- `hilavitkutin-str/src/ergonomics.rs:4,23,27,33` — old form `-- interner wrapper`

## Decision

Reformat all nine sites to the `reason: <why>; tracked: #72` form.
No semantic change; the underlying escapes remain justified (sealed
marker traits, type names containing "String" for no-alloc-named
types, rust-grammar-required `usize` at array-size position). Track
all nine under #72 as residual format cleanup; the underlying
remediation (moving away from `usize` arrays where possible,
documenting sealed traits) is the larger sweep's scope.

## Out of scope

- Removing any lint:allow escape (all nine are structurally justified).
- Adding trait docstrings to seal markers (orthogonal; mod-level
  docs already explain the seal pattern).
- Converting `MAX_TABLES: usize` / `MAX_COLUMNS_PER_TABLE: usize` to
  `Cap` (rust grammar requires `usize` at array-length position —
  same #121 class as arvo-tensor's `cap_size`).
