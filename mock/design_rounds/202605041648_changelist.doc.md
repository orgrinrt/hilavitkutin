```markdown
## Trampoline preservation under LTO

The proc-macro emits two trampolines (`__ext_init_trampoline_<Struct>`, `__ext_shutdown_trampoline_<Struct>`) that wrap the consumer's `init` / `shutdown` types. Their addresses are stored as fn-pointer constants in the `__HILAVITKUTIN_EXT_DESCRIPTOR` static. The static is `#[used]`, and the descriptor function (`__hilavitkutin_extension_descriptor`, `#[unsafe(no_mangle)] pub extern "C"`) returns its address. This chain preserves the trampolines under aggressive LTO + cdylib stripping: the export forces the descriptor function to keep, the function returns the static, the static is `#[used]`, and LLVM cannot elide a function whose address has been taken in a preserved static. The trampolines also carry `#[inline(never)]` to defend against an exotic LTO path that might inline the trampoline body into the descriptor's initializer expression. The trampolines are not `#[no_mangle]`: their addresses suffice for the descriptor's runtime use, and avoiding external linkage avoids cross-extension symbol collisions if a host ever loads two extensions whose struct names overlap.

A regression test under `hilavitkutin-extensions-macros/tests/lto_smoke.rs` builds a sample cdylib with `lto = "fat"` and asserts the trampoline symbols survive in `nm` output. Runtime callability is verified once a real consumer extension exercises the descriptor's fn-pointer.
```
