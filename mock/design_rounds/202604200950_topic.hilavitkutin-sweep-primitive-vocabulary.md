## What this round is

The primitive-vocabulary retrofit (Task #72) expanded in scope once
the first-pass audit gave way to a full-file walk. Two things
changed:

1. **Drift count doubled.** The subagent first pass counted ~25
   in-scope sites focused on `is_empty() -> bool` and
   `Option<T>` / `Result<T, E>` returns. A direct grep over every
   `pub fn` signature in `hilavitkutin/mock/crates/` surfaced another
   ~20 sites: `pub fn ... -> u64` / `(core_count: u16, ...)` /
   `MorselRange.start: u64` public fields / const-generic
   `<const N: usize>` type params / `AccessMask::raw() -> u128`
   raw-bits accessors / resource-accumulator `get/set(i: usize)`.
2. **Semantic aliases, not just newtypes.** A direct flip from
   `u32` to `USize` / `Cap` reads as poorly as the raw form
   — `has_edge(from: USize, to: USize)` tells a reader the *width*,
   not the *meaning*. The round adopts semantic domain aliases in
   every public position: `UnitId` (already exists) for graph
   nodes, `StoreId` (already exists in hilavitkutin-api) for
   access/dirty mask indices, new `pub type` aliases (`RecordIndex`,
   `CoreCount`, `FiberCount`, `MorselLen`, `SieveWeight`,
   `ContentHash`, `CodegenUnits`, `Hash64`) for the positions
   without a pre-existing domain type.

The round is therefore larger than the subagent's triage suggested
— roughly 45 in-scope signature sites + 4 const-generic parameters
+ 2 pub-field declarations + a handful of `allow-bare-numeric`
annotations for genuinely representation-layer accessors. Still a
single round — same shape as arvo sweep (#71), but with domain
naming applied throughout.

## Existing semantic aliases reused (no new definition)

- `UnitId` — `hilavitkutin::plan::inputs::UnitId(pub u32)`. Used by
  `DependencyGraph::{has_edge, add_edge}`.
- `StoreId` — `hilavitkutin_api::id::StoreId(pub USize)`. Used by
  `AccessMask::{set, contains}` and `DirtyMask::{set, contains}`.
  Converts `u32` param → `StoreId` at the API boundary; the body's
  `idx >= 128` bound check becomes `idx.0 >= Cap::from_usize(128)`
  (or the equivalent USize comparison).
- `FiberId` — already used as `dispatch::sync::Sync::new(fiber_id:
  FiberId, …)`; stays.
- `Str` — hilavitkutin-str's interned string handle; stays for
  `Interner::resolve`.

## New semantic aliases introduced in this round

Each new alias is a `pub type X = <representation>;` — pure
rename, no wrapping. Placed near the top of its crate's `lib.rs`
(or a tiny `aliases` module where multiple aliases live). Rationale
is "semantic aliases" memory: compile-time readability win, zero
runtime cost, no breakage because aliases are structurally
transparent.

- `hilavitkutin::aliases::RecordIndex = u64` — record index within a
  column/store. Used by `ProgressCounter::{new, store, load}` (the
  AtomicUsize inside flips to AtomicU64 to match, so the public
  type is coherent), `MorselRange::start` (pub field),
  `MorselRange::end() -> RecordIndex`, `dispatch::select_approach::record_count`,
  `dispatch::sync::Sync::new::min_records`, `plan::steps::size_morsels::record_count`.
- `hilavitkutin::aliases::CoreCount = u16` — CPU core count. Used by
  `thread::pool::ThreadPool::new::core_count`, `thread::classify_cores::{total_cores, p_cores}`.
- `hilavitkutin::aliases::FiberCount = u16` — concurrent fibers in a
  pipeline. Used by `dispatch::select_approach::fiber_count`.
- `hilavitkutin::aliases::MorselLen = u32` — records per morsel.
  Used by `MorselRange::len` (pub field), `MorselRange::new::len`,
  return slot of `plan::steps::size_morsels`.
- `hilavitkutin-persistence::aliases::SieveWeight = u64` — LFU
  admission weight. Used by `Sieve::insert::weight`.
- `hilavitkutin-persistence::aliases::ContentHash = u32` — interned-
  string content hash. Used by `StringTable::lookup::content_hash`
  and `archive_str::evict_str -> ContentHash`.
- `hilavitkutin-str::aliases::Hash64 = u64` — FNV-1a hash result.
  Used by `const_fnv1a(&str) -> Hash64`.
- `hilavitkutin-build::aliases::CodegenUnits = u8` — rustc codegen-
  units count (small integer). Used by
  `BuildPragma::parallel_codegen_units -> Maybe<CodegenUnits>`.

## In-scope drift (full list, by crate)

### hilavitkutin

`plan/access.rs` (`AccessMask`):
- `pub struct AccessMask<const MAX_STORES: usize>` → `<const MAX_STORES: Cap>`.
- `is_empty(&self) -> bool` → `-> Bool`.
- `set(self, idx: u32) -> Self` → `(idx: StoreId)`.
- `contains(&self, idx: u32) -> bool` → `(idx: StoreId) -> Bool`.
- `overlaps(&self, other: &Self) -> bool` → `-> Bool`.
- `raw(&self) -> u128` — kept `u128`; `// allow-bare-numeric: tracked: #72` (raw bit-pattern accessor, explicitly low-level).

`plan/dirty.rs` (`DirtyMask`): same pattern as `AccessMask`.
- `<const MAX_STORES: usize>` → `Cap`.
- `is_empty -> Bool`; `set(idx: StoreId)`; `contains(idx: StoreId) -> Bool`.
- `raw -> u128` kept with `allow-bare-numeric: tracked: #72`.

`plan/graph.rs` (`DependencyGraph`):
- `<const MAX_UNITS: usize>` → `<const MAX_UNITS: Cap>`.
- `has_edge(&self, from: u32, to: u32) -> bool` → `(from: UnitId, to: UnitId) -> Bool`.
- `add_edge(&mut self, from: u32, to: u32)` → `(from: UnitId, to: UnitId)`.

`plan/steps.rs`:
- `size_morsels<const MAX_FIBERS: usize>(record_count: u64) -> [u32; MAX_FIBERS]`
  → `<const MAX_FIBERS: Cap>(record_count: RecordIndex) -> [MorselLen; cap_size(MAX_FIBERS)]`.

`dispatch/mod.rs`:
- `select_approach(record_count: u64, fiber_count: u16) -> DispatchApproach`
  → `(record_count: RecordIndex, fiber_count: FiberCount)`.

`dispatch/morsel.rs` (`MorselRange`):
- `pub start: u64` → `pub start: RecordIndex`.
- `pub len: u32` → `pub len: MorselLen`.
- `new(start: u64, len: u32)` → `(start: RecordIndex, len: MorselLen)`.
- `end(&self) -> u64` → `-> RecordIndex`.
- `is_empty(&self) -> bool` → `-> Bool`.

`dispatch/progress.rs` (`ProgressCounter`):
- Inner `AtomicUsize` → `AtomicU64` (matches the external `RecordIndex = u64`).
- `new(start: usize)` → `(start: RecordIndex)`.
- `store(&self, value: usize)` → `(value: RecordIndex)`.
- `load(&self) -> usize` → `-> RecordIndex`.

`dispatch/sync.rs`:
- `Sync::new(fiber_id: FiberId, min_records: u64)` → `(..., min_records: RecordIndex)`.

`thread/pool.rs`:
- `ThreadPool::new(core_count: u16, wake: WakeStrategy)` → `(core_count: CoreCount, ...)`.

`thread/mod.rs`:
- `classify_cores(total_cores: u16, p_cores: u16) -> [CoreClass; 256]`
  → `(total_cores: CoreCount, p_cores: CoreCount) -> [CoreClass; 256]`.
  Return array stays `[_; 256]` — fixed-size descriptor, not a
  semantic count.

`resource/accumulator.rs` / `resource/cache.rs` / `resource/snapshot.rs`:
- Every `pub fn get(&self, i: usize) -> T` / `pub fn set(&mut self, i: usize, ...)` → `(i: USize, ...)`.
  These are low-level slot accessors inside fixed arrays; the
  abstraction is "index into my fixed array", which `USize` names
  precisely. No domain alias needed.

### hilavitkutin-str

`src/handle.rs` (`Str`):
- `is_const(self) -> bool` → `-> Bool`.
- `is_runtime(self) -> bool` → `-> Bool`.
- `id(self) -> u32` — kept `u32`; `// allow-bare-numeric: tracked: #72` (opaque handle representation — the 28-bit ID + flag bits are literally this value).
- `__make(id: u32)` / `__runtime(id: u32)` — kept; same rationale (construction takes the opaque representation).

`src/hash.rs`:
- `const_fnv1a(s: &str) -> u64` → `-> Hash64`.

`src/interner.rs`:
- `Interner::resolve(&self, s: Str) -> Option<&str>` → `-> Maybe<&str>`.

### hilavitkutin-persistence

`src/sieve.rs` (`SieveCache`):
- `<K, V, const CAP: usize>` → `<K, V, const CAP: Cap>`.
- `len(&self) -> usize` → `-> USize`.
- `is_empty(&self) -> bool` → `-> Bool`.
- `capacity(&self) -> usize` → `-> USize` (runtime-readable echo of the Cap const).
- `insert(&mut self, key: K, value: V, weight: u64) -> Option<V>` → `(…, weight: SieveWeight) -> Maybe<V>`.
- `get(&mut self, key: &K) -> Option<&V>` → `-> Maybe<&V>`.
- `evict(&mut self) -> Option<(K, V)>` → `-> Maybe<(K, V)>`.

`src/cold_store.rs`:
- `open(...) -> Result<Self, PersistenceError>` → `-> Outcome<Self, PersistenceError>`.
- `flush / load / snapshot -> Result<(), PersistenceError>` → `-> Outcome<(), PersistenceError>`.

`src/string_table.rs`:
- `lookup(&self, content_hash: u32) -> Option<&[u8]>` → `(content_hash: ContentHash) -> Maybe<&[u8]>`.

`src/archive_str.rs`:
- `evict_str<A: ArenaInterner>(handle: Str, interner: &StringInterner<A>) -> u32` → `-> ContentHash`.

### hilavitkutin-build

`src/config.rs`:
- `BuildConfig::fast_math(&self) -> bool` → `-> Bool`.

`src/pragma.rs`:
- `BuildPragma::contains(self, p: Pragma) -> bool` → `-> Bool`.
- `BuildPragma::parallel_codegen_units(self) -> Option<u8>` → `-> Maybe<CodegenUnits>`.

## Allowed escape hatches (not in scope)

- `impl Iterator for PragmaIter { fn next(&mut self) -> Option<Pragma> }` — std trait, stays.
- `impl Display / Debug { fmt::Result }` — std trait.
- `#[repr(u8/u16/u32/u64/u128)]` enum discriminants — low-level ABI layout.
- Internal (private) fields and let-bindings.
- `AccessMask::raw / DirtyMask::raw / Str::id / Str::__make / Str::__runtime` — representation-layer accessors carrying `allow-bare-numeric: tracked: #72` with documented rationale.
