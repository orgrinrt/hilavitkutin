## CHANGE: macro emission `init_trampoline` and `shutdown_trampoline` ADD per-role byte-string discriminator via `core::hint::black_box`

File: `mock/crates/hilavitkutin-extensions-macros/src/lib.rs:170-180, 192-202`

Each trampoline body gains a leading line `let _ = ::core::hint::black_box(b"<role>");` where `<role>` is `init` or `shutdown`. The `black_box` is a no-op assembly fence that LLVM cannot see through, which forces the trampoline bodies to differ at the byte level even when the wrapped impls are byte-identical. ICF cannot alias them.

Verification: `grep -c 'black_box(b"init")' mock/crates/hilavitkutin-extensions-macros/src/lib.rs` returns 1, and `grep -c 'black_box(b"shutdown")' mock/crates/hilavitkutin-extensions-macros/src/lib.rs` returns 1.

## CHANGE: integration test `tests/lto_smoke.rs` EXTEND with dlopen + descriptor read + distinct-address assert

File: `mock/crates/hilavitkutin-extensions-macros/tests/lto_smoke.rs`

After the existing `nm` substring assertions, the test now:

1. dlopens the cdylib via `libloading::Library::new` (libloading is added as a dev-dep).
2. Resolves `__hilavitkutin_extension_descriptor` as `unsafe extern "C" fn() -> *const ExtensionDescriptor`.
3. Calls it, reads the descriptor.
4. Asserts `descriptor.init_fn.is_some()` and `descriptor.shutdown_fn.is_some()`.
5. Asserts the two `Option<unsafe extern "C" fn(*mut c_void) -> ExtensionAbiStatus>` values are distinct as fn-pointers.

The dev-dep on `libloading` is added to `hilavitkutin-extensions-macros/Cargo.toml`. The forbidden-imports lint on the macros crate forbids `hilavitkutin_linking::*` runtime use, which `libloading` is not.

Verification: `grep -c "libloading::Library" mock/crates/hilavitkutin-extensions-macros/tests/lto_smoke.rs` returns at least 1, and `grep -c "init_fn != shutdown_fn\|distinct addresses" mock/crates/hilavitkutin-extensions-macros/tests/lto_smoke.rs` returns at least 1.
