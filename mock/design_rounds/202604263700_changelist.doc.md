# Doc Changelist: hilavitkutin engine 5a3 — dispatch + intrinsics skeleton

**Date:** 2026-04-26

**Source topics:**

## Purpose

Third-to-last engine sub-round. Ships the public type surface +
code-generation scaffolding for dispatch (domain 17) and the
low-level intrinsics shim (domain 13). Skeleton only: every
code-emit function stubs to `todo!()`; intrinsics stub to no-ops
or `feature = "intrinsics-std"`-gated `core::hint` / `core::sync`
primitives. Downstream rounds fill in real LLVM hooks.

Dispatch is what turns an `ExecutionPlan` (5a2 output) into
executable code: per-fiber monomorphised dispatch functions,
per-core compiled pipelines, progress counters. Intrinsics is
the low-level shim the engine uses for atomics + prefetch +
cache hints + optimiser barriers.

Landing the skeleton now unblocks 5a4 (thread scheduler) which
needs to reference `FiberDispatch`, `CoreDispatch`, `ProgressCounter`,
and `SyncPoint` in its own signatures.

## Deliverables

### Dispatch types (domain 17)

All under new `src/dispatch/` submodule tree.

- `FiberDispatch<const MAX_CORES: usize>` — per-fiber dispatch
  function record. Holds a function pointer + owned metadata
  (fiber_id, phase, morsel_range, sync_points).
- `CoreDispatch<const MAX_FIBERS: usize>` — per-core monomorphised
  pipeline. Phases + record ranges + morsel boundaries + sync
  points + fiber assignments.
- `DispatchApproach` enum: `IndirectPerFiber`, `TrunkMega`,
  `ScheduleMega`. Selected by record count (see DESIGN
  §"Dispatch codegen").
- `ProgressCounter` — newtype `#[repr(transparent)]` over
  `AtomicUsize`. Release store, Acquire load. Lock-free. Per-fiber
  monotonic record index.
- `SyncPoint { fiber_id: FiberId, min_records: u64 }` — phase-join
  gate. Phase N+1 reads this to check if phase N produced enough
  records to start the next morsel.
- `MorselRange { start: u64, len: u32 }` — one morsel's record
  slice.
- `WuFn<Ctx>` type alias — function-pointer shape used inside
  fiber dispatch records. Matches the `WorkUnit::execute(&self, &Ctx)`
  contract from hilavitkutin-api v2; the monomorphised emit form
  is `fn(&Ctx)`.

### Intrinsics (domain 13)

New `src/intrinsics/mod.rs` — the engine's portable low-level
shim. Ships zero-cost no-ops on stable; gated upgrades behind
`intrinsics-std` feature for platforms where std-backed paths
are available.

- `fn prefetch_l1<T>(ptr: *const T)` — L1 prefetch. No-op on
  stable; optional `core::intrinsics::prefetch_read_data` call
  under `intrinsics-std`.
- `fn prefetch_l2<T>(ptr: *const T)` — L2 prefetch. Same shape
  with locality 1.
- `fn compiler_fence_release()` — `core::sync::atomic::compiler_fence(Ordering::Release)`.
- `fn compiler_fence_acquire()` — same with Acquire.
- `fn noinline_barrier<T>(val: T) -> T` — optimiser barrier.
  `core::hint::black_box` under `intrinsics-std`, identity
  otherwise.

### Deferrals (all → BACKLOG)

- `select_approach` body (needs benchmarked thresholds; 10K
  cutover is the documented target).
- `codegen_fiber` body (needs LLVM hooks or a build-time plugin
  from hilavitkutin-build).
- `codegen_core` body (encodes phases + morsel boundaries + sync
  points; depends on `codegen_fiber`).
- ExpandedLto pragma integration (depends on hilavitkutin-build
  5a4+).
- Rust-pipe pattern: stack-cached resources, fiber-input reads at
  morsel start, DSE-friendly internal column reads, grouped
  output stores.
- Progress counter thread-safety stress tests.
- Intrinsics: real `core::intrinsics::prefetch_*` wiring on
  nightly.
- Intrinsics: platform-specific inline asm paths (x86_64
  PREFETCHT0, aarch64 PRFM).
