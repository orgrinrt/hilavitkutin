# Deliverables

## D1 — expand DESIGN.md.tmpl public surface

Update `mock/crates/hilavitkutin-extensions/DESIGN.md.tmpl`:

1. "Public items" list gains `StaticRef` (new typed wrapper for pointer-to-static-data symbol resolution, distinct from `Symbol` which covers function pointers).
2. "Shape" section extends to describe:
   - `Extension::resolve_static<T: 'static>(&self, name: &[u8]) -> Outcome<StaticRef<'_, T>, ExtensionError>` — resolves a symbol whose address holds a `T` (typical static-data pattern for plugin descriptor exports).
   - `StaticRef::get(&self) -> &T` — borrows the static data; lifetime tied to `Extension` via the `'ext` parameter.
   - `Extension::close(self) -> Outcome<(), ExtensionError>` — explicit unload with error reporting. Drop still works (RAII default); `close` is the explicit form.
3. Note in Shape that `ExtensionSymbol` now admits arities zero through eight.
4. Scope-cut note in Non-scope that `Symbol` (fn-pointer) and `StaticRef` (static-data) are deliberately two types — pointer reinterpretation rules differ; mixing them would hide soundness behind unsafe. Both types co-exist.
