# Topic: rehab engine test fixtures past USize/Bool/Bits API drift

## Scope

Pre-existing test compile failures across the engine crate, exposed by
arvo Round D adoption (USize const-arith, Bool typed predicate, UFixed
Nanos type) and not addressed in earlier sweeps (#291, #292). The
hilavitkutin PR #73 (Provider unification) merged with these test
failures still latent because the senior reviewer flagged them as
pre-existing rather than caused by the round. Tracked under task #399.

The rehab is mechanical: every bare integer literal in a test that
constructs an arvo-typed value gets wrapped in `USize(...)` (or stays
bare where the receiving newtype takes a fixed bit-width raw, like
`FiberId(pub u16)` or `PhaseId(pub u8)`). Every `assert_eq!` against a
`Bool` predicate gets the explicit `Bool::TRUE` / `Bool::FALSE` form.
Every `Nanos` (= `UFixed<64,0,Hot>`) value gets `to_raw()` applied
before format / comparison since UFixed lacks Display + PartialOrd at
this layer.

## Decisions

1. **No new lint:allow tokens beyond the established `no-bare-numeric`
   pattern.** Each test-side wrap or literal that survives carries a
   trailing `// lint:allow(no-bare-numeric) reason: ...; tracked: #399`
   comment, per workspace rule `lint-allow-same-line-only.md` and
   `no-complex-lint-allows.md`.

2. **`Slot(UFixed::from_raw(N))` wrapping** for the `ResourceSnapshot`
   tests. `Slot(pub UFixed<64,0,Hot>)` source-side; bare integer
   literal cannot coerce into `UFixed`. `to_raw()` for read-side
   comparisons: UFixed lacks `Debug` so `assert_eq!` against a UFixed
   value fails the trait-solve.

3. **Maybe predicate name is `isnt()`**, not `is_none()`. Notko's pinned
   API at `b8509c9` exposes `is()` / `isnt()` / `is_some_and` /
   `is_none_or`. The std-parallel `is_none()` retired earlier; the test
   migrates to the substrate name.

4. **Bool comparison via `Bool::TRUE` / `Bool::FALSE`**, not via std
   `assert!`. Bool does not auto-convert into the `bool`-typed
   predicate-position that std `assert!` requires; the workaround would
   be `.as_bool()`, but `assert_eq!(x, Bool::TRUE)` reads cleaner and
   exercises the typed equality path.

5. **`Nanos.to_raw()` for monotonicity comparison + Display**. The
   `os_clock_is_monotonic` test compared two `Nanos` values directly;
   `UFixed<64,0,Hot>` lacks `PartialOrd` + `Display`. The migration
   unwraps both to raw `u64` at the comparison and format sites.

6. **`AdaptConfig::default()` removed**: source no longer provides a
   `Default` impl on `AdaptConfig`. The test's `adapt_config_default_is_zero`
   path now spells the zero-construction explicitly via `AdaptConfig::new`
   with `USize::ZERO` for each numeric field.
