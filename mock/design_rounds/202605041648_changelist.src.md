# Source changelist: trampoline LTO survival (revised)

Replaces the deprecated `202605041648_changelist.src.deprecated.md`. The deprecated CL planned `#[inline(never)]` only on the trampolines; the smoke test that this CL ships caught the LLVM identical-code-folding pass merging the two trampolines because their bodies were structurally identical after the consumer's `Init::init` and `Shutdown::shutdown` impls inlined to `ExtensionAbiStatus::Ok`. The init trampoline's symbol survived; the shutdown trampoline's symbol disappeared and the descriptor's `shutdown_fn` slot ended up pointing at the same address as `init_fn`. This revised CL adds `#[unsafe(no_mangle)]` alongside `#[inline(never)]` on both trampoline emissions.

## CHANGE: macro emission `init_trampoline` and `shutdown_trampoline` ADD `#[unsafe(no_mangle)]` and `#[inline(never)]`

File: `mock/crates/hilavitkutin-extensions-macros/src/lib.rs:170-180, 192-202` (line numbers approximate; the trampoline `quote! { ... }` blocks gain two attribute lines each).

Both trampolines emit:

```rust
#[unsafe(no_mangle)]
#[inline(never)]
unsafe extern "C" fn #fn_ident(...) -> ExtensionAbiStatus { ... }
```

The init trampoline carries an inline comment block explaining the load-bearing role of `no_mangle` against ICF. The shutdown trampoline elides the comment as repetition would not aid the reader.
