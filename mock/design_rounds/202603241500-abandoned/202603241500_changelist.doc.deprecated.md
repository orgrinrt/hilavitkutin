## Comparison to deprecated changelist

The deprecated changelist (202603231200_changelist.doc.deprecated.md)
was the first doc CL attempt. It was deprecated because the arvo
strategy markers (Hot/Warm/Cold/Precise) were undefined — a design
gap discovered during src planning.

This changelist replaces it with:
- Strategy markers fully designed across three new topic files
- New crate: arvo-bits (bit-level contracts, sits between arvo and arvo-bitmask)
- 10 crates instead of 9
- Revised arvo dependency tree
- Sub-crates generic over trait bounds, not concrete types
- Strategy-generic semantic aliases (`Angle<S: Strategy = Warm>`)

### Comparison to deprecated changelist

The deprecated CL had 9 crates without strategy markers. This CL has
10 crates with fully designed strategy markers. All types, traits,
and macros from the deprecated CL (AccessSet, AccessMask, Always,
On, Clock, MemoryProvider, ThreadPool, Column, Resource, Virtual,
Context, Contains, WorkUnit, ColumnValue, SchedulingHint, Field,
Seq, Map) are carried forward unchanged. The only removal is
WorkUnit::NAME (replaced by consumer-side naming).

### Per-crate changes

Deprecated CL had: arvo, arvo-bitmask, arvo-graph, arvo-sparse,
arvo-spectral, arvo-comb, hilavitkutin-api, hilavitkutin,
hilavitkutin-build (9 crates).

This CL adds: arvo-bits (new crate between arvo and arvo-bitmask).
Updates: arvo (strategy markers), arvo-bitmask (deps: arvo-bits
instead of arvo), all sub-crates (generic over trait bounds).
hilavitkutin-api and hilavitkutin unchanged structurally.
hilavitkutin-build unchanged.

### Scope

Same scope as deprecated: 10 crates (7 arvo + 3 hilavitkutin),
zero dependency on polka-*. Designed for extraction to separate
repositories.

### Crate structure

Revised from 9 to 10 crates. arvo-bits inserted between arvo and
arvo-bitmask.

### 1. CREATE mock/crates/hilavitkutin-api/

Same types as deprecated CL. Strategy markers now part of the
ColumnValue system — UFixed/IFixed with strategy parameter satisfy
the ColumnValue blanket impl. AccessSet, AccessMask, Column,
Resource, Virtual, Always, On<V>, Context, Contains, MemoryProvider,
ThreadPool, Clock all unchanged.

WorkUnit: NAME dropped (consumer concern). Scheduling hints replace
order field. All other types unchanged.

### 2. CREATE mock/crates/hilavitkutin/

Same domain map as deprecated. Strategy markers affect domain 14
(feasibility check uses size_of which depends on strategy container)
and domain 12 (morsel sizing). Both already designed for type-native
stride (R3). No structural changes.

### 3. docs/DESIGN.md

Per-crate DESIGN.md.tmpl files. Content updated for strategy markers
and arvo-bits.

### Files NOT changed

No polka-* templates modified.

### Execution order

Dependency-ordered by mockspace generation. No manual ordering.

## Crate structure

```
arvo                   numeric primitives + strategy markers (standalone)
arvo-bits              bit-level contracts (deps: arvo)
arvo-bitmask           fixed-width bitmask types (deps: arvo-bits)
arvo-sparse            sparse matrix (deps: arvo, arvo-bitmask)
arvo-graph             DAG algorithms (deps: arvo, arvo-bitmask)
arvo-spectral          spectral methods (deps: arvo, arvo-sparse)
arvo-comb              combinatorial optimisation (deps: arvo, arvo-bitmask)
hilavitkutin-api       consumer-facing types, traits (deps: arvo, arvo-bitmask)
hilavitkutin           engine (deps: all arvo + hilavitkutin-api)
hilavitkutin-build     build-time optimisations (standalone)
```

Dependency flow:
```
arvo                 (no deps)
arvo-bits            (deps: arvo)
arvo-bitmask         (deps: arvo-bits)
arvo-graph           (deps: arvo, arvo-bitmask)
arvo-sparse          (deps: arvo, arvo-bitmask)
arvo-comb            (deps: arvo, arvo-bitmask)
arvo-spectral        (deps: arvo, arvo-sparse)
hilavitkutin-api     (deps: arvo, arvo-bitmask)
hilavitkutin         (deps: all above)
hilavitkutin-build   (no deps — standalone build-dep)
```

## Per-crate changes

### arvo
Update DESIGN.md.tmpl: add strategy markers (Hot/Warm/Cold/Precise),
strategy-parameterised UFixed/IFixed, strategy-generic semantic
aliases, Float type alias, container width tables.

### arvo-bits
New crate. BitWidth, BitAccess, BitSequence traits. Semantic aliases
(Bit, Byte, Word, DWord, QWord). Blanket impls for arvo numerics.

### arvo-bitmask
Update DESIGN.md.tmpl: dependency changes from arvo to arvo-bits.
Mask types built on BitSequence/BitAccess contracts.

### arvo-sparse
Update DESIGN.md.tmpl: generic over numeric trait bounds for matrix
values. CSR generic over value type.

### arvo-graph
Update DESIGN.md.tmpl: generic over numeric trait bounds for weights.
Show trait bound signatures.

### arvo-spectral
Update DESIGN.md.tmpl: generic over numeric traits. Weight→float
conversion at API boundary via Into<F>.

### arvo-comb
Update DESIGN.md.tmpl: generic over numeric trait bounds for costs.
Greedy grouping fully generic via closures.

### hilavitkutin-api
Update DESIGN.md.tmpl: ColumnValue interaction with strategy markers.
BIT_WIDTH is logical width, size_of is physical (container) width.
Platform trait parameters note arvo newtypes for final API.

### hilavitkutin
Update DESIGN.md.tmpl: note that concrete arvo type instantiations
happen here (the application), not in arvo sub-crates (libraries).

### hilavitkutin-build
No changes needed. Already correct from T4 verification.
