# Deliverables

## S1. `hilavitkutin-extensions` source conformance

Rewrite in place:

- `src/error.rs`: `ExtensionError` variants become `LinkFailed { cause: LinkError }`, `DescriptorMissing`, `DescriptorInvalid`, `AbiVersionMismatch { expected: u32, got: u32 }`, `ExtensionVersionUnsupported`, `RequiredHostCapabilityMissing { cap: CapabilityId }`, `InitFailed { status: ExtensionAbiStatus }`, `ShutdownFailed { status: ExtensionAbiStatus }`. Enum stays `#[non_exhaustive] #[repr(C)]`. Unused variants from prior source (`Link`, `NotSupported`, `InvalidArg`, `Internal`, `InitFailed { platform_code }`) are removed.

- `src/descriptor.rs`: `ExtensionAbiStatus` becomes `#[repr(u32)]` with variants `Ok = 0`, `InitFailed`, `InvalidArg`, `NotSupported`, `Internal`. The current variants (`Exact`, `HostNewer`, `ExtensionNewer`) are dropped; they modelled a different concept (version-comparison discriminant) that has no place in the DESIGN. Version comparison is expressed in `ExtensionError::AbiVersionMismatch` directly, carrying `expected` and `got` `u32` values. `ExtensionVersion` gains a fourth `u16` padding field (named `_reserved`, value zero) for future expansion as the DESIGN declares. `CapabilityId` changes from `#[repr(C)] pub struct CapabilityId(pub u64)` to `#[repr(transparent)] pub struct CapabilityId(pub u64)`. `from_name` body is unchanged.

- `src/traits.rs`: `ExtensionMeta::NAME` becomes `&'static [u8]`. `InitHandler::init` return type becomes `ExtensionAbiStatus`. `ShutdownHandler::shutdown` return type becomes `ExtensionAbiStatus`. `CapabilityExport` gains `const VTABLE_PTR: *const c_void` alongside existing `const ID: CapabilityId`.

- `src/host.rs`: `ExtensionHost::new` accepts `host_capabilities: &'static [CapabilityId]` and stores it. `load` signature becomes `load(&self, path: &[u8], requirement: ExtensionRequirement, host_ctx: *mut c_void) -> Outcome<Maybe<Extension>, ExtensionError>` using notko `Maybe` (not core `Option`). Per-call `host_ctx` threads into descriptor validation (`init_fn` gets this specific call's pointer) and is stored into the returned `Extension` so `shutdown_fn` sees the matching pointer. `required_host_caps` verification: for each id in the descriptor's required list, check membership in `host_capabilities`; on miss, emit `ExtensionError::RequiredHostCapabilityMissing { cap }` and route through the failure policy. `has_capability` helper reads from the stored set instead of unconditional `false`. `load` converts `ExtensionAbiStatus::Ok` vs non-`Ok` from `init_fn` into `ExtensionError::InitFailed { status }` when non-`Ok`.

- `src/extension.rs`: `Extension::capability` returns `Maybe<*const c_void>` reading `vtable_ptr` from the matched `CapabilityEntry`. The `&CapabilityEntry`-returning shape is removed. `Extension::close` runs `shutdown_fn(host_ctx)` and returns `Outcome<(), ExtensionError>` carrying `ShutdownFailed { status }` when non-`Ok`. `Drop` runs the same sequence and swallows the error.

- `src/lib.rs`: exports updated to match. The `Outcome<Option<Extension>, _>` imports flipping to `Outcome<Maybe<Extension>, _>` drive one edit at the host site; otherwise lib.rs re-exports are stable.

## S2. `hilavitkutin-extensions-macros` full emission

Rewrite `src/lib.rs` so `#[export_extension]` emits the full v1 shape the DESIGN declares:

- Parse attribute parameters via hand-written `syn` (per DESIGN non-scope: `darling` deferred). Supported keys: `name = "..."`, `version = "MAJOR.MINOR.PATCH"`, `required_host_caps = [PATH, PATH, ...]`. Defaults: `name` from struct ident in kebab-case; `version` via `env!("CARGO_PKG_VERSION")` parsed at build time; `required_host_caps` empty slice.

- Parse the target item as `ItemStruct`. Reject `impl`-block placement for v1 (DESIGN defers it).

- Emit the four components:
  1. `const CAPABILITIES: &[::hilavitkutin_extensions::CapabilityEntry] = &[ ... ];` populated from capability entries known at macro expansion. For v1, capability discovery mechanism: the attribute carries an implicit contract that each `CapabilityExport` impl for the target type declares `const ID` + `const VTABLE_PTR`; the macro does not introspect arbitrary `impl` blocks (DESIGN). Consumers with multiple capabilities declare them via consumer-domain macros that wrap `#[export_extension]`, or pass a `capabilities = [...]` attribute (v1 accepts a `capabilities = [TypePath, TypePath]` list of types implementing `CapabilityExport`; macro emits each as `CapabilityEntry { id: <T as CapabilityExport>::ID, vtable_ptr: <T as CapabilityExport>::VTABLE_PTR }`).
  2. `const DESCRIPTOR: ::hilavitkutin_extensions::ExtensionDescriptor = ExtensionDescriptor { ... }` filled from attribute params and const-evaluated capability table slice pointer plus length.
  3. `#[no_mangle] pub extern "C" fn __hilavitkutin_extension_descriptor() -> *const ::hilavitkutin_extensions::ExtensionDescriptor { &DESCRIPTOR }`.
  4. Init / shutdown trampolines conditionally emitted when the user's struct has `InitHandler` / `ShutdownHandler` impls. V1 discovery mechanism: attribute keys `init = TypePath` / `shutdown = TypePath` (default: same as the target struct); emission wraps the trait method call in an `extern "C" fn` and installs its address as `DESCRIPTOR.init_fn` / `shutdown_fn`. Omitted keys produce `None` slots in the descriptor.

- Emission uses fully qualified `::hilavitkutin_extensions::*` paths so consumer import layout is irrelevant.

- Error handling: attribute parse errors surface as `syn::Error` compile errors at the attribute span. BACKLOG: rich diagnostics polish (deferred).
