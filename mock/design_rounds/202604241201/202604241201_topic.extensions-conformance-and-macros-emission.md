# Premise

Round 202604240615 locked DESIGN.md.tmpl for both `hilavitkutin-extensions` and `hilavitkutin-extensions-macros`, then shipped source. A post-merge fidelity audit found the source does not match the locked DESIGN on several concrete field, variant, method, and attribute shapes. The `design-doc-source-mismatch` lint accepts the source because it only verifies type-name presence, not internal shape.

The prior round's SRC CL was written under a time-constrained combined commit (ref: the SRC CL deviation note in `feedback_mockspace_build_gate_workflow.md` and the session state memory). Under time pressure, the shipped source introduced shapes that diverged from the DESIGN. The divergences are the bug; the DESIGN is the spec.

# Current state (mismatch ledger)

Each row names the concrete divergence, the DESIGN-declared target, and the shipped-source drift.

## Host layer (`hilavitkutin-extensions`)

- `ExtensionError` variants. DESIGN: `LinkFailed { cause: LinkError }`, `DescriptorMissing`, `DescriptorInvalid`, `AbiVersionMismatch { expected: u32, got: u32 }`, `ExtensionVersionUnsupported`, `RequiredHostCapabilityMissing { cap: CapabilityId }`, `InitFailed { status: ExtensionAbiStatus }`, `ShutdownFailed { status: ExtensionAbiStatus }`. Source: `Link(LinkError)`, `DescriptorMissing`, `DescriptorInvalid`, `ExtensionVersionUnsupported` (no payload), `InitFailed { platform_code: USize }`, `NotSupported { missing: CapabilityId }`, `InvalidArg`, `Internal`. Source missing `AbiVersionMismatch` with payload, missing `ShutdownFailed`, wraps link via tuple variant instead of struct variant, carries wrong type on `InitFailed`.

- `ExtensionAbiStatus`. DESIGN: `#[repr(u32)]` enum with variants `Ok`, `InitFailed`, `InvalidArg`, `NotSupported`, `Internal`. Models the value returned by init and shutdown C-ABI pointers. Source: `#[repr(C)]` enum with `Exact`, `HostNewer`, `ExtensionNewer`. Source conflates ABI-version comparison semantics with abi-status, and uses the wrong repr for a C-ABI return value.

- `ExtensionVersion`. DESIGN: `#[repr(C)]` struct with four `u16` fields major, minor, patch, plus padding slot for future use. Source: three `u16` fields without the padding slot.

- `CapabilityId`. DESIGN: `#[repr(transparent)]` u64 newtype, permitting safe transmute between repr and value for ABI-stable wire formats. Source: `#[repr(C)]` struct wrapper, which is not transmute-equivalent to `u64` across platforms.

- `CapabilityExport` trait. DESIGN: `const ID: CapabilityId` plus `const VTABLE_PTR: *const c_void`. The vtable pointer is how the macro-generated trampoline finds the implementation to call. Source: only `const ID`; vtable pointer missing. Without the vtable const, the macro has nothing to splice into `CapabilityEntry::vtable_ptr`.

- `ExtensionMeta::NAME`. DESIGN: `&'static [u8]` (matches the byte pointer layout inside `ExtensionDescriptor`). Source: `&'static str`. The `str` form forces an `.as_bytes()` marshalling step the DESIGN did not declare and the macro emission cannot skip.

- `InitHandler::init` return type. DESIGN: `ExtensionAbiStatus`. Source: `i32`. The drift forces conversion at every trampoline call site and is not what the macro is supposed to emit.

- `ExtensionHost::new` signature. DESIGN: `new(host_capabilities)` takes the host's advertised capability set at construction. The set is load-bearing for `required_host_caps` verification per the DESIGN's "Shape" section. Source: `new()` no args; capability registry is stubbed to return false. Source drift means `required_host_caps` can never succeed.

- `ExtensionHost::load` signature. DESIGN: `load(path, requirement, host_ctx)` threads `host_ctx` per-call so different extensions can receive different context pointers. Source: `load(path, requirement)` and stores a single `host_ctx` on the host instance. Source drift means all extensions share one context, breaking the DESIGN's "per-extension host-opaque context" claim under the arbitrary-time linking invariant.

- `Extension::capability` return. DESIGN: `Maybe<*const c_void>` (the vtable pointer; consumer casts back to its trampoline type). Source: `Maybe<&CapabilityEntry>` (includes the id; caller-side extra work to extract the pointer). DESIGN is stricter; source leaks an internal type.

- `Outcome<Option<Extension>, ExtensionError>` at `load`. DESIGN: shape is `Outcome<Maybe<Extension>, ExtensionError>` (notko `Maybe`, not core `Option`). Source: uses core `Option`. Violates the stack's no-bare-option discipline. The `no-bare-option` lint is at `warn`-level pre-sweep, which is why this passed.

## Proc-macro crate (`hilavitkutin-extensions-macros`)

- `#[export_extension]` body. DESIGN: emits `const DESCRIPTOR`, `const CAPABILITIES`, `#[no_mangle] pub extern "C" fn __hilavitkutin_extension_descriptor`, per-capability trampolines, and optional init/shutdown trampolines conditional on `InitHandler`/`ShutdownHandler` impls. Attribute params `name`, `version`, `required_host_caps`. Source: skeleton that passes the input struct through and emits a phantom-data marker. Nothing of the DESIGN's declared emission is implemented.

- Emission paths. DESIGN: `::hilavitkutin_extensions::*` qualified. Source: no emission at all, so trivially consistent but also trivially incomplete.

- `trybuild` fixtures in `tests/`. DESIGN: positive and negative cases covering bare struct defaults, explicit attrs, `InitHandler` presence, multiple capabilities, malformed params. Source: no tests directory exists.

# Decision

Source follows DESIGN for every mismatch above. Rationale: docs are the design source of truth (mockspace rule 1). Source that drifted from a locked DESIGN is drift, not an alternative valid state. No per-item negotiation.

This round's SRC CL fixes each host-layer divergence to match DESIGN exactly, implements the full `#[export_extension]` macro emission, and adds the missing integration tests plus `trybuild` fixtures. Doc CL is minimal: BACKLOG housekeeping in `hilavitkutin-extensions-macros` (items like enhanced error messages, `impl`-block mode, and darling integration stay deferred; macro emission itself moves out of deferred status as of this round).

# Arbitrary-time linking invariant check

Each source fix preserves the invariant. Per-call `host_ctx` (fix on `ExtensionHost::load`) is a strict tightening: extensions cannot accidentally share a host context pointer. Per-capability vtable-pointer const (fix on `CapabilityExport`) keeps capability dispatch per-extension. No fix introduces a cross-extension registry, collector, or barrier.
