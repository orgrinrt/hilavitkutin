## CHANGE: hilavitkutin-extensions Capability* RENAMES to Provider*

Files: `mock/crates/hilavitkutin-extensions/src/{descriptor,error,extension,host,lib,traits}.rs`

Renames:

- `CapabilityId` → `ProviderId`
- `CapabilityEntry` → `ProviderEntry`
- `CapabilityExport` → `ProviderExport`
- `RequiredHostCapabilityMissing { cap: CapabilityId }` → `RequiredHostProviderMissing { provider: ProviderId }`
- field `capabilities` → `providers` (descriptor type)
- field `capabilities_len` → `providers_len`
- field `required_host_caps` → `required_host_providers`
- method `Extension::capability(id)` → `Extension::provider(id)`
- doc-comment prose mentions of "capability" → "provider" where they describe the ABI primitives (not unrelated trait-capability concepts)
- consumer-facing const naming convention shifts: `CAP_X: CapabilityId` → `PROVIDER_X: ProviderId` (referenced in the doc CL via the `viola::PROVIDER_NAM` example)

Verification: `! grep -nE "CapabilityId|CapabilityEntry|CapabilityExport|RequiredHostCapabilityMissing|capabilities_len|required_host_caps|fn capability\b" mock/crates/hilavitkutin-extensions/src/` returns zero matches; `grep -nE "ProviderId|ProviderEntry|ProviderExport|RequiredHostProviderMissing|providers_len|required_host_providers|fn provider\b" mock/crates/hilavitkutin-extensions/src/` returns matches across the renamed files.
