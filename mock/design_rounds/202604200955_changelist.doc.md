# Doc Changelist: hilavitkutin sweep — primitive vocabulary + semantic aliases

**Date:** 2026-04-20
**Phase:** DOC
**Scope:** `hilavitkutin` (core engine) + `hilavitkutin-str` +
`hilavitkutin-persistence` + `hilavitkutin-build`. hilavitkutin-api
and hilavitkutin-ctx clean.
**Source topics:**
- 202604200950_topic.hilavitkutin-sweep-primitive-vocabulary.md

## Purpose

Execute the hilavitkutin side of the three-repo primitive-vocabulary
retrofit (Task #72) while applying the "semantic aliases over naked
newtypes" discipline (memory:
`feedback_semantic_aliases_over_naked_newtypes.md`). Replace bare
`Option<T>` / `Result<T, E>` / `bool` / `usize` / `u32` / `u64` /
`u16` / `u8` in public signatures. Use existing domain types
(`UnitId`, `StoreId`, `FiberId`, `Str`, `Cap`, `USize`) where they
exist; introduce new `pub type` aliases in the owning crate where
they don't. Keep std-trait impls and representation-layer accessors
on their original types (escape hatch + `allow-bare-numeric:
tracked: #72`).

## Const-generic `usize` → `Cap`

Five const-generic parameters across four files need type-widening
from `usize` to `Cap` (the arvo capacity newtype):

- `AccessMask<const MAX_STORES: usize>` → `Cap`.
- `DependencyGraph<const MAX_UNITS: usize>` → `Cap`.
- `DirtyMask<const MAX_STORES: usize>` → `Cap`.
- `SieveCache<K, V, const CAP: usize>` → `Cap`.
- `size_morsels<const MAX_FIBERS: usize>` → `Cap`.

Array sizing uses `cap_size(c: Cap) -> usize` from `arvo-bitmask`
(the shared helper tracked in arvo-BACKLOG for L0 promotion). The
existing signature-to-storage pattern already used in arvo-sparse
applies verbatim.

## Pub-field type flips

Two `pub` fields change their surface type:

- `MorselRange.start: u64` → `RecordIndex`.
- `MorselRange.len: u32` → `MorselLen`.

Both are aliases (`= u64` / `= u32`), so consumer code compiles
without edits — but the signature reads semantically.

## Representation-layer escape hatches (tracked #72)

`allow-bare-numeric: tracked: #72` + doc-comment rationale on:

- `hilavitkutin::plan::access::AccessMask::raw -> u128`
- `hilavitkutin::plan::dirty::DirtyMask::raw -> u128`
- `hilavitkutin_str::handle::Str::id -> u32`
- `hilavitkutin_str::handle::Str::__make(id: u32)` (construction)
- `hilavitkutin_str::handle::Str::__runtime(id: u32)` (construction)

These are representation accessors / constructors, not domain-level
APIs. Tracked for eventual newtype collapse if the domain semantics
grow.
