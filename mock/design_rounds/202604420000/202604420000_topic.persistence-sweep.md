# Changes proposed

- **primitives.rs domain newtypes** (`SchemaVersion`, `EvictionWeight`,
  `BitWidth`): wire-format bit widths are load-bearing. Keep the raw
  backing (`u32` / `u64` / `u32`). Add trailing inline `lint:allow`
  covering the struct pub-field and the three `const fn` accessors.
  Rationale line: "wire-format bit width fixed; exact-width refinement
  tracked: #72".
- **sieve.rs `visited: bool`** (line 22): private Slot field; flip to
  `arvo::newtype::Bool`. The type already imports `arvo::Bool` and uses
  it for `is_empty` return. Init `false` → `Bool::FALSE`; assignments
  `slot.visited = false` / `true` → `Bool::FALSE` / `Bool::TRUE`; read
  `if slot.visited` → `if slot.visited.0`.
- **sieve.rs private impl fields** (`head`, `count` at lines 43–44):
  flip to `arvo::USize`. Arithmetic `(self.head + 1) % CAP` →
  `(self.head.0 + 1) % CAP`; array index `self.slots[self.head]` →
  `self.slots[self.head.0]`; count compares use `.0` accessor; inits
  `0` → `USize(0)`.
- **sieve.rs const-generic sites** (lines 41, 47, 186): trailing inline
  `lint:allow(no-bare-numeric) lint:allow(arvo-types-only)` with reason
  "const-generic array size; rust grammar requires usize; tracked: #121".
- **manifest.rs array-size consts** (lines 18, 24): move the existing
  `// lint:allow(no-bare-numeric)` preceding-line comment to a trailing
  inline comment on the same line as `: usize`. Add `arvo-types-only`
  to the allow list. Reason: "array-size const must be usize (rust
  grammar); tracked: #121".
- **string_table.rs lookup return** (line 46): `&[u8]` is the right
  primitive for borrowed byte data from the mmap buffer. Trailing inline
  `lint:allow(no-bare-numeric) lint:allow(arvo-types-only)` with reason
  "mmap byte-slice return; u8 is the canonical byte primitive; tracked:
  #72".
