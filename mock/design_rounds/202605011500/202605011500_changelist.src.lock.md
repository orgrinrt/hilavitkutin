### `mock/crates/hilavitkutin-api/src/macros.rs`

New file. Two declarative macros:

```rust
//! Cons-list-shape macros for `WorkUnit::Read` / `WorkUnit::Write`.
//!
//! `read![T0, T1, T2]` expands to `(T0, (T1, (T2, ())))`. The cons-list
//! shape lets `WuSatisfied` reduce by recursion at any depth (no per-arity
//! cap). Empty `read![]` yields `()`. Single-store `read![T]` yields
//! `(T, ())`.

#[macro_export]
macro_rules! read {
    () => { () };
    ($T:ty $(,)?) => { ($T, ()) };
    ($T:ty, $($rest:ty),+ $(,)?) => { ($T, $crate::read!($($rest),+)) };
}

#[macro_export]
macro_rules! write {
    () => { () };
    ($T:ty $(,)?) => { ($T, ()) };
    ($T:ty, $($rest:ty),+ $(,)?) => { ($T, $crate::write!($($rest),+)) };
}
```

Module declared in `lib.rs`. Macros re-exported through `#[macro_export]` (top-level visibility).

       pub trait Sealed {}

   /// Total element count of a cons-list shape. `LEN` from `AccessSet`
   /// reports immediate-tuple-arity (always 2 for cons-list cells);
   /// `Depth::D` reports recursive total. Impl'd ONLY on `()` and
   /// `(H, R: Depth)`; flat tuples of arity 3+ deliberately do not
