# Changes proposed

## Axis 1 — counts/indices → `USize`

All u16/u32 fields, params, and array element types switch to `USize`.
Matches the pattern established in strategy/resource/adapt rounds.
Uniform replacement keeps the skeleton simple; exact-width refinement
(UFixed<16,0,Hot> vs UFixed<32,0,Hot>) can happen when 5a4 ships the
real threading body.

## Axis 2 — `ThreadHandle(pub u16)` → `ThreadHandle(pub USize)`

Same substitution. The newtype purpose (semantic index identity)
stays; bit-width constraint relaxes from 16→native (`usize`).

## Axis 3 — `u16::MAX` sentinel → named const

Introduce in `assignment.rs`:
```rust
/// Sentinel meaning "core has no trunk assigned" in `trunk_index`.
/// Kept distinct from a valid trunk index by lying above any
/// realistic trunk count.
pub const NO_TRUNK: USize = USize(u16::MAX as usize);
```

Trailing `lint:allow` on that line (references `u16::MAX` + `usize`
cast). The const captures the sentinel semantic at one place; call
sites use `NO_TRUNK` with no numeric reference.

## Axis 4 — const-generic `MAX_CORES: usize`

Trailing `lint:allow(no-bare-numeric) lint:allow(arvo-types-only)
reason: const-generic array size; rust grammar requires usize;
tracked: #121` on the four `const MAX_CORES: usize` sites.

## Derive adjustments

`ThreadPool` and `WakeStrategy` derive `Hash` / `Default` / similar;
after switching to `USize`, some derives may fail (same arvo gap as
round 202604360000). Trim derives where needed; skeleton types have
no consumers gating on those traits.
