## Motivation

The 2026-04-20 cross-repo review surfaced two CRITICAL findings
and three IMPORTANT findings that would block future plan-stage

notes so the next consumer round doesn't re-discover them.

## Changes proposed

### Fix 1 (CRITICAL): Rename `plan::inputs::NodeId` → `UnitId`

`arvo_bitmask::NodeId` wraps `USize` (repr-transparent). The
skeleton's `plan::inputs::NodeId` wraps `u32` (repr-transparent).
Identical names, incompatible semantics. When a future plan-stage
impl round calls `arvo_graph::topo_sort` or
`arvo_sparse::rcm_reorder`, the returned `arvo_bitmask::NodeId`
will collide with hilavitkutin's own `NodeId`.

Rename hilavitkutin-side to `UnitId` (work-unit identifier). No
semantic change, just namespace hygiene. Propagates through
`plan/*`, tests, and any `pub use` re-exports at `lib.rs`.

### Fix 2 (IMPORTANT): `default-features = false` on arvo deps

Add `default-features = false` to the 7 arvo git deps in
`mock/Cargo.toml`'s `[workspace.dependencies]`. Arvo currently
ships `default = []` on all its feature tables, but if any arvo
crate ever gains an optional std/alloc feature as default, a
hilavitkutin no_os build would silently pull it in.

### Fix 3 (IMPORTANT): Mask64/Mask256 swap-target note

The BACKLOG-style comments in `plan/access.rs` and `plan/dirty.rs`
say "swap for arvo-bitmask when const-generic bitmask support
lands". Expand to specify: Mask64 for MAX_STORES ≤ 64, Mask256 for
MAX_STORES ≤ 256, const-generic Mask<N> (arvo BACKLOG) for larger.

### Fix 4 (IMPORTANT): Forbidden-imports lint for arvo-bits

Add a `forbidden-imports` lint entry in `mock/mockspace.toml` (or
wherever the lint config file lives) blocking `arvo_bits::*` in
the `hilavitkutin` crate scope. The engine should reach bit
contracts via `arvo-bitmask` (L2), not `arvo-bits` (L1) directly.

### Fix 5: BACKLOG notes for pre-empted impl issues

Append a `## Cross-repo integration pre-empted issues` section to
`mock/crates/hilavitkutin/BACKLOG.md.tmpl`:

- `Cap` literal in const-generic position (generic_const_exprs
  limitation — needs named local const pattern).
- `upward_rank` weight type `u32` needs `arvo::FromConstant`.
- `spectral_partition` adapter: DependencyGraph bool matrix →
  arvo `Matrix<W, N>` weight.
- `rcm_reorder` length: `cap_size(Cap(USize(MAX_UNITS))) ==
  MAX_UNITS` not expressible as where-clause; use runtime
  `debug_assert_eq!`.
- `block_diagonal` stub returns `bool`; arvo returns
  `(USize, [USize; N])` + `DulmageMendelsohn` struct. Split
  step 6 into two.
