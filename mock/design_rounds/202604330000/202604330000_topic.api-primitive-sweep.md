# Current state

Public contract crate `hilavitkutin-api` currently carries 16 unique
`arvo-types-only` / `no-bare-numeric` warnings across 6 files. Count
doubles to 32 warnings because the two lints fire on overlapping
sites. The warnings break into three structural categories:

1. **Trait associated constants typed as bare primitives.** The
   engine threads `AccessSet::LEN: usize`, `ColumnValue::BIT_WIDTH:
   usize`, `WorkUnit::COMMUTATIVE: bool` through monomorphisation.
   These consts cross API boundaries into consumer code and must
   carry domain types, not bare primitives.

2. **Clock return type.** `ClockApi::now_ns(&self) -> u64` returns
   nanoseconds-since-platform-epoch. Bare `u64` has no semantic
   shape. A domain alias grounded on `arvo::UFixed<64, 0, Hot>`
   names the quantity at the API.

3. **Byte-stream and raw-pointer ABIs.** Five sites are
   fundamentally bytes-in / raw-pointer ABI:
   - `codec::Decoder::feed(..., chunk: &'a [u8], ...)` — byte codec
     input chunk
   - `sink::ByteEmitter: Push<u8> + BulkPush<u8>` — byte-stream
     trait bound (twice: trait def + blanket impl)
   - `platform::MemoryProviderApi::allocate(...) -> *mut u8` — allocator
     raw pointer return
   - `platform::MemoryProviderApi::deallocate(ptr: *mut u8, ...)` — allocator
     raw pointer argument
   - `platform::MemoryProviderApi::protect(ptr: *mut u8, ...)` — page-protect
     raw pointer argument

   Bytes are eight-bit machine bytes by definition; allocator
   return types are raw pointers by ABI contract. Wrapping either
   destroys the contract's usefulness.

# Changes proposed

## Axis 1 — trait associated consts: `usize` → `USize`, `bool` → `Bool`

Sites (post-change):

| File | Before | After |
|------|--------|-------|
| `access.rs:17` | `const LEN: usize;` | `const LEN: USize;` |
| `access.rs:36,47,...` | `const LEN: usize = N;` | `const LEN: USize = USize(N);` |
| `column_value.rs:19` | `const BIT_WIDTH: usize;` | `const BIT_WIDTH: USize;` |
| `column_value.rs:23` | `= core::mem::size_of::<Self>() * 8` | `= USize(core::mem::size_of::<Self>() * 8)` |
| `column_value.rs:32,36,40` | `= 1 / 2 / 4` | `= USize(1) / USize(2) / USize(4)` |
| `work_unit.rs:57` | `const COMMUTATIVE: bool = false;` | `const COMMUTATIVE: Bool = Bool::FALSE;` |

`arvo::USize(pub usize)` is a `#[repr(transparent)]` tuple struct;
the public field lets us construct in const context. `Bool::TRUE` /
`Bool::FALSE` are the canonical const forms.

## Axis 2 — clock instant alias

Introduce in `hilavitkutin-api::platform` a semantic alias:

```rust
/// Nanoseconds since a platform-defined epoch.
///
/// Monotonic instant type returned by `ClockApi::now_ns`. Backed by
/// `arvo::UFixed<64, 0, Hot>`; the `Hot` strategy dispatches to the
/// host's native 64-bit unsigned.
pub type Nanos = UFixed<{ IBits(64) }, { FBits::ZERO }, Hot>;
```

Then:

| File | Before | After |
|------|--------|-------|
| `platform.rs:63` | `fn now_ns(&self) -> u64;` | `fn now_ns(&self) -> Nanos;` |

`Nanos` is exported from `platform.rs` and re-exported via `lib.rs`.

## Axis 3 — structural byte-stream / allocator escapes

These sites carry post-#80 `lint:allow` annotations tracked under
#72. Justification is identical in shape across all six: the lint
fires on a type that is **definitionally** a byte or allocator raw
pointer at the contract's boundary. Reformulation would replace the
byte-stream contract with a byte-emitter-of-a-byte-newtype contract,
converting a one-byte-per-push operation into a wrapping-convert
noop that adds ceremony without shifting any invariant.

| File | Site | Annotation |
|------|------|------------|
| `codec.rs:86` | `bytes: &[u8]` in `Decoder::feed` | `reason: chunked-codec byte stream; bytes are the 8-bit I/O unit of the contract; tracked: #72` |
| `sink.rs:36` | `pub trait ByteEmitter: Push<u8> + BulkPush<u8>` | `reason: byte-stream trait bound; bytes are the 8-bit I/O unit of the contract; tracked: #72` |
| `sink.rs:37` | blanket impl `impl<S: Push<u8> + BulkPush<u8> + ?Sized> ByteEmitter for S {}` | `reason: matches trait bound above; tracked: #72` |
| `platform.rs:23` | `allocate(...) -> *mut u8` | `reason: allocator ABI returns raw pointer by contract; tracked: #72` |
| `platform.rs:31` | `deallocate(ptr: *mut u8, ...)` | `reason: allocator ABI raw pointer; tracked: #72` |
| `platform.rs:39` | `protect(ptr: *mut u8, ...)` | `reason: allocator ABI raw pointer; tracked: #72` |

# Non-goals

- No changes to `access.rs::impl_access_set!` macro body beyond the
  `const LEN` site — the per-arity `impl<T0..>` bodies stay identical.
- No changes to `hint.rs` / `id.rs` / `capability.rs` — already clean.
- No changes to templates (docs/DESIGN/PRINCIPLES). This is a source-
  only round. The new `Nanos` type is mentioned in-code via its
  rustdoc; DESIGN.md.tmpl listings for platform.rs will pick it up at
  next docs regen without manual CL entry.
- No rename of byte-stream contracts (`ByteEmitter`, `Encoder`,
  `Decoder`) to use a `Byte` wrapper type. That is a separate design
  round if ever taken — would ripple through codec, sink, and every
  downstream consumer.
