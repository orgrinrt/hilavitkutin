# Descriptor v1.1 review-fix

The senior reviewer audit on PR #70 (round 202605090011) returned a
`MERGE AFTER FIXES` verdict with three findings. Two require source
edits the round-202605090011 close window cannot reopen. This
review-fix round handles them on the same branch:
`feat/extensions-descriptor-v1-1-abi-hardening`.

## Findings to fix

**F2 (test header comment is stale).** The integration test header at
`mock/crates/hilavitkutin-extensions/tests/descriptor_validation.rs:1-11`
describes a design that was iterated past during round 202605090011
implementation: it claims `validate_descriptor` is `pub(crate)` and
the test reaches it via a "`pub` test hook gated on `cfg(test)`". The
shipped helper is `pub fn` (in `host.rs`, re-exported through
`lib.rs`) so consumers can probe descriptors outside the standard
load path; the test calls the public symbol directly. The header
comment misleads a future reader about both the visibility and the
access mechanism.

**F3 (cast-safety comment).** `mock/crates/hilavitkutin-extensions/src/extension.rs`
has three `as usize` casts on `u32` length fields (`capabilities_len`,
`name_len`, twice for slice construction). The cast cannot truncate
under any supported configuration: every supported target has
`usize >= 32 bits`, and after `validate_descriptor` runs at load time
the length is bounded at `MAX_DESCRIPTOR_LIST_LEN = 1 << 20`. The
soundness chain is implicit and worth a one-line note for the next
reader.

## What this round does NOT fix

**F1 (CL claim wording).** The locked
`202605090011_changelist.src.lock.md:103` describes
`validate_descriptor` as a "private host-side helper". The shipped
function is `pub`. Per the
`cl-claim-sketch-discipline.md` rule, locked CLs are frozen forever;
the formal correction path is deprecate-and-replace via
`cargo mock deprecate` on the round, which is heavy ceremony for a
wording disagreement that has no behavioural impact. The right place
to record the deviation is a note here, plus a tracked task to either
strengthen the lint (post-#318) or perform the deprecate-replace if
audit clarity ever demands it.

The wording-vs-reality contract gap is recorded as task #398 under
this round.

**F4 (CL test count drift, six vs ten).** The locked CL named six
test functions; the shipped suite has ten (the four extras strengthen
the contract per topic decisions D6 and D7 plus the forward-evolution
requirement, all additive). Drift is purely toward stronger coverage,
not weaker. No action required.

## Decisions

### D1. Rewrite the test header comment to match shipped reality

`mock/crates/hilavitkutin-extensions/tests/descriptor_validation.rs:1-11`
becomes a short paragraph: "`validate_descriptor` is part of the
public surface; consumers probing descriptors outside the standard
`ExtensionHost::load` path call it directly. These tests exercise it
the same way: construct a descriptor literal in test memory, hand it
to the helper, assert the surfaced outcome." The reworded comment
reflects what the test does and what the function is.

### D2. Add cast-safety inline comment at the first u32-to-usize site

The cast lives in three places inside `extension.rs`. One inline
comment at the first cast site (`Extension::capability` at the
`capabilities_len as usize` line) carries the soundness statement;
the other two sites are already documented by proximity (the
`SAFETY:` blocks above each `from_raw_parts` call).

The comment text: "the u32 to usize cast cannot truncate: descriptor
went through validate_descriptor at load time, which bounds the
length at MAX_DESCRIPTOR_LIST_LEN (1 << 20) and every supported
target has usize >= 32 bits."
