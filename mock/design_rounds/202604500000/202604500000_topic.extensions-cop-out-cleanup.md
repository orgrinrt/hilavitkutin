# Topic: extensions cop-out cleanup

**Date:** 2026-04-24
**Phase:** TOPIC
**Scope:** `hilavitkutin-extensions` (crate-local surface, non-FFI-crossing types).

## Current state

`hilavitkutin-extensions` ships with three host-side surfaces that skip the semantic-newtype discipline for convenience:

- `DESCRIPTOR_SYMBOL: &[u8] = b"__hilavitkutin_extension_descriptor\0"` — a nul-terminated raw byte slice. Callers use it as a C string; the type does not advertise that.
- `HOST_ABI_VERSION: u32 = 1` — bare `u32` constant. Host-side comparison target only; not transmitted through FFI as a bare `u32` (the FFI field is `ExtensionDescriptor.abi_version: u32`, a separate surface).
- `ExtensionError::AbiVersionMismatch { expected: u32, got: u32 }` — host-side error payload. Not a value that crosses the ABI; it is constructed host-side after reading the descriptor's `abi_version`.

None of the three sits on an FFI-descriptor boundary. Wrapping them in semantic types is a pure host-side cleanup and leaves `#[repr(C)]` descriptor layout untouched.

## Out of scope

- `ExtensionDescriptor` `#[repr(C)]` fields (`abi_version: u32`, `name_ptr`, `name_len: usize`, `capabilities_*`, `required_host_caps_*`). These are the FFI-wire surface covered by #206.
- `CapabilityId(pub u64)`, `ExtensionAbiStatus` repr, `ExtensionVersion { major: u16, ... }`. Same rationale; FFI-wire surfaces, #206 territory.
- `ExtensionMeta::NAME: &'static [u8]`. Pairs with the descriptor's `name_ptr` / `name_len`; changing the trait-level type requires deciding how the FFI pair is represented. Deferred to #206.

## Changes proposed

Introduce an `AbiVersion` newtype and swap the three non-FFI surfaces to use it or to `&CStr`:

- `pub struct AbiVersion(pub u32)` with `#[repr(transparent)]`. Same wire layout as `u32`, named at the host-side semantic level.
- `DESCRIPTOR_SYMBOL: &CStr` via `CStr::from_bytes_with_nul_unchecked` (const, safe because the literal's nul is a compile-time guarantee).
- `HOST_ABI_VERSION: AbiVersion = AbiVersion(1)`.
- `ExtensionError::AbiVersionMismatch { expected: AbiVersion, got: AbiVersion }` — same bits on the wire (the enum is `#[repr(C)]`), stronger type at the API.

The `ExtensionDescriptor.abi_version: u32` FFI field stays as raw `u32`. Host-side code that reads it wraps into `AbiVersion` explicitly before comparison with `HOST_ABI_VERSION`. That keeps the FFI-wire contract intact while giving the host-side error path semantic types.

Descriptor wire layout is unchanged. Only host-side types shift.
