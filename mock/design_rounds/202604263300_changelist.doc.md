## Purpose

Third engine sub-round. Ships the strategy-selection types + the
Scheduler builder-API skeleton per DESIGN domains 21 and 23.

## Deliverables

### Strategy (domain 21)

- `Strategy` enum: `Sequential`, `Adaptive`, `PipeChase`, `Phased`.
- `StrategySelector` trait — plan-time selection based on record
  count + pipeline shape. Default impl follows DESIGN thresholds
  (<10K → Sequential; deep → Sequential; wide → Adaptive/
  PipeChase; mixed → Phased).
- `PhaseStrategy` — per-phase config (MAX_FUSE / BALANCED /
  MAX_SPLIT hints + fiber count + morsel size).

### Scheduler (domain 23)

- `Scheduler<const MAX_UNITS: usize, const MAX_STORES: usize, const MAX_LANES: usize>`.
- `SchedulerBuilder` — `.add::<WU>()` / `.resource::<T>(init)` /
  `.column::<T>()` / `.memory(provider)` / `.threads(pool)` /
  `.clock(clock)` / `.build()`. Static composition (no runtime
  registration).
- `PipelineResult` per-fiber status: `Completed` / `Failed` /
  `Poisoned`.
- `SchedulerMetrics` resource — frame time, morsel counts,
  change-frequency counters (read-only from consumer).
- `ExecutionPlan` meta-resource — cached plan reused across
  frames.
- `LaneAssignment` meta-resource — per-lane WU assignment.

### Deferrals

- Real execution (actual pipeline runs) — needs 5a2/5a3/5a4 to
  be in place.
- Self-hosting meta pipeline (PlanStage → ScheduleReady →
  PassStart → ScheduleEnd) — depends on a working engine first;
  this round ships only the resource types.
- Version stamps parallel arrays — depends on 5a2 defining the
  store layout.
