# Doc changelist: trampoline LTO preservation (revised after probe)

The first iteration of this CL was locked, applied, then the SRC CL ran the verification harness it shipped. The harness caught a real bug: the LLVM linker's identical-code-folding pass merged the two trampolines because their bodies were structurally identical after the consumer's trivial impls inlined. The init trampoline survived; the shutdown trampoline disappeared from the symbol table and the descriptor's `shutdown_fn` slot ended up pointing at the same address as `init_fn`.

The runtime call still worked (the bytes were the right ones for the simple case of "both return Ok"), but any consumer whose init and shutdown impls actually differ at the bytecode level would observe a silent miscompile: shutdown executing init's body. That is exactly the failure mode F7 flagged.

This revised CL captures the corrected design: `#[unsafe(no_mangle)]` on the trampolines is load-bearing (gives external linkage which makes them ineligible for ICF merging), not omitted as initially analysed.

## Comparison to deprecated changelist

Previous (`202605041648_changelist.src.deprecated.md` and the locked-then-deprecated doc CL):

- DESIGN.md.tmpl said the trampolines are "not `#[no_mangle]`", justifying that with collision-risk concerns.
- SRC CL added only `#[inline(never)]` on the trampoline emission.

This revision:

- DESIGN.md.tmpl says the trampolines carry `#[unsafe(no_mangle)] #[inline(never)]`, with the `no_mangle` framed as the load-bearing attribute against ICF and a paragraph dedicated to bounding the collision concern (which is real but addressed by the `RTLD_LOCAL` default that `hilavitkutin-linking` uses).
- The new src CL (drafted in the SRC phase that follows this lock) will add `#[unsafe(no_mangle)]` alongside `#[inline(never)]` on both trampolines.

The new prose, applied to `mock/crates/hilavitkutin-extensions/DESIGN.md.tmpl`:

```markdown
## Trampoline preservation under LTO

The proc-macro emits two trampolines (`__ext_init_trampoline_<Struct>`, `__ext_shutdown_trampoline_<Struct>`) that wrap the consumer's `init` / `shutdown` types. Their addresses are stored as fn-pointer constants in the `__HILAVITKUTIN_EXT_DESCRIPTOR` static. The static is `#[used]`, and the descriptor function (`__hilavitkutin_extension_descriptor`, `#[unsafe(no_mangle)] pub extern "C"`) returns its address. This chain preserves the trampoline bytes under aggressive LTO + cdylib stripping: the export forces the descriptor function to keep, the function returns the static, the static is `#[used]`, and LLVM cannot elide a function whose address has been taken in a preserved static.

The trampolines additionally carry `#[unsafe(no_mangle)] #[inline(never)]`. The `no_mangle` attribute is load-bearing, not cosmetic: when two trampolines have structurally identical bodies (which happens whenever the consumer's `InitHandler::init` and `ShutdownHandler::shutdown` impls inline to the same return value, e.g. both returning `ExtensionAbiStatus::Ok`), the linker's identical-code-folding (ICF) pass merges them into one symbol. The descriptor's two fn-pointer slots then both end up referencing the same address, the runtime call still works, but one of the trampoline names disappears from the symbol table. With `#[unsafe(no_mangle)]` the trampolines have external linkage, which makes them ineligible for ICF merging because the linker cannot prove that no external code observes their distinct addresses. `#[inline(never)]` is additional belt-and-suspenders against an LTO path that would inline a trampoline body into the descriptor static's initializer expression.

The collision concern from `#[no_mangle]` on the trampoline names (`__ext_init_trampoline_<Struct>` and `__ext_shutdown_trampoline_<Struct>`) is bounded: the names embed the consumer's struct ident, so two extensions collide only when they both define a struct with the same name AND the host loads them into the same process namespace AND the namespace is `RTLD_GLOBAL`. The standard `RTLD_LOCAL` (which `hilavitkutin-linking` defaults to) keeps each cdylib's symbols private to the loading scope, so collisions cannot occur across extensions in practice.

A regression test under `hilavitkutin-extensions-macros/tests/lto_smoke.rs` builds a sample cdylib with `lto = "fat"` and asserts the trampoline symbols survive in `nm` output. The test caught the ICF collapse during this round's verification; without `#[unsafe(no_mangle)]` the shutdown trampoline was being merged into the init trampoline. Runtime callability is verified once a real consumer extension exercises the descriptor's fn-pointer.
```
